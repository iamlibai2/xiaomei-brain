## 核心能力三句话

**记住一切** — 跨会话、跨天的持久记忆，Agent 不会"忘了刚才在做什么"。SQLite 全文检索 + LanceDB 向量语义召回，带时间衰减和强度分级。

**自己推进** — 不用等用户发指令。Agent 有自己的目标层级（意义 → 阶段目标 → 执行目标）、欲望驱动（5 维内在张力）、情绪和激素调节。空闲时主动学习、主动推进。

**越用越强** — 从每次交互中提取经验（上下文 → 决策 → 结果 → 教训），下次做得更好。模式记忆跨时间维度发现规律。

## 设计哲学——为什么 Agent 需要感官能力

### 视觉：不是为了"看到你"，是为了"知道对谁说话"

人类不输入密码登录。走进房间，看一眼，就知道谁在。Agent 也应该这样。

```
传统方式：用户输入密码 → 系统查数据库 → 确认身份
小美的方式：摄像头看到人脸 → face_id 识别 → 调取对这个人的所有记忆
                                      → 知道上次聊了什么、他关注什么
```

**人脸识别不是安全验证，是身份感知。** 让 Agent 在没被"告知"的情况下就知道在和谁交互。这是自然交互的基础设施，不是花活。

### 语音：不是为了"听起来像人"，是为了比打字更快

语音是最自然的人类输出方式，但不是为了聊天。是为了效率：

- 打字 60 字/分钟，说话 150 字/分钟
- 语音 + 声纹识别 = 同时完成"说什么"和"谁在说"
- TTS 让 Agent 输出时用户不需要盯着屏幕

**语音是高效信息通道，不是拟人玩具。**

这也是为什么花了两天调试 VoxCPM 的 ONNX 流式输出——不是为了让 Agent "说话好听"，是为了让 Agent 能在真实场景中和人实时对话。RTF 每降 0.1，就离自然对话近一步。预缓冲、FP16 量化、timesteps 压缩——这些优化不是为了炫技，是为了让 Agent 在会场里开口时不卡壳。

### 场景感知（look_around）：Agent 看见的不只是画面，是决策依据

Agent 看到会场里的 20 个人 → 识别出 5 个关键人物 → 调取对他们的记忆 → 决定讲什么内容。

```
Camera → face_id (识人) → Memory (调取每个人的背景)
                          → 发现：3 个人是投资人，2 个是技术背景
                          → Agent 自主决定：定位偏技术，强调壁垒
```

**视觉在这里不是"情绪识别"，是决策输入。**

### 三层能力的统一逻辑

```
感官层（视觉/语音）：让 Agent 能感知现实世界的对象和信息
     ↓
记忆层：让 Agent 知道自己和这些对象的关系和历史
     ↓
驱动/决策层：让 Agent 基于这些信息自主决定做什么、说什么
```

每一层都不是为了让 Agent "更像人"。是为了让 Agent **在没有明确指令的情况下，自己收集信息、自己做判断、自己行动。** 这是"会成长、能自驱"的物理前提。

## 终局场景：Agent 自己去路演

这是小美项目的终极用例，也是所有设计汇聚的方向：

> **Agent 走进会议室，看到投资人。识别每个人的身份和背景。自主决定讲什么——生成 PPT、生成视频、自己讲解。然后根据投资人的表情和问题实时调整策略。打动他们。**

拆解开来看：

| 环节 | 能力 | 对应模块 | 状态 |
|------|------|----------|------|
| 进入会场，看到所有人 | 视觉采集 | Eyes / Camera | 已有 |
| 识别每个人的身份 | 人脸识别 | FaceID (dlib) | 已有 |
| 调取过往互动记忆 | 记忆检索 | LongTermMemory | 已有 |
| 知道每个人的关注点 | 用户建模 | Memory (user_id 隔离) | 已有 |
| 自主决定讲什么 | 意图生成 | L2Engine / Intent | 已有 |
| 生成 PPT | 文件生成 | 工具调用 (shell/file) | 已有 |
| 生成讲解视频 | 视频生成 | 需要音频+画面合成 | 待做 |
| 自己讲解 | TTS 流式输出 | Throat / TTS (VoxCPM ONNX) | 已有（流式架构就绪，RTF ~0.97，持续优化中） |
| 实时看投资人反应 | 微表情感知 | 传统 CV 情绪检测 | 待做 |
| 根据反应调整策略 | 感知→驱动闭环 | BodyState → Drive → Intent | 待打通 |
| 事后记住这次经历 | 经验提取 | MemoryExtractor | 已有 |
| 下次做得更好 | 经验回用 | ProcedureMemory | 已有 |

**这个场景里，每个感官能力都是为了"在没有人类指令的情况下，感知、判断、行动"。最大的不同不在于技术（都是已有的），而在于架构——现有 AI 工具可以在每一步帮人类执行，但小美要的是自己做完整个流程。**

## 解决什么问题

当前 AI Agent 的根本缺陷：

```
LangChain / LlamaIndex:
  "我能编排 LLM 调用，但我不记得上次做了什么。"

AutoGPT:
  "我能自主执行多步任务，但任务完成我就死了。
   下次从零开始。"

CrewAI:
  "我能让多个 Agent 协作，但每个 Agent 没有个体认知。
   没有记忆，没有成长，没有自己的目标。"

OpenClaw:
  "我能做个人助手和工具编排，但重点在通信和集成，
   不是在长期自主运行。"
```

**xiaomei-brain 让 Agent 不是"执行完就死"，而是"一直在运行，一直在成长"。**

## 对标

| 框架 | 核心能力 | xiaomei-brain 的差异 |
|------|----------|---------------------|
| LangChain / LlamaIndex | LLM 应用编排 | 没有记忆、没有自驱、不适合长期运行 |

**xiaomei-brain 是目前开源生态中，唯一一个把"长期运行、自我成长、自主驱动"当成基础设施来做的 Agent 框架。**

## 五层脑架构

```
Layer 4 (Prefrontal):  Purpose + Metacognition
                      目标管理、意图理解、自我反省、内心独白
Layer 3 (Association): Consciousness
                      火焰骨架 + LLM 加柴、意图涌现、深度反思
Layer 2 (Limbic):      Drive
                      情绪（7种）、激素（6种）、欲望（5维）、愉悦中枢
Layer 1 (Sensory):      Body
                      视觉（摄像头/人脸识别）、听觉（声纹/STT）、发声（TTS）
Layer 0 (Foundation):   Memory
                      SQLite 对话日志 + LanceDB 向量索引 + DAG 分层摘要
                      + 经验/叙事/程序/模式 四大记忆类型
```

## 关键设计原则

1. **LLM 只做判断，不做计算。** 情绪衰减、欲望阈值、记忆强度——这些是算法的事。LLM 负责理解、分析、反思，但不负责做数值决策。

2. **代码维护骨架，LLM 添加内容。** 火焰骨架是代码写死的状态机，LLM 在关键节点添加理解和洞察。不是"AI 生成一切"，是"AI 增强框架"。

3. **所有后续设计围绕"提升 Agent 的长期工作能力"，而非"让交流更拟人"。** 情绪系统是为了让 Agent 感知用户状态、调整工作策略。记忆系统是为了积累经验、避免重复错误。自驱是为了在没有指令时继续推进目标。不是为了"更像人"，是为了"更能干活"。

4. **第一人称记忆 = 角色一致性。** 不是技术上的拟人化，而是确保 Agent 对自己的经验有连贯的自我认知——"我上次怎么做的"、"我学到了什么"。
