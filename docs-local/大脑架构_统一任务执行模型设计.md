# 统一任务执行模型设计

## 问题

当前任务执行有两条完全不共享的路径，PACE 内部能力以 if/elif 打补丁：

```
任务 →
  GoalManager._route_goal_by_type()
    ├── 有子目标 → PACE（有 introspection）
    └── 无子目标 → ReAct（无 introspection，蒙眼冲）

PACE 循环内部（runner.py:_run_loop，~500行）：
  LLM输出 → detect_surprises → step_check → InnerVoice
  → if ESCALATE: ...
  → elif REPORT_PARTIAL/CLARIFY: ...
  → elif RETRY_DIFFERENT/SIMPLIFY: {视角切换, 价值重估, IV升级}  ← 3个能力塞在一个分支
  → else: _maybe_auto_advance
```

问题本质：**能力是独立打补丁加上去的，没有一个统一的执行模型把它们串起来**。

## 方案：CognitiveLoop — 统一任务执行循环

所有任务（有子目标 / 无子目标）走同一个认知循环。聊天保持不变。

### 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                    CognitiveLoop                            │
│                                                             │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────┐│
│  │ PERCEIVE  │────▶│ ASSESS   │────▶│  DECIDE  │────▶│ ACT  ││
│  │           │     │          │     │          │     │      ││
│  │ 收集事实  │     │ 产生理解  │     │ 统一路由  │     │ 执行 ││
│  │ 不判断    │     │ 不决策    │     │ 不执行    │     │      ││
│  └──────────┘     └──────────┘     └──────────┘     └──────┘│
│       ▲                                                  │  │
│       └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

四个阶段是管道，不是分支。每个阶段只有明确的输入/输出契约，不关心里面发生了什么。

### 各阶段详解

#### PERCEIVE — "我看到了什么"

只收集事实，不做判断。

```
输入：LLM 原始输出
输出：Observation {
    llm_output: str,
    progress_tag: dict | None,
    tool_names: list[str],
    tool_count: int,
    elapsed: float,
    surprises: list[SurpriseType],  // EMPTY_RESPONSE / TOOL_STORM / GAVE_UP / ...
}
```

对应现有组件：`detect_surprises()` + `parse_progress_tag()` + `_extract_tool_names()`

#### ASSESS — "这意味着什么"

产生理解，不决策。所有信号在此阶段平等产生。

```
输入：Observation + 历史 + 当前目标
输出：Assessment {
    step_check: StepCheckResult,   // CONTINUE / RETRY / CLARIFY / ESCALATE / ...
    iv_signals: {                  // InnerVoice 信号
        retry: bool,               // "方向不对" → 可能需要换视角
        block: bool,               // "等待用户" → 阻塞当前子目标
        insert: list[dict],        // "---INSERT---" → 新子目标
        escalate: bool,            // "做不了" → 升级
    },
    value_ok: bool,                // 价值重估：还值得做吗？
    perspective_tried: bool,
    retry_count: int,
}
```

对应现有组件：`_step_check()` + `_invoke_inner_voice_task_step()` + `_value_reassess()`

#### DECIDE — "我该做什么"

唯一决策点。综合所有信号，按优先级仲裁。

```python
def decide(assessment: Assessment) -> Action:
    """
    路由规则（优先级从高到低）：

    1. iv_escalate + retries >= 2        → ESCALATE
    2. step_check == ESCALATE            → ESCALATE
    3. iv_block                          → EXIT_WAITING
    4. iv_insert 有内容                  → INSERT_SUB_GOALS
    5. iv_retry + not perspective_tried  → SWITCH_PERSPECTIVE
    6. retries == 3 + not value_ok       → SKIP_CURRENT
    7. step_check == RETRY_DIFFERENT     → RETRY
       step_check == SIMPLIFY            → RETRY
    8. step_check == CLARIFY             → EXIT_WAITING
    9. step_check == CONTINUE            → COMPLETE_AND_ADVANCE（或 CONTINUE）
    10. step_check == REPORT_PARTIAL     → COMPLETE_AND_ADVANCE
    """
```

#### ACT — "动手"

根据 DECIDE 输出的 Action 执行。

```
Action 到执行的映射：

  CONTINUE              → LLM stream，保持上下文
  RETRY                 → LLM stream + 注入重试提示
  SWITCH_PERSPECTIVE    → PerspectiveEngine.generate() → 注入上下文 → LLM stream
  INSERT_SUB_GOALS      → PurposeEngine 插入子目标 → 刷新上下文 → LLM stream
  SKIP_CURRENT          → task_executor.apply_skip() → 推进到下一个
  COMPLETE_AND_ADVANCE  → task_executor 标记完成 → 推进下一个
  ESCALATE              → 退出，通知用户
  EXIT_WAITING          → 退出，等待用户
```

### 组件保留，改变的是连接方式

| 现有组件 | 保留 | 新位置 |
|---------|------|--------|
| `detect_surprises()` | 不动 | PERCEIVE 阶段 |
| `parse_progress_tag()` | 不动 | PERCEIVE 阶段 |
| `_extract_tool_names()` | 不动 | PERCEIVE 阶段 |
| `_step_check()` | 不动 | ASSESS 阶段 |
| `InnerVoice` | 不动 | ASSESS 阶段（产生 iv_signals） |
| `_value_reassess()` | 不动 | ASSESS 阶段 |
| `PerspectiveEngine` | 不动 | ACT 阶段（SWITCH_PERSPECTIVE 时调用） |
| `PurposeEngine` | 不动 | ACT 阶段（INSERT/推进时调用） |
| `task_executor` | 不动 | ACT 阶段（跳过/完成/推进） |

### 聊天不变

聊天保持 `conversation_driver._run_react()`，不走 CognitiveLoop。

```
聊天 → _run_react()           # 不动
任务 → CognitiveLoop.run()    # 统一入口（取代 PACE + ReAct 两条路径）
```

## 改动范围

| 文件 | 改动 |
|------|------|
| `metacognition/runner.py` | `_run_loop` 重构为 `CognitiveLoop` 四个阶段方法 |
| `consciousness/goal_manager.py` | `_route_goal_direct` 改为走 CognitiveLoop，不再调 `_run_react` |
| `consciousness/conversation_driver.py` | `_run_react` 保留不动；移除任务模式的 ReAct 分支 |

## 不改动的

- InnerVoice、PerspectiveEngine、surprise detection、step_check
- task_executor、PurposeEngine
- GoalManager 的外部接口
- `conversation_driver._run_react`（纯聊天用）
- 聊天流程
