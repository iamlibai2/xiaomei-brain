## 里程碑 1：评测体系搭建（2周）

**问题：没有量化数据证明"意识系统"有用，所有评价都是主观感受。**

### 1.1 设计评测维度

不是评测"意识是否存在"（不可证），而是评测**意识系统的各子系统对对话质量的影响**：

| 评测维度 | 对照组 | 实验组 | 指标 |
|----------|--------|--------|------|
| 记忆持续性 | 无长期记忆 | 有LongTermMemory | 24h后复现对话准确率 |
| 记忆衰减效果 | 无衰减（全保留） | 有5级衰减 | 上下文token用量 vs 关键信息保留率 |
| 主动行为质量 | 无主动输出 | ActionDispatcher驱动 | 用户对主动消息的正面回应率 |
| 情绪一致性 | 无Drive | 有Drive | 对话情绪连贯性评分(人工/LLM评估) |
| 目标推进率 | 无Purpose | 有Purpose | 多步任务完成率 |
| 生命周期价值 | 无梦境 | 有梦境巩固 | 梦境后记忆保留率提升 |

评分维度：

### 5.3 多 Agent 并发

当前是单进程单用户，API Server 需要多 Agent 并发：

```python
# server/agent_pool.py
class AgentPool:
    """Agent 池：每个 agent_id 一个 Living 实例"""

    def __init__(self, max_agents: int = 10):
        self._agents: dict[str, ConsciousLiving] = {}
        self._lock = threading.Lock()
        self._max = max_agents

    def get_or_create(self, agent_id: str, config: dict) -> ConsciousLiving:
        with self._lock:
            if agent_id not in self._agents:
                if len(self._agents) >= self._max:
                    # LRU 淘汰最久未活跃的
                    self._evict_oldest()
                agent = build_agent(agent_id, config)
                living = ConsciousLiving(agent)
                self._agents[agent_id] = living
                threading.Thread(target=living.run, daemon=True).start()
            return self._agents[agent_id]
```

## 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 评测数据不显著 | 中 | Paper被拒 | 增加场景数量，调整评测维度 |
| "AI意识"引发伦理争议 | 中 | 负面关注 | Paper明确声明"不做意识主张"，强调工程价值 |
| 模型API变动 | 低 | 适配成本 | PromptAdapter层隔离 |
| 开源后无人关注 | 高 | 白做 | 先发Paper建立权威，再开源 |
| LLM成本过高 | 中 | Demo无法持续 | 提供轻量模式（无L2/L3），降低API调用量 |
