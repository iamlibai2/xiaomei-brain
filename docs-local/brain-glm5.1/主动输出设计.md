# ProactiveOutput 设计

> 日期：2026-04-20
> 说明：统一管理 agent 主动说话的能力

---

## 1. 核心概念

所有"agent 主动产生的消息"，统一为 ProactiveOutput。

与被动响应（core.py 的 ReAct 循环）完全不同：

```
被动（core.py）：用户消息 → ReAct → 工具调用 → 回复
主动（ProactiveOutput）：触发条件满足 → 生成内容 → 推送给渠道
```

## 2. 触发源

| 触发类型 | 何时 | 例子 | 需要 LLM |
|---|---|---|---|
| REMINDER | 到期/定时 | "你昨天说今天要交报告" | 否 |
| RECALL | 状态变化/上下文 | "好久没聊音乐了" | 可能 |
| WAKE | 启动醒来 | "早上好，昨天你说..." | 是 |
| IDLE | 长时间空闲 | "我在，有什么需要吗" | 否 |

四种触发源，产出同一种东西：ProactiveMessage。

## 3. 数据结构

```python
class ProactiveTrigger(Enum):
    REMINDER = "reminder"    # 到期提醒
    RECALL = "recall"        # 主动想起
    WAKE = "wake"            # 醒来问候
    IDLE = "idle"            # 空闲主动

@dataclass
class ProactiveMessage:
    content: str                           # 要说的话
    trigger: ProactiveTrigger              # 触发类型
    target_channel: str = ""               # 目标渠道，"" = 全部
    priority: int = 0                      # 优先级，高优先先发
    metadata: dict = field(default_factory=dict)  # 结构化数据
```

## 4. ProactiveOutput 类

```python
class ProactiveOutput:
    """统一管理 agent 主动输出。"""

    def __init__(self, agent_instance):
        self.agent = agent_instance

    def check(self, trigger: ProactiveTrigger) -> list[ProactiveMessage]:
        """检查指定触发类型的主动消息。"""
        messages = []
        if trigger == ProactiveTrigger.REMINDER:
            messages.extend(self._check_reminders())
        elif trigger == ProactiveTrigger.RECALL:
            messages.extend(self._check_recall())
        elif trigger == ProactiveTrigger.WAKE:
            messages.extend(self._check_wake())
        elif trigger == ProactiveTrigger.IDLE:
            messages.extend(self._check_idle())
        return messages
```

## 5. 各触发源实现

### 5.1 REMINDER — 到期提醒

不依赖 LLM，纯规则匹配：

```python
def _check_reminders(self) -> list[ProactiveMessage]:
    """检查到期的提醒。"""
    messages = []
    # 从 reminder_manager 获取到期提醒
    if self.agent.reminder_manager:
        due = self.agent.reminder_manager.check_due()
        for r in due:
            messages.append(ProactiveMessage(
                content=f"提醒：{r.content}",
                trigger=ProactiveTrigger.REMINDER,
                metadata={"reminder_id": r.id},
            ))
    return messages
```

### 5.3 WAKE — 醒来问候

需要 LLM，基于自我定位生成：

```python
def _check_wake(self) -> list[ProactiveMessage]:
    """生成晨间自语。"""
    # 1. 检查到期提醒（复用 REMINDER）
    # 2. 回顾昨天的 growth_log
    # 3. 组装一句"晨间自语"
    #    例如："早上好！昨天你说想学吉他，今天有空试试吗？"
    content = self._generate_wake_greeting()
    if content:
        return [ProactiveMessage(
            content=content,
            trigger=ProactiveTrigger.WAKE,
            priority=10,  # 醒来问候高优先
        )]
    return []
```

## 6. 与 AgentLiving 的集成

AgentLiving 在状态转换时调用 ProactiveOutput（同步调用，非 async）：

```python
class AgentLiving:
    def _on_wake(self):
        # 检查主动输出
        messages = self.proactive.check(ProactiveTrigger.WAKE)
        for msg in messages:
            self._send_proactive(msg)

    def _loop_sleeping(self):
        # 睡眠中也检查提醒
        messages = self.proactive.check(ProactiveTrigger.REMINDER)
        for msg in messages:
            self._send_proactive(msg)

    def _on_wake_up(self):
        # 被唤醒时检查提醒 + 梦境召回
```

## 7. 与渠道的关系

```
ProactiveOutput → ProactiveMessage → Channel.send()

渠道是消息的消费者，不关心消息是主动还是被动产生的。
```

## 9. 实现优先级

1. **REMINDER** — 复用现有 ReminderManager，最快能跑
2. **WAKE** — 晨间问候，是 on_wake 的核心
3. **RECALL** — 最复杂，需要召回策略
4. **IDLE** — 最简单，最后做
