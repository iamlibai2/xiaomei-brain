### 运行方式对比

| 方式 | L1时机 | L2时机 | 优点 | 缺点 |
|------|--------|--------|------|------|
| **同步（主线程）** | onWake时 | 梦境阶段 | 简单、无锁 | 阻塞主循环 |
| **后台线程** | onWake时 | 独立线程定时 | 不阻塞 | 线程安全复杂 |
| **混合** | onWake同步 | 梦境阶段 | 两全其美 | 需要改造 reflect_phase |

### 为什么推荐混合

**L1 同步的理由**：
- 必须在问候前完成
- 规则判断很快（<50ms）
- 结果立即被 ProactiveOutput 使用

**L2 放梦境阶段的理由**：
- 梦境阶段本来就是"后台思考"
- `_reflect_phase()` 已经预留，待实现
- 不引入新线程，避免锁问题
- 用户不感知延迟

**当前架构是纯同步**，后台线程会带来：
- `agent.messages` 线程安全问题
- `self_model` 读写需要锁
- 与主循环状态机冲突

## 意识的持续运行

人脑的意识是**连续流**，不是离散事件：
- 醒着时：持续感知、持续思考
- 睡觉时：梦境也是意识的一种形式

当前架构的问题：**事件驱动**
```
onWake → 意识检查一次 → 结束
```

应该改成：**持续运行**
```
每个状态转换、每个时刻，意识都在运行
形成意识流（stream of consciousness）
```

## 改进方案

## 人类身份的分层

类比人类，身份确实有不同的"稳定度"：

```
┌─────────────────────────────────────────────┐
│              人类身份分层                     │
├─────────────────────────────────────────────┤
│                                             │
│  L0: 先天身份（不可变）                       │
│      - 性别、出生时间、出生地                 │
│      - 这些定义了"我是谁"的基底               │
│      - 改变 = 变成另一个人                    │
│                                             │
│  L1: 基础性格（极难变）                       │
│      - 内向/外向、敏感/钝感                   │
│      - 形成于童年，成年后几乎不变             │
│      - 改变需要重大事件或长期努力             │
│                                             │
│  L2: 认知能力（缓慢变化）                     │
│      - 知识、技能、价值观                     │
│      - 教育和经历影响                         │
│      - 每年可能有微小变化                     │
│                                             │
│  L3: 社会身份（动态变化）                     │
│      - 职位、角色、关系状态                   │
│      - 随环境和选择变化                       │
│      - 可能每天/每周都不同                    │
│                                             │
│  L4: 状态身份（实时变化）                     │
│      - 当前心情、能量、注意力                 │
│      - 每秒都可能不同                         │
│                                             │
└─────────────────────────────────────────────┘
```

## 关键洞察

**稳定性 = 身份认同**

如果一切都可变，就没有"我"了：
- 每天的"我"都不一样
- 无法建立稳定的自我认知
- 无法建立稳定的关系

**变化性 = 成长适应**

如果一切都不可变，就是死物：
- 无法学习、无法成长
- 无法适应新环境
- 无法发展关系

def interpret_changes(self) -> list[str]:
    """L1: 规则解读变化"""
    interpretations = []
    
    # 用户状态变化
    if self.user_idle_duration > 300:
        interpretations.append("用户长时间没说话")
    elif self.user_idle_duration > 60:
        interpretations.append("用户暂时离开")
    
    # 关系变化
    if "relationship_depth" in self.accumulated_changes:
        delta = self.accumulated_changes["relationship_depth"]["delta"]
        if delta > 0.1:
            interpretations.append("关系更进一步")
        elif delta < -0.1:
            interpretations.append("关系有所疏远")
    
    # 能量变化

        interpretations.append("感到疲惫")
