# 主循环重构：从"聊天服务器"改为"行为选择器"

## 问题

当前 `run_conscious_living.py` 的主循环围绕"等消息→聊天"组织生命。自主行为（意图决策、工作、学习）被分散在 Layer2 线程和 surge 周期任务里，三方各干各的：

```
Main Thread:  等消息 → chat → 等 → 等 → 滑向 SLEEPING（不知道 Layer2 在干嘛）
Layer2 Thread: L2加柴 → 意图决策 → 意图入队（自己玩自己的）
surge 60s:     ActionDispatcher → 执行动作（在所有状态都触发，包括聊天中）
```

具体问题：

1. **聊天中 surge 可能突然插入自主工作** — surge 通过 `register_periodic` 注册，`_tick_periodic()` 在所有状态的循环里都会触发（AWAKE/IDLE/SLEEPING/DREAMING），正在跟用户聊天时可能突然跑 `_do_work()`
2. **Layer2 意图生成只有时间驱动** — 空闲 5 分钟或定期 30 分钟触发，不考虑内在欲望状态。意图应该是"想做某事"的冲动，不是"时间到了硬想一个"
3. **SLEEPING 的唯一真实用途是进入 DREAMING** — agent 不需要像人一样睡觉。SLEEPING 是为了 DREAMING，而 DREAMING 里做的是记忆巩固、模式提取等

## 目标

1. **删掉 surge 周期任务** — 意图消费只在 IDLE（没人说话时）进行
2. **AWAKE 保持现有机制** — 等消息→聊天，按配置的 `idle_short`/`idle_threshold` 切换状态，不动
3. **IDLE 重写为自主行为主循环** — 消费意图、执行动作、判断是否该 SLEEPING
4. **拆开 L2 的意图决策和意识涌现** — 两件事本质不同，分开独立调度
5. **Layer2 意图决策增加欲望驱动** — 保留现有时间触发，额外增加欲望阈值触发
6. **Layer0 / Layer2 线程完全不动** — 它们是自主神经系统和默认网络，保持独立

```python
def _loop_idle(self):
    """IDLE 自主行为主循环。

    覆盖 Living._loop_idle()：
    1. 有消息 → 切 AWAKE 聊天
    2. 消费意图队列 → 执行动作
    3. 无事 → 计空闲 → SLEEPING
    """
    while True:
        self._heartbeat_result = HEARTBEAT_NORMAL
        self._tick_periodic(self.state)

        # 1. 先检查消息
        msg = self._wait_message(self.tick_interval)
        if msg is not None:
            self._on_wake_up()
            self._transition(LivingState.AWAKE)
            self._handle_message(msg)
            self._last_active = time.time()
            return

        # 2. 消费意图队列
        if self._load_consciousness:
            si = self.consciousness.get_self_image()
            self._dispatcher.tick(si)
            executed = self._dispatcher.process_queue()
            if executed:
                self._last_active = time.time()
                continue  # 有动作执行了，继续消费

        # 3. 无事可做 → 计空闲 → SLEEPING
        idle_time = time.time() - self._last_active
        if idle_time >= self.idle_threshold:
            self._transition(LivingState.SLEEPING)
            return
```

**AWAKE 不动** — 保持 `Living._loop_awake()` 现有逻辑。

### 2. `consciousness/core.py` — 拆开意图决策和意识涌现

**背景：** 当前 `_should_l2()` 返回 True 后调用 `tick_L2()`，跑两件完全不同的事：

| | 意图决策 | 意识涌现 |
|---|---|---|
| **问题** | "我该做什么" | "我此刻怎样" |
| **方向** | 面向未来（行动） | 面向当下（感知） |
| **触发应是什么** | 欲望推动 / 情境变化 / 空闲 | 内在节律 / 素材积累 |
| **成本** | 轻（选一个意图） | 重（内心独白+事件+叙事+感知） |
| **代码入口** | `tick_L2_intent()` | `tick_L2_emergence()` |

拆成两个独立调度方法：

#### `_should_intent()` — 意图决策触发

```python
def _should_intent(self, agent_state: str = "awake") -> bool:
    """是否需要做意图决策。

    触发条件：
    - 欲望超阈值（欲望驱动，主要机制）
    - 用户空闲够久（时间兜底）
    - 定期审视（低频兜底）
    """
    si = self.self_image
    elapsed = time.time() - self._last_intent_time

    if agent_state in ("sleeping", "dreaming"):
        return False
    if si.body.energy < 0.15:
        return False

    # 冷却
    cooldown = self._cc.l2_cooldown
    if si.body.energy < 0.3:
        cooldown *= 2.0
    if elapsed < cooldown:
        return False

    # 空闲触发（保留）
    if agent_state == "idle" and si.perception.user_idle_duration > self._cc.l2_idle_trigger:
        return True

    # 定期触发（保留）
    if elapsed > self._cc.l2_periodic_interval:
        return True

    # 欲望驱动（新增，仅 IDLE）
    if agent_state == "idle":
        t = self._cc.l2_desire_thresholds
        if si.body.desire_belonging > t.get("belonging", 0.6):
            return True
        if si.body.desire_cognition > t.get("cognition", 0.6):
            return True
        if si.body.desire_achievement > t.get("achievement", 0.5):
            return True
        if si.body.desire_expression > t.get("expression", 0.6):
            return True

    return False
```

#### `_should_emerge()` — 意识涌现触发

```python
def _should_emerge(self, agent_state: str = "awake") -> bool:
    """是否需要做意识涌现（内心独白 + 事件分析 + 社交感知）。

    触发条件：
    - 定期节奏（内在节律，主要机制）
    - 累积状态变化够多（素材驱动）

    注意：欲望不驱动涌现。饿了不会让人突然写日记。
    """
    si = self.self_image
    elapsed = time.time() - self._last_emerge_time

    if si.body.energy < 0.2:
        return False

    # 冷却
    if elapsed < self._cc.l2_emergence_cooldown:
        return False

    # 定期节奏（主要机制）
    if elapsed > self._cc.l2_emergence_interval:
        return True

    # 状态变化积累够多
    if self._state_buffer.should_trigger_l2():
        return True

    return False
```

### 4. `action_dispatcher.py` — `process_queue()` 返回是否执行了动作

```python
def process_queue(self) -> bool:
    """按优先级顺序执行队列中的所有 ActionItem。

    Returns:
        True 表示至少有一个动作被执行
    """
    if not self._queue:
        return False

    self._queue.sort(key=lambda item: item.sort_key())

    executed = False
    for item in self._queue:
        try:
            success = self._executor.execute(item)
            if success:
                self._record_fired(item.cooldown_key)
                executed = True
        except Exception as e:
            logger.error("[ActionDispatcher] 执行失败: %s, error=%s", item, e)

    self._queue.clear()
    return executed
```

```

      ├─ 定期节奏             → tick_L2_emergence()
```
