# 睡眠系统设计

## 睡眠的生物学基础

人的睡眠由两个独立系统共同调控：

### 1. 褪黑素（昼夜节律 / Process C）

- **驱动源**：视交叉上核（SCN）→ 光信号 → 松果体分泌褪黑素
- **节律类型**：严格的日周期（~24h），余弦型
- **曲线特征**：
  - 白天（06:00-20:00）：光线抑制，褪黑素接近零
  - 黄昏（20:00-22:00）：开始缓慢上升（DLMO）
  - 夜间（22:00-02:00）：快速攀升到峰值
  - 凌晨（02:00-04:00）：峰值平台
  - 黎明（04:00-07:00）：快速下降，光线照射后立即抑制
- **作用**：睡眠门控——打开睡眠通道，降低入睡阻力。不是催眠药，不直接产生困意

### 2. 腺苷（睡眠压力 / Process S）

- **驱动源**：ATP 消耗 → 腺苷累积
- **节律类型**：锯齿型，累积-清除循环
- **曲线特征**：
  - 清醒期：线性累积，每小时约涨 5-10%
  - 睡眠期（尤其深睡）：快速清除，6-8h 回到接近零
- **作用**：产生困意本身——腺苷越多，觉醒中枢越被抑制

### 3. 午后困意

中午 13:00-15:00 的困意不是褪黑素引起的（此时褪黑素正处于谷值），而是：

1. **SCN 警觉信号的自然波谷**——演化留下的"午睡窗口"
2. **腺苷累积过半**——从早上醒来后堆了大半天
3. **餐后血糖波动**——碳水 → 胰岛素 → 抑制 orexin（觉醒肽）

实际上是两个睡眠窗口：大窗口在夜间（褪黑素开门），小窗口在午后（警觉信号短暂撤开）。

### 4. 困意 = 褪黑素 + 腺苷 - 警觉信号

三者的组合决定最终的睡眠倾向：

| 系统 | 节律类型 | 对映 agent |
|------|---------|-----------|
| 褪黑素 | 日周期余弦 | 纯粹看时间，22-06 高 |
| 腺苷 | 清醒累积 / 睡眠清除 | AWAKE/IDLE 线性涨，SLEEPING/DREAMING 降 |
| 警觉信号 | SCN 输出 + 去甲肾上腺素 | norepinephrine + 用户活跃度 |

## 已实现：褪黑素

### 数据模型

- **位置**：`drive/state.py` — `HormoneState.melatonin`（0.0 ~ 1.0）
- **初始值**：0.5（`drive/config.py` — `HormoneConfig.defaults.melatonin`）

### 计算公式

```
# 余弦曲线，峰值凌晨2点(0.9)，谷值下午2点(0.1)
hour = now.hour + now.minute / 60.0
melatonin = 0.5 + 0.4 * cos((hour - 2) * π / 12)
```

### 更新时机

- **启动时**：`DriveEngine.__init__()` 末尾调用 `_update_melatonin()`
- **加载后**：`DriveEngine.load()` 末尾调用（防止存储的旧值覆盖）
- **每小时**：`DriveEngine.tick_hour()` 调用 `_update_melatonin()`

### 不受外部影响

褪黑素不参与任何事件处理（`on_praise`/`on_criticism`/`apply_social_signal` 等都不会碰它），不走常规激素衰减循环。`tick_hour()` 中先用衰减循环处理所有激素（褪黑素也会被衰减系数碰一下），紧接着用 `_update_melatonin()` 覆盖为当前时间节律值。

### 渲染

- **位置**：`self_modules.py` — `SelfBody.melatonin` 代理到 `drive.hormone.melatonin`
- **渲染**：`render_consciousness_v3.py` — `_render_body()` 的激素列表中，使用自然语言标签

```
褪黑素标签：
  0.70+："褪黑素很高，身体在告诉你该睡了——困意很重"
  0.50+："褪黑素中等，不困也不精神"
  0.30+："褪黑素很低，身体很清醒，没有睡眠压力"
```

### 配置模板

`cli.py` — `_CONFIG_YAML_TEMPLATE` 中 `drive.hormone` 段：

```yaml
initial:
  melatonin: 0.5  # 褪黑素 — 纯日夜节律驱动
decay_rates:
  melatonin: 0.95 # 占位，实际由日夜节律覆盖
```

## 待实现：腺苷 + 睡眠压力 + 主动入眠

### 腺苷建模思路

```
# tick_minute() 或 tick_hour()
if agent_state in (AWAKE, IDLE):
    adenosine += 0.01 * (1.0 - adenosine)   # 线性累积，上界 1.0
elif agent_state in (SLEEPING, DREAMING):
    adenosine *= 0.95                        # 指数衰减

# 渲染到 body
if adenosine > 0.7:  "睡眠压力很重，脑子转不动了"
if adenosine > 0.4:  "有些困了，反应开始变慢"
```

### 睡眠压力 = 褪黑素 × 腺苷

两者的交互不是加法：褪黑素高 + 腺苷高 = 强烈的睡眠冲动。褪黑素低 + 腺苷高 = 午后那种"困但还能扛"。

### 主动入眠

跟 GREET/CARE 一个模式，睡眠压力超阈值 → L2 产生 REST 意图：

```
sleep_pressure > 0.8 → Intent REST → "我困得不行了，先去睡会儿" → 主动切 SLEEPING
```

这比现在的被动模式（用户离开 N 秒 → 睡）更自然——agent 不是突然消失，而是有社交过渡。

### 睡眠质量与能量恢复

- 褪黑素高 + 皮质醇低 → 高效恢复，energy += 0.15/h
- 褪黑素高 + 皮质醇高 → "睡不好"（困但睡不着），energy += 0.05/h
- 褪黑素低 → 即使休眠也恢复慢
