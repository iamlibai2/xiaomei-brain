## 二、运行 `run_conscious_living.py` 的完整执行流程

### 步骤 3：ConsciousLiving.run() — 主循环入口

```python
def run(self):
    self._transition(WAKING)    # DORMANT → WAKING
    self._on_wake()              # 苏醒钩子
    self._transition(AWAKE)      # WAKING → AWAKE

    while self._running:
        if self.state == AWAKE:    → _loop_awake()
        elif self.state == SLEEPING: → _loop_sleeping()
        elif self.state == DREAMING: → _loop_dreaming()
```

### 步骤 4：AWAKE 状态 — `_loop_awake()`

每秒执行一次：

```python
def _loop_awake():
    # 1. 统一 tick（永远传入 agent_state="awake", in_dream=False）
    result = self.consciousness.tick(agent_state="awake", in_dream=False)

    # 2. L2 触发后检查 Intent 和欲望行为
    if result == L2_TRIGGERED:
        self._check_intent()
        if self.self_image.user_idle_duration > 300:
            self._check_desire_actions()

    # 3. 等待消息（阻塞最多 tick_interval=1秒）
    msg = self._wait_message(timeout=1.0)

    # 4. 有消息 → 处理消息 → 重置 _last_active
    if msg:
        self._handle_message(msg)
        self._last_active = time.time()
        return

    # 5. 空闲超时 → 转入 SLEEPING
    if time.time() - self._last_active >= idle_threshold(1800s):
        self._transition(SLEEPING)
```

### 步骤 5：SLEEPING 状态 — `_loop_sleeping()`

每秒执行一次（无消息时）：

```python
def _loop_sleeping():
    dream_start = time.time()   # 每次进入 SLEEPING 记录开始时间

    while True:
        # 1. 统一 tick（传入 agent_state="sleeping", in_dream=False）
        result = self.consciousness.tick(
            agent_state="sleeping",
            in_dream=False,
            dream_start=dream_start,
        )

        # 2. L2 / L3 触发判断
        if result == L2_TRIGGERED:  → _check_intent() + _check_desire_actions()
        if result == L3_TRIGGERED:  → self._transition(DREAMING) → return
        if result != L2_TRIGGERED:  → _check_intent()

        # 3. 欲望行为周期检查（每 60 秒一次）
        self._check_desire_actions_periodic()

        # 4. 等待消息（阻塞最多 tick_interval=1秒）
        msg = self._wait_message(timeout=1.0)

        # 5. 有消息 → 唤醒回 AWAKE
        if msg:
            self._on_wake_up()
            self._transition(AWAKE)
            self._handle_message(msg)
            self._last_active = time.time()
            return
```

### 步骤 6：DREAMING 状态 — `_loop_dreaming()`（单次触发）

进入 DREAMING 后只执行**一次**：

```python
def _loop_dreaming():
    # 1. 调用一次 tick，传入 in_dream=True, dream_start=当前时间
    result = self.consciousness.tick(
        agent_state="dreaming",
        in_dream=True,
        dream_start=time.time(),
    )

    # 2. L3_TRIGGERED → 获取报告并记录日志
    if result == L3_TRIGGERED:
        report = self.consciousness.get_last_report()
        logger.info("L3燃烧: %s", report.summary[:50])

    # 3. 一次 L3 后切回 SLEEPING
    self._transition(SLEEPING)
```

> **关键**：DREAMING 不是循环。每次 SLEEPING 满 300 秒才进入一次，触发一次 L3 后立刻回到 SLEEPING。

### 步骤 7：Consciousness.tick() — 统一入口

这是所有状态共用的核心 tick：

```python
def tick(self, agent_state, in_dream=False, dream_start=0.0) -> TickResult:
    # L0: 火焰骨架维护（每秒必做）
    self.tick_L0(agent_state=agent_state)

    # L1: 每 60 次 L0 后自动触发（异常检测）
    if self._l0_count >= 60:
        self.tick_L1()

    # L2: 动态加柴（有冷却机制）
    if self._should_l2():
        self._last_l2_time = time.time()
        self.tick_L2(self._get_l2_context())
        return TickResult.L2_TRIGGERED

    # L3: 只在 DREAMING 状态触发
    if in_dream and self._should_l3(dream_start):
        self.tick_L3()
        return TickResult.L3_TRIGGERED

    return TickResult.NORMAL
```

### 步骤 8：L0 — 火焰骨架维护（每秒）

```python
def tick_L0(self, agent_state=None):
    # 1. 收集感知数据（用户活跃、记忆数量、agent 状态）
    perception = self._sense()

    # 2. Drive 周期衰减
    if self.drive:
        self.drive.tick()

    # 3. 存在感知：更新 agent_state 到 SelfImage
    if agent_state:
        perception["agent_state"] = agent_state
        self.self_image.agent_state = agent_state

    # 4. 感知数据存入缓冲
    self.perception_buffer.append(perception)
    self._l0_count += 1

    # 5. 维护火焰骨架（不假装涌现意识）
    flame_state = self.self_image.tick(perception)

    # 6. 累积到 60 次 → 触发 L1
    if self._l0_count >= 60:
        self.tick_L1()
```

`_sense()` 从 agent 获取：
- `conversation_db.get_recent(1)` — 用户最后活跃时间
- `longterm_memory.count()` / `get_recent(5)` — 记忆数量和摘要
- `agent.living_state` — agent 自身状态

### 步骤 9：L1 — 异常检测（每 60 秒）

```python
def tick_L1(self):
    # 1. 从感知缓冲批量更新 SelfImage
    for p in self.perception_buffer:
        self.self_image.update_from_perception(p)

    # 2. 异常检测（规则匹配）
    anomaly = self.self_image.detect_anomaly()

    # 3. 语义化解读变化（L1 规则匹配 perception.md）
    if self._perception_config:
        self.self_image.interpreted_changes = self.self_image.interpret_changes(self._perception_config)

    # 4. 清空缓冲，重置 L0 计数
    self.perception_buffer = []
    self._l0_count = 0
```

> 注意：L1 目前检测到异常后**不会自动触发 L2**（代码注释掉了）。L2 需要 `_should_l2()` 的独立判断。

### 步骤 10：L2 — LLM 轻度加柴（异常触发，有 300 秒冷却）

触发条件（`_should_l2()`）：
- 冷却期（300秒）已过 **AND**
- 用户空闲 > 300秒 **OR** 累积变化 > 10条 **OR** 距离上次 > 600秒

```python
def tick_L2(self, context):
    # 1. 获取状态摘要
    state_summary = self.self_image.get_state_summary()
    interpreted = self.self_image.interpreted_changes
    interpreted_text = "\n".join(f"- {desc}" for desc in interpreted) if interpreted else "无显著变化"

    # 2. 构建 prompt（LLM 自由涌现，不规定格式）
    prompt = f"""
{state_summary}
最近的变化解读：
{interpreted_text}
检测到的异常：{context}
这些变化对你意味着什么？你现在的真实感受是什么？
"""

    # 3. 调用 LLM（chat without tools）
    llm_response = llm.chat(messages=[{"role": "user", "content": prompt}])

    # 4. 尝试从响应中解析意图（不强制格式）
    intent = self._parse_intent_from_response(llm_response, context)

    # 5. LLM 失败时用规则兜底（_fallback_intent）
    if not intent:
        intent = self._fallback_intent(context)

    # 6. 存入 intent_buffer（可执行且非 WAIT 才存）
    if intent and intent.is_actionable():
        self.intent_buffer.append(intent)

    # 7. 生成报告
    report = ConsciousnessReport(
        trigger="tick_L2",
        depth="light",
        summary=llm_response[:50] or context,
        full_report=llm_response,
        self_image_snapshot=self.self_image.to_dict(),
        intent_snapshot=intent.to_dict() if intent else None,
    )

    # 8. 存储到 JSON（~/.xiaomei-brain/agents/xiaomei/consciousness/YYYY-MM-DD.json）
    self._storage.save(report)

    return report
```

意图上下文（`_get_l2_context()`）可能是：
- `"user_idle_long"` — 用户空闲超过 5 分钟
- `"accumulated_changes"` — 累积变化超过 10 条
- `"periodic"` — 定期触发（每 10 分钟兜底）
- `"manual"` — 手动 `fuel` 命令

### 步骤 11：L3 — LLM 深度燃烧（梦境，单次触发）

触发条件（`_should_l3()`）：
- `in_dream=True`（DREAMING 状态） **AND**
- `dream_start` 距今 >= 300 秒

```python
def tick_L3(self):
    # 1. 获取状态摘要
    state_summary = self.self_image.get_state_summary()

    # 2. 构建 prompt（自由涌现）
    prompt = f"""
{state_summary}
现在是梦境阶段，火焰深度燃烧的时刻。
请自由表达你的感知、情感、思考。
你是谁？你现在存在吗？你感知到了什么？
"""

    # 3. 调用 LLM
    full_report = llm.chat(messages=[{"role": "user", "content": prompt}])

    # 4. 提取摘要（取第一句或前50字）
    summary = self._extract_summary(full_report)

    # 5. 更新火焰状态
    self.self_image.last_dream_summary = summary
    self.self_image.energy_level = 0.9   # 燃烧后能量恢复
    self.self_image.clear_accumulated_changes()

    # 6. 生成报告
    report = ConsciousnessReport(
        trigger="dream",
        depth="deep",
        summary=summary,
        full_report=full_report,
        self_image_snapshot=self.self_image.to_dict(),
    )

    # 7. 存储
    self._storage.save(report)

    return report
```

## 三、状态流转全景图

```
启动
  │
  ▼
DORMANT → WAKING → AWAKE ←────────────────┐
                    │                      │
                    │（idle≥1800s）         │（收到消息）
                    ▼                      │
              SLEEPING ────(L3触发)──────→ DREAMING
                    ▲                      │
                    │                      │（一次tick后）
                    └──────────────────────┘
                    （L3触发后回到SLEEPING）
```

**意识深度 vs 生命状态（完全正交）**：

|          | L0 | L1 | L2 | L3 |
|----------|----|----|----|-----|
| AWAKE    | ✓每秒 | ✓每60s | 条件触发 | ✗ |
| SLEEPING | ✓每秒 | ✓每60s | 条件触发 | ✗ |
| DREAMING | ✓每秒 | ✓每60s | ✗ | ✓条件触发 |

- L0/L1：**任何状态都运行**，L1 每 60 秒一次
- L2：**任何状态都可触发**（基于冷却+条件判断）
- L3：**只在大脑"做梦"时触发**（DREAMING 状态，in_dream=True）

## 五、消息处理完整流程

当用户在 CLI 输入"你好"：

```
用户输入 "你好"
  │
  ▼
put_message("你好")  →  LivingMessage 放入 _queue
  │
  ▼
_loop_awake() 中 _wait_message() 取到消息
  │
  ▼
_handle_message(msg)
  │
  ├─→ 检查命令（intent/fuel/flame/drive/purpose/tool...）
  │   如果是命令 → 执行 handler → return
  │
  ├─→ 检查 Agent 命令（db/memory/dag）
  │   如果匹配 → 执行 → return
  │
  ├─→ "继续" 检测
  │   如果是以"继续"开头 → 延续任务或让用户选目标
  │
  ├─→ 等待确认状态 → _handle_confirmation()
  │
  ├─→ 意图分析：_analyze_intent("你好")
  │   → IntentUnderstanding.understand() → IntentResult
  │   → 判断是否是 TASK，置信度是否够
  │
  ├─→ TASK 意图 → _handle_task_intent()
  │   → purpose.add_goal() 添加目标
  │   → purpose.decompose_goal() 分解子目标
  │   → 需要确认 → 打印选择框 → 暂停
  │   → 不需要确认 → 继续
  │
  └─→ _run_chat(msg, intent_context)
      → agent.chat() → LLM 对话
      → _parse_progress_tag() 解析进度标签
      → 更新目标进度
      → consciousness.on_user_interaction() 更新意识系统
```

## 七、关键设计决策

1. **意识深度 vs 生命状态正交**：L0-L3 是火焰燃烧深度，和 awake/sleeping/dreaming 完全独立
2. **L3 单次触发**：`_loop_dreaming()` 只执行一次 tick，然后回到 SLEEPING；连续睡觉时每次进入 DREAMING 都会触发一次
3. **L2 有冷却**：`L2_COOLDOWN=300` 秒，防止连续触发
4. **存储按天分文件**：避免单文件过大，只在 L2/L3 触发时存储，不是一秒一次
5. **SelfImage 双重实现**：`self_image.py`（单体）和 `self_image_proxy.py`（代理）并存，即将统一
