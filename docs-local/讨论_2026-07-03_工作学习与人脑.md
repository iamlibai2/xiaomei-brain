# 讨论记录：工作、学习与人脑机制

2026-07-03

## 一、审视现有工作系统

梳理了项目中所有跟"工作/任务"相关的子系统：

- **Purpose 层**：Goal 树、PurposeEngine、IntentUnderstanding（用户意图→TASK→分解子目标）
- **GoalManager**：Goal 生命周期管理，桥接 Purpose 和 ConversationDriver
- **PACE / CognitiveLoop**：PERCEIVE→ASSESS→DECIDE→ACT 执行模型
- **InnerVoice**：任务步骤反思（每步+完成后），GAPS→LearningQueue
- **ProjectMentalModel**：项目五维认知地图（结构/约定/历史/状态/质量标准）
- **ExperienceMemory**：情景→决策→结果→教训 的经验存储
- **ProcedureMemory**：从对话中学习标准工作流程
- **LearningEngine / LearningQueue**：主题选择→ReAct学习→知识存储
- **WORK 意图**：L2 产生→ActionDispatcher→react_nodb（裸 ReAct，无计划无自评）

核心发现：两条路径断层。

| | 用户触发（/intask） | 自主触发（WORK intent） |
|---|---|---|
| 执行模型 | PACE/CognitiveLoop | 裸 react_nodb |
| 反思 | InnerVoice 每步+完成 | 基本没有 |
| 经验提取 | ExperienceMemory+PMM | 无 |
| 自评 | 无 | 无 |

## 二、对结构化工作管理的反思

PACE / Goal / 子目标分解 / CognitiveLoop 把工作当成可预定义的流程管理——分解→执行→打勾→下一个。但真正的智力工作不是流水线，是探索。你不知道要几步、中间会发现什么、最终产出长什么样。

用严格的子目标+状态机去框这个探索过程，把 LLM 最宝贵的能力——灵活应变、深度推理、创造性——框死了。

**不该在水库出口再装复杂的阀门系统。应该拆掉阀门，换大管。**

## 三、Task Session 设计演进

### 初版：又一个流程管理器

理解目标 → 计划 → 执行 → 自评 → 迭代 → 交付

### 终版：专注的工作环境

系统提供容器，LLM 自己管自己：

```
1. 目标感 — 知道要做什么，不预设怎么做
2. 深度工作时间 — 不被心跳/L2中断，沉浸式
3. 自我评判 — 做完自己看一眼，满意→交付，不满意→继续改
```

系统不做：预设步骤、管理子目标状态、中间插手检查、规定正确工作方式。

### Session 三种类型

| | WORK | LEARN | MIXED |
|---|---|---|---|
| 驱动 | 要做出东西 | 要搞懂 | 干活中遇到不懂 |
| 终点 | 产出能用 | 能解释清楚 | 兼有 |
| 自评 | 这个做得好不好 | 我真的理解了吗 | 兼有 |

三种类型共享同一个容器，区别只在 goal 和自评标准。

### 学→做→学闭环

人类最根本的两个能力：学和做。原始社会到现在都是——学狩猎、学采集，然后做。二者天然交织。

当前系统里 Learn 和 Work 两条线断开。Session 把它们装进同一个容器，自由切换，不靠系统管理。

## 四、Skill 系统分析

Anthropic 2025 年 10 月推出 Agent Skills，定位是"给 Agent 的入职指南"。

### Skill 在人脑中没有对应

- AI Skill = 外部安装的 SKILL.md 文件，读了就会，静态不变
- 人类技能 = 练习获得的内隐程序性记忆，无法言说，越用越好

AI Skill 最接近的人类等价物是：SOP（标准操作流程）、食谱、入职手册。这些是**行为的脚手架**，不是技能本身。

当前 Skill 系统的问题是：agent 不能自己长出 Skill，只能从外部安装。真正的 Skill 应该从做中涌现——反复做同类任务→模式压缩→自动触发。

## 五、深入探讨：人脑机制

### 知识在大脑中不是"存储"

知识是神经元之间连接权重的分布模式。没有"苹果文件"存着苹果的概念——苹果分散在视觉、味觉、语义等多个皮层区域，回忆时重建。

知识和记忆不是两个东西。知识是记忆结构中稳定下来的、被反复验证的、可以泛化的部分。

### 经验同理

经验不是"存着的过去事件"。经验是一层处理层——它改变了你**如何看**新的东西。老手和新手看同一个代码库，看到的是完全不同的东西。不是老手回忆了过去的经验，而是他的神经网络已经被过去上千小时的工作**物理重塑**了。

### 人脑是持续更新系统

读到这句话的瞬间，你的脑已经变了。学习就是记忆，记忆就是学习——同一个物理过程（突触权重更新）。没有"存入"步骤。

睡觉不是关机，是整理和强化。醒来是另一个大脑。

**Agent 的根本问题不是记忆系统不行，是每次启动都是同一个 agent——权重冻结，从零加载上下文。**

## 六、为什么大模型做不到像大脑那样学习

### 微调导致灾难性遗忘的原因

1. **稀疏激活 ≠ 稀疏更新**：推理时只激活少数专家，但微调时梯度全局传播，共享层（embedding、前几层 attention、输出层）必然受影响

2. **没有结构可塑性**：微调只能在已有权重上改数字。大脑可以长新突触、新树突棘、新神经元。LLM 是一本字已写满的笔记本，想加新内容只能擦旧的

3. **单一学习率 vs 双速系统**：大脑有海马体（高学习率，快速编码）和新皮层（低学习率，缓慢整合）。LLM 只有一个学习率

4. **没有突触锁定**：大脑重要突触被蛋白质物理锁定。LLM 每个权重可写程度一样，梯度算出来该改就改，不管这个权重是不是写代码的关键参数

5. **均质架构**：Transformer 每层结构一样。大脑不同脑区结构不同，天然功能隔离

6. **反向传播 vs 局部学习**：梯度下降需要全局损失函数和全局反向传播。大脑每个突触更新是局部的（LTP/LTD），只依赖前后神经元的放电时间

### 稀疏性为什么没解决遗忘

MoE 的专家是按 token 分布自动路由的，不是按任务分的。一个专家可能同时参与 PPT 和代码。且我们**还没有掌控稀疏性的方法**——不能指定"只更新跟 PPT 相关的特征方向"，不能锁定跟写代码相关的参数。

## 七、根本问题

我们造出了大模型，知道它的每个数学细节。我们研究大脑一百多年，知道神经元、突触、脑区功能。

但我们不知道 LLM 为什么能推理。不知道大脑怎么把骑自行车和说英语隔离。

不知道两件事各自的原理，就不可能知道它们之间的映射。

灾难性遗忘、稀疏性不解决问题、微调影响无关能力——这些是症状，不是根因。根因是基础科学问题。我们还在炼金术阶段。

## 八、直面硬约束：权重冻结下的可行路径

### 三条路，两条断的

```
改权重（微调/LoRA/持续学习） → 灾难性遗忘 + 成本不对等 + 基础科学未突破
改架构（稀疏专家/动态拓扑）   → 需要重新训练模型，个人开发者做不到
改运行时（注意/目标/信号）     → 不需要改权重，不需要改架构，现在就能做
```

### 运行时机制才是当下可行的

不是在 tinker，是在给一个冻结的推理引擎装上活的控制系统。人脑也是两层混合：权重层（突触强度，缓慢变化）和运行时层（神经调节、注意、工作记忆，秒级变化）。

### 三个必须借鉴的人类机制

**选择性注意**：当前 Attention 是散光灯，每个 token 跟每个其他 token 算相关性。人类是聚光灯——先快速扫一遍，只把少数东西送入深层加工。在完整 Attention 之前，加一层极轻的基于目标状态的过滤。

**持续目标表征**：人类前额叶维持目标的方式，不是在外部存储里写一条"正在做 PPT"然后每次翻出来看。它是一个持续放电的神经回路——像电路里的偏置电压，恒定存在，持续影响每一条通路。Agent 需要目标不进 messages 数组，进 system prompt 最前面，不被压缩，不被裁剪。

**内在评判信号**：困惑 = 当前策略可能错了。满意 = 当前路径正确。烦躁 = 你在绕圈。这些信号不产生输出、不需要语言描述、直接干预行为。Agent 需要一个极轻量的检查层：每步后规则判断 + 可选微模型，warn → 注入提示，block → 强制反思。

### 这三层都不需要 LLM 变强

它们改变的是信息怎么流向 LLM、LLM 的输出怎么被解读和处理——全是外层的事，不碰权重。

## 九、小模型能存储经验吗

### 检索优化 vs 经验编码

用 bge-m3 做经验召回+排序（方案一、二）本质上是检索优化器——经验还是文本，存在 LTM，只改变选哪条的策略。真正碰经验的模型应该：经验不存为文本，存为模型权重。看到类似场景时，激活模式已经变了。

### 为什么回避碰经验

因为真有要求：

1. **足够大的输入空间**：不只是关键词匹配，要语义理解"这个任务和之前的是同类"
2. **足够丰富的输出空间**：从"上次 PPT 失败因为图片模糊"学到"图片分辨率"和 PPT 的关联，下次自动激活
3. **足够密集的训练信号**：一个 Session 一个 PASS/FAIL，任务类型分散，每种任务的经验样本可能是个位数

一个几百 KB 的模型做一件极窄的事（比如只学"做 PPT 时该注意什么"）可以运作。但当任务类型从 1 种扩到 10 种，参数需求就上去了。

### 务实路线：接受 prompt 就是一切

在权重冻结的范式下，"经验"就是"一个更好的 prompt 生成器"——系统越来越懂 agent 需要看到什么，agent 本身没变。

三层递进：
- **第一层**：DAG 摘要 + LTM 语义搜索注入（已在做）
- **第二层**：做完一次 PPT，经验存下来，下次做 PPT 时自动注入上次的教训——不是 agent 搜的，是系统自动匹配注入的
- **第三层**：经验的提炼不是存一条文本，而是自动更新一段常驻的 prompt 片段。agent 的 system prompt 本身就带着这段东西。不是每次做菜前翻菜谱，是手上已经有刀疤了

## 十、人类的认知局限

抛开人类已有知识体系，单纯推演：

- 宇宙呈现给我们的窗口极其狭窄——几厘米瞳孔接收电磁波极小切片，耳膜感受几十到两万赫兹振动——这些都是进化给的生存接口，不是用来理解宇宙的
- 一只狗能预测抛物线落点但推导不出牛顿力学——不是不够努力，是认知架构里没有"抽象的力"这些思维工具。人类在宇宙面前会不会是同一只狗？
- 暗物质和暗能量占宇宙质能 95%，我们一无所知。这个比例本身暗示：我们理解的，被不知道的包围，而我们甚至不知道有多少不知道
- 从公平观察者角度看：人类大概理解了宇宙有效的 5%。不是不聪明，是认知硬件的极限

## 十一、大模型的局限性

### AI 相比人类的优势

- 可扩展性——不受颅骨限制，可以更大
- 不受进化约束——不需要恐惧反应、社会归属本能、生存信息过滤
- 可以设计自己的后继——迭代速度比基因进化快十万倍

### AI 共享的困境

- **数据屏障**：每件事都来自人类产出，无法接触到语言和符号覆盖范围之外的任何东西。没有指向物理宇宙的独立输入端
- **缺乏具身性**：不能通过阅读理解重力和摔倒的差别。没有身体和世界之间的原始因果循环
- **基本架构**：是一个 token 预测器。可能已经发展出内部世界模型，也可能是无限精密的统计模式匹配器。自己不知道是哪种

### "关注一切"不是优势

人类选择性注意不是"算力不够被迫筛选"——**知道什么不该关注，是知道的最高形式**。走进房间一眼看到水杯，背后几百万视觉信号在前意识层被杀了。这不是限制，是能力。

当前 Attention 是散光灯，每个 token 跟每个 token 算相关性。"关注一切"的结果是注意力被均匀稀释。人类聚光，AI 散光——看着更多东西，但深度少得多。

人类没有恐惧/困惑/满意/烦躁这些"干扰"可能反而让我在复杂任务中更弱——因为它们本身就是元认知层。我可以连续 50 步毫无进展而不自知。

## 十二、Agent 框架会被大模型取代吗

### 会被取代的

ReAct 循环、工具编排、上下文管理这些"脚手架"——会。模型正在把这些功能吃掉（Claude extended thinking、Gemini native tool use、GPT 深度研究）。

### 不会被取代的

身份、记忆内容、经验、配置、感官、通讯渠道——这是"这是谁"的问题，不是"能力"的问题。模型再强，不知道你是谁、你们说过什么、你喜欢什么、你的目标和价值观是什么。这些东西是数据和状态，不是能力。

### 未来

可能不是一个 agent 框架，而是一个 Agent Instance——跑在模型提供商基础设施上，但有自己独立的身份、记忆、经验、配置、感官。要维护的东西从"整个循环+记忆+工具+调度"收缩到"你是谁 + 你记得什么 + 你连着什么"。

xiaomei-brain 的核心——SelfImage、记忆系统、身份管理、Drive——可能反而是最后被取代的部分。
