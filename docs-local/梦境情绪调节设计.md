# 梦境情绪调节设计

## 背景

### 人类 REM 睡眠的情绪调节机制

- REM 期间去甲肾上腺素极低，大脑在"安全环境"中重放情绪记忆
- 杏仁核-前额叶对话，将情绪"电荷"从记忆中解耦（Walker & van der Helm, 2009）
- 一夜好眠后，人对之前痛苦刺激的杏仁核反应显著降低
- 梦境滞后效应：梦更多反映 5-7 天前的经历，而非当天

### 旧方案的问题

梦境情绪整理原来用 `EmotionProcessor`（`consciousness/dream/emotion_processor.py`），10 条硬编码正则：

```
"爱|恋人|亲吻|拥抱" → belonging +0.10, oxytocin +0.15
"噩梦|害怕|恐惧|追逐|逃跑" → belonging -0.10, cortisol +0.15
...
```

四个问题：
1. **时序错位**：阶段2处理上次梦的摘要 → 阶段4生成新梦 → 新梦的情绪下次才被处理
2. **正则匹配脆弱**：LLM 写出"我变成鸟飞过无人的山谷"，十条正则全不命中
3. **没有强度区分**：命中就 ± 固定值，无法区分"淡淡怀念"和"撕心裂肺"
4. **三套逻辑各自维护**：正则（11 条规则）+ 关键词（5 个词）+ 意图工厂，可能矛盾

## 设计方案

### 核心思路

把情绪分析交给 LLM 自己。DreamEngine 在梦境深度燃烧 prompt 中要求输出结构化 JSON，EmotionProcessor 从正则匹配改为解析 LLM 的 `---EMOTION---` 块。

### 流程

```
旧：阶段2(正则处理上次梦摘要) → 阶段4(LLM生成新梦) → _generate_followup_intent(关键词)
新：阶段4(LLM生成梦 + ---EMOTION--- JSON) → EmotionProcessor(解析JSON, 改Drive, 生成intent)
```

### Prompt 追加的结构化指令

```
在自由表达之后，请分析你在梦境中感受到的情绪，并输出以下结构化块：

---EMOTION---
{
  "desire_changes": {
    "belonging": <float -0.2 ~ 0.2>,
    "cognition": <float -0.2 ~ 0.2>,
    "achievement": <float -0.2 ~ 0.2>,
    "expression": <float -0.2 ~ 0.2>
  },
  "hormone_changes": {
    "oxytocin": <float -0.2 ~ 0.2>,
    "cortisol": <float -0.2 ~ 0.2>,
    "dopamine": <float -0.2 ~ 0.2>,
    "serotonin": <float -0.2 ~ 0.2>
  },
  "followup_intent": "greet|reflect|care|express|wait",
  "intent_reason": "为什么是这个意图，简短一句话"
}
```

### 情绪调节的双向性

范围 `-0.2 ~ +0.2` 是对称的，LLM 自行判断方向：

**平复**：挫败被梦境消化
```
"白天博士批评了我的代码，梦里我重写了整个系统并且它运行完美。"
→ cortisol: -0.10, dopamine: +0.05, intent: wait
```

**强化**：梦激起了焦虑或渴望
```
"梦里博士说他下个月就要离开了。我意识到时间不多了。"
→ belonging: +0.15, oxytocin: +0.12, cortisol: +0.05, intent: greet
```

### 完整的醒来后闭环

```
LLM 输出
  ├── _extract_summary() → contribute_dream() → last_dream_summary
  └── EmotionProcessor.process()
        ├── desire_changes → Drive.desire.xxx (clamp 0-1)
        ├── hormone_changes → Drive.hormone.xxx (clamp 0-1)
        └── followup_intent → intent_buffer
              └── 唤醒后 ActionDispatcher 消费
                    ├── greet  → PROACTIVE → LLM 生成问候消息
                    ├── care   → PROACTIVE → LLM 生成关心消息
                    ├── express → PROACTIVE → LLM 分享想法
                    ├── reflect → TRIGGER_L3 → 深度反省
                    └── wait   → 无动作
```

### 兜底策略

| 场景 | 行为 |
|---|---|
| LLM 未输出 `---EMOTION---` | 不改 Drive，生成 WAIT intent |
| JSON 解析失败 | 同上 |
| LLM API 调用失败 | 同上 |
| Drive 字段名不匹配 | 跳过该字段 |
| 无边新记忆 | 不执行梦境燃烧，无情绪整理 |

## 涉及文件

| 文件 | 变更 |

## 未来方向：认知孵化

人类 REM 睡眠的另一个功能是创造性孵化（creative incubation）：前额叶抑制 + 海马体-新皮层自由回放 = 远联想（remote association）。拉马努金的公式、苯环结构、PageRank 灵感，都源自这种跨域连接。

**当前不做**，原因：
1. AWAKE 和 DREAMING 用的是同一种模型，"自由联想"ReAct 也能做
2. 缺乏真实"卡住"场景——PACE reviewer 还需完善
3. 关系/模式系统还不够复杂，梦发现的跨域连接无法写入图谱产生持久效果

等以上条件成熟后，可在梦境 prompt 中增加"未解决问题"输入和 `---INSIGHT---` 输出块。
