# xiaomei-brain v0.1.1 Release Notes

## Body Sensory System

Multi-modal perception pipeline, enabling the agent to interact with the physical world.

- RealMicrophone: direct microphone input via PyAudio, replacing text-only interaction
- RealCamera: camera capture pipeline for visual awareness
- Tactile channel: body signals fed into SelfImage, driving emotion and desire modulation
- VoxCPM TTS: natural speech synthesis for voice output

## Speaker Recognition & Attention Gating

Voiceprint-based identity resolution with noise-aware attention control.

- Automatic speaker login: voiceprint matching identifies the speaker without manual identity switching
- Wake word detection: fuzzy matching tolerates 1-character STT errors (e.g., "bot" vs "boot"), case-insensitive
- Attention gating: ZCR (Zero Crossing Rate) speaker detection + VAD noise filtering — only activates on actual human speech
- Multi-user session switching: conversation context automatically transitions when speaker identity changes
- Voiceprint registration quality assessment: validates enrollment quality before accepting

## Dynamic Tool Loading (Tool RAG)

Embedding-based tool retrieval to prevent prompt token explosion as tool count scales.

- 100+ tools indexed, only 20 sent per step (7 core + dynamic top-K)
- Per-step embedding recall: LLM only sees tools relevant to the current context
- Step growth mechanism: +3 dynamic slots per ReAct step, capped at 50
- Supports unlimited MCP servers without blowing up context window
- Sequential Thinking MCP integrated for structured multi-step reasoning
- Measured recall rate: 100% (30/30 test scenarios), ~200ms overhead per step

## MCP Integration & Config Hot Reload

- 4 MCP servers out of the box: filesystem, playwright, chrome-devtools, sequential-thinking
- Config file change detection with automatic reload — no restart required
- Dynamic index rebuild on MCP/plugin changes (ConfigReloader + notify_tools_changed)
- Tool registration consolidated to single init path with lazy dependency binding
