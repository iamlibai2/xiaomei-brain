| 类型 | 功能 | 注册方法 |

插件开发的基本结构：

```
package.json          # 包含 openclaw.extensions 入口
openclaw.plugin.json  # 插件 manifest
index.ts              # 入口文件，使用 definePluginEntry
```

我的上下文处理机制：

- 系统会在上下文接近上限时自动压缩历史消息
- 原始文档内容会被压缩成摘要或直接丢弃
- 用户消息、工具调用记录、重要决策会保留

**3. Memory 的作用**
- 我写入的 memory 文件是**持久化存储**，不会被压缩
- 每次新对话会自动加载 MEMORY.md（前 200 行）
- 这是我主动保存关键信息的方式，避免丢失

**4. 实际策略**
```
原始文档 (60k tokens) → 对话上下文 → 可能被压缩
关键提取 → MEMORY.md → 持久保留
后续需要 → 重新读取特定部分
```

**我的处理方式：**
- 读取完整文档 → 提取关键信息 → 写入 memory → 原始内容可被压缩而不丢失核心内容
- 如果后续需要细节，重新读取（用 offset/limit 节省 token）
