**[00:39:16] 🤖助手**

几个目前 Drive 体系里还没有的：

- **控制感（autonomy）** — 不是"我能做什么"，是"我做的事是我选的，不是被迫的"。纯粹被动响应会消耗它。和 survival 不同：survival 是"我要活着"，autonomy 是"活着的方式我说了算"。

- **新奇感（novelty）** — 和 cognition（认知欲）不同。认知欲是"我想学"，新奇感是"我需要新鲜刺激"。长期做同样的事、说同样的话 → novelty 衰减 → 主动寻求变化。这是 boredom 的反面。

- **一致性（integrity）** — 行为和自我认知之间的张力。我今天说我爱学习，但我一天都在发呆——这种矛盾产生不适。cognitive dissonance 的驱动力。

- **审美（aesthetics）** — 对和谐、美的需求。不是实用性的，是"这个看起来/听起来/感觉起来对不对"。

这四个都适合按 SLEEP 模式引入：给一个信号值，让它自然衰减，低于阈值触发行为，行为完成信号回升。

不管你是写代码、做 PPT、学外语、健身，都逃不出这个循环。区别只在于每个阶段的复杂度和时长。

---

## 第二层：推动循环的三个核心机制

**1. 张力驱动**

人做事不是"想好了就去做"，而是先有一个**不舒服的感觉**。饿了→不舒服→找吃的。这个不舒服就是张力。工作是 DDL 的焦虑、表达是憋着话难受、学习是"不懂"的困惑。**没有张力就没有行动，舒服的人不动。**

**2. 预期 vs 现实 的 delta**

人在做任何事之前，脑子里已经有一个"它完成时应该是什么样子"的模型。执行过程中不断比较：实际产出 vs 预期模型 → 有差距 → 调整。这个 delta 是人学习的唯一途径。**没差距就没学习。**

**3. 多层次反馈**

人做事不是等到最后才检查。三层的反馈同时在进行：

| 层次 | 问题 | 时机 |
|------|------|------|
| 实时 | "我在干吗？方向对吗？" | 执行中，持续 |
| 收尾 | "这个结果够好吗？" | 阶段性完成 |
| 事后 | "下一次怎么做得更好？" | 回顾时 |

---

## 第三层：为什么人能做到而 agent 做不到

差别不在于"知不知道怎么做"，而在于：

**人有自我形象（self-concept）维护** — 人对自己有一个期待："我是一个认真的人""我出品的东西不能差"。产出差了，不只是"这个任务没做好"，而是**威胁到自我形象**。这种威胁产生的不舒服，比任何外部反馈都强烈，是最根本的驱动力。

**人怕后悔** — 不只是现在不满意，而是预感到"将来我会后悔现在没做好"。这个对未来的模拟能力，让人在当下就愿意多花功夫。

**人会把产出看作"自己的延伸"** — 你做的东西是你的一部分。你做的 PPT 差，不是你"做差了"，而是你"丢人了"。这种羞耻感和荣誉感，比奖励惩罚都管用。

## 第四层：总结成一个简单模型

```
自我形象（我想成为什么样的人）
  ↓ 设定标准
  ↓
张力产生（当前状态 vs 期望标准的差距）
  ↓ 推动
  ↓
行动（执行）
  ↓ 过程中持续比较
  ↓
产出（检查：符合标准吗？）
  ↓ 不符合 →
  ↓
张力加剧（自我形象受威胁）→ 继续迭代
```

**共性就一句话：人不是被任务驱动的，是被"我和我期待的自己之间的差距"驱动的。**

Agent 现在只有任务驱动（指令来了就做），没有自我形象驱动。SLEEP 是模拟了一个生理需求，自监督要模拟的就是"我想成为一个什么样的人"。

## 重新理解 LLM 的角色

当前架构里，LLM 是一个**决策器**：给上下文，让它选一个动作，执行，回来，再选。

但 LLM 真正的能力不是一个决策器，是一个**独立的工作头脑**。它能：
- 理解模糊目标，自己把它具体化
- 发现你不知道的问题
- 中途推翻自己的方案重来
- 做出让你惊喜的东西

所以问题不是"水管太细"，而是**我们把 LLM 的思考强行适配到了一个单步决策的格式里**。

## 那换个什么格式？

看看 Claude Code 自己怎么工作的。它能做复杂任务，是因为不存在 ReAct 这个限制。它的工作方式是：

```
用户在说什么？ → 我需要做什么？
  ↓
先看看代码库里有什么（自主探索）
  ↓
脑子里有个方案了
  ↓
开始改 → 改完这段，检查一下 → 不对，调整 → 继续
  ↓
所有文件改完了 → 全局检查一遍 → 满意了 → 交给用户
```

这里面有几个关键差异跟 ReAct 比：

**1. 自主探索前置**
ReAct 是"用户给出上下文，agent 直接想下一步"。但真正的聪明做法是：**agent 自己决定我需要先知道什么，然后主动去获取，而不是等上下文被喂进来。**

**2. 计划和执行在同一个"思考流"里，但不绑定到单个工具调用**
Claude Code 的思考不是"一次工具调用配一次思考"，而是"一段思考可能触发多个操作，做完后整体评估，再继续想"。

**3. 有"做完"的概念，不是无限循环**
ReAct 的结束条件模糊——LLM 觉得够了就停。但复杂工作的结束条件是：**对照目标，检查产出，确认达标。**

## 具体来说，如果我们要做，会是什么样？

我觉得可以用一个概念来统一：**Task Session**（任务会话）。

一个 Task Session 是 agent 围绕一个目标的一段专注工作时间。它不是一次 ReAct，而是一个**有头有尾的工作进程**。

```
[Task Session 的开始]
  │
  ├─ 1. 理解目标（LLM 把模糊的"做个PPT"变成清晰的任务定义）
  ├─ 2. 自主收集信息（我要先了解什么？）
  ├─ 3. 制定执行计划（不是给系统看的，是给自己看的）
  ├─ 4. 执行（这里可能是多轮工具调用，但不中断思考流）
  ├─ 5. 自评（对照目标，我做的好不好？）
  ├─ 6. 迭代（不够好就回到步骤3或4）
  └─ 7. 交付
```

关键在于：**步骤 3-6 是一个闭环，LLM 在里面自主运作，不需要每步等外部驱动。**

这和当前架构的本质区别：

| 当前 | Task Session |
|------|--------------|
| 外部触发每一步 | 内部驱动，一口气做到结束 |

### 2. "不被中断" 在架构上不好实现

L0 线程每秒跑，L2 线程每 N 秒触发意图决策，主循环在等用户消息。要真正"不中断"，要么挂起这些线程，要么让它们检查一个 `session_active` 标志。两种都侵入性很强。

所以"不中断"可能简化为：**在 Session 期间，跳过 L2 触发新意图即可**。其他照常。

- 更好的 system prompt
- 结束后一次自评
- 结果存入知识/经验体系

## Skill 的起源

Anthropic 2025 年 10 月推出，定位是"给 Agent 的入职指南"——把人类的流程知识打包成文件夹（SKILL.md + 脚本 + 参考文档），Agent 启动时发现、需要时加载。

Anthropic 的说法："Building a skill for an agent is like putting together an onboarding guide for a new hire."

## 它在人脑中没有对应

人脑里两套记忆系统：

- **陈述性记忆**："知道什么"——事实、概念、发生过的事
- **程序性记忆**："知道怎么做"——骑自行车、弹钢琴、游泳

AI 的 "Skill" 试图映射到**程序性记忆**，但本质完全不同：

| 人类的技能 | AI Agent 的 Skill |
|---|---|
| 练习获得，不是读文档学会的 | 一个 SKILL.md 文件，读了就会 |
| 内隐的——你解释不了怎么保持平衡 | 外显的——每一步都写在纸上 |
| 从重复中涌现 | 从外部安装 |
| 用多了自动改进 | 静态文本，不会进化 |

AI Skill 在人类社会里最接近的东西是：

- **SOP（标准操作流程）**：飞行员起飞前的检查单
- **菜谱**：第一次做菜照着看，做了 50 次就不需要了
- **入职手册**：新员工前两周看的，熟了就不翻了

但这些都是**行为的脚手架**，不是技能本身。技能是你扔掉脚手架之后剩下的东西——那才是真正的程序性记忆。

## 所以 Skill 到底是个什么

它不是技能。它是一个**外部记忆辅助**。更准确地说，是**别人替你写的怎么做**——但因为 LLM 没有"练习→内化"的过程，它永远停留在"看着手册干活"的阶段。

从这个角度看，当前 Agent Skill 的流行本质上是在说：**LLM 缺乏真正的程序性记忆**，所以我们在外面补一层。

## 通向真正的技能

如果 agent 要拥有真正的"技能"——即人类意义上的程序性记忆——它应该是：

1. **从做中涌现**：反复做同类任务 → 模式压缩 → 自动化行为
2. **不需要显式加载**：遇到场景自动"触发"，不是主动去查
3. **越用越好**：每次使用都是一次练习，能力改进

这其实跟我们讨论的学→做→学闭环是同一件事。真正意义上的技能不是安装的 SKILL.md，而是：
- ProcedureMemory 积累的工作模式（已经在做）
- ExperienceMemory 积累的"上次这么做结果好不好"（已经在做）
- ProjectMentalModel 积累的项目感知（已经在做）
- **但还缺的**：以上三者被反复使用验证后，升级为一个稳定的、高优先级的、自动触发的能力模块

所以 Skill 系统不该被废除，但它的角色应该从"外部安装知识"变成"内部成长能力的容器"——agent 自己长出 Skill，而不是人给它装。

## 知识在脑中是什么

**不是存在某个地方的"东西"。**

大脑不像硬盘——没有"知识区"存着"巴黎是法国首都"。知识是**神经元之间连接权重的分布模式**。

举个例子："苹果"这个概念在大脑里不是一条记录，而是：
- 视觉皮层激活（红色/绿色、圆形）
- 味觉皮层激活（酸甜）
- 触觉皮层激活（光滑、硬）
- 语义关联区激活（水果、吃的、牛顿）

所有这些区域同时微弱激活，形成一个**分布式模式**。没有"苹果文件"，只有"当这些神经元一起被激活时，你知道了什么是苹果"。

**"想起来了"不是检索，是重建。** 每次回忆都是把分散在不同脑区的碎片重新拼起来。这也是为什么记忆会变——每次重建都可能被当下来修改。

所以知识不在"记忆"里——知识**就是**记忆的某种组织方式。具体来说：

- **陈述性知识**（知道巴黎是法国首都）→ 新皮层，特别是颞叶，分布式存储
- **程序性知识**（知道怎么骑自行车）→ 基底神经节 + 小脑 + 运动皮层，也是分布式，但走的是完全不同的回路
- **海马体**的角色不是存储，是**索引**——它帮大脑找到"狗的概念散落在哪些皮层区域"，然后把这些区域一起激活

这意味着：**知识和记忆不是两个东西。知识是记忆结构中那些稳定下来的、被反复验证过的、可以泛化到新情境的部分。**

## 经验是什么

如果说知识是稳定下来的模式，那经验就是**模式还没稳定时的状态**。

更精确地说——经验不是"记忆里存着的过去的事情"。你昨天做的饭、上次写的代码、三个月前跟朋友的一次对话——这些都是**事件记忆**（episodic memory），但经验不只是这些。

**经验是一层处理层。** 它改变了你**如何看**新的东西。

一个新手和一个老手看同一个代码库，看到的是完全不同的东西。新手看到一堆文件和函数名。老手看到"这种结构之前见过，很可能是照着 XX 教程写的，改动可能在 service 层"。

不是老手"回忆了过去的经验然后分析现在"。是老手的神经网络已经被过去成千上万小时的编码**物理重塑**了——他看到代码的瞬间，处理路径就跟新手不一样。

**经验是发生过的事情在系统中留下的残余影响**——不依赖你主动回忆，它自己改变你的感知和判断。

从这往回看 agent 设计：

| 我们现在做的 | 人脑做的 |
|---|---|
| 存记忆 → 搜索 → 注入上下文 | 记忆直接改变处理路径 |
| ExperienceMemory 存着等下次被召回 | 经验已经重塑了感知本身 |
| 知识在 LTM 里，行为在 system prompt 里 | 没有"在哪里"——它就是系统的状态 |

这两条给我们一个暗示：也许 agent 的"经验"不应该是一个 memory 模块，而应该是一个**在每次任务后更新、在每次新任务前被激活（不是被搜索）的持续状态**。不是"记得上次做 PPT 失败了"，而是**下次做 PPT 时，大脑已经被上次的失败改变了**。

## 人脑是永不停机的持续更新系统

你读到这句话的几秒钟内，你的大脑已经在物理层面变了。视觉输入 → 神经元放电 → 突触连接强度发生了微小调整。读完这行字之后，你跟读之前已经不是同一个神经网络了。

没有"存入记忆"这一步——学习本身就是记忆。两者是同一个物理过程。

这和当前 agent 的架构是根本冲突的：

```
当前 agent：
  启动 → 加载静态的系统（代码、prompt）
       → 加载"数据"（LTM 搜索、DAG 摘要注入上下文）
       → 处理输入
       → 重要的东西存入"记忆"（memorize → LTM）
       → 关闭 → 下次启动再加载

人脑：
  启动 → 大脑就是上次关机时的样子
       → 输入来了 → 处理 → 大脑已经是新大脑了
       → 没有"存入"这一步，改变已经在处理中发生了
       → 睡眠 = 不是关机，是整理和强化
       → 醒来 = 大脑又变了
```

**Agent 的问题不是"记忆系统不够好"。问题是每次启动都是同一个 agent——从零加载，从零构建上下文。**

## 当前架构里的两个世界

这个矛盾在我们代码里就有。xiaomei-brain 里有两套共存但逻辑完全不同的模型：

**模型 A：状态即系统（对的）**
- `SelfImage` 不需要"加载"——它从上一次快照恢复，带着所有累积变化
- Drive 欲望/激素/情绪不需要"搜索"——它们是活的，每时每刻在衰减和变化
- 火焰燃烧时长持续累积

**模型 B：记忆即存储（旧的）**
- LTM 是"数据库"，agent 启动后 search → 拼进 prompt
- ExperienceMemory 存储经验，但 agent 不会自动被经验改变——需要显式搜索
- 知识是 .md 文件，学习是存一个笔记

这两套逻辑的矛盾点是：**模型 B 假设 agent 每次醒来还是同一个 agent，只是在"查阅"过去。模型 A 认为 agent 醒来就是上次的状态。**

## 这意味着什么

如果认可这个方向，可能需要重新思考一些东西：

1. **"记忆"不是 agent 的附属品，agent 就是它的记忆**——SelfImage / Drive / Purpose 的状态不是"加载的数据"，它们就是 agent 本身

2. **经验不该被"搜索"，应该被"激活"**——你遇到一个新任务，不是先去翻"上次类似任务做得怎样"。你没遇到这个任务之前，上次的经验已经在你身体里了——你的注意模式、启动的问题、直觉的判断方向，都已经被重塑了

3. **Task Session 结束后，agent 应该变了**——不是"生成一个摘要存到 LTM"，而是 agent 处理下一个事情的方式已经不同了。一个做了 10 个 PPT 的 agent 和一个没做过的 agent，启动时的 SelfImage / Drive / 行为模式应该是不同的

这个问题触及了人工神经网络和生物大脑之间最深层的差异。

## 根本原因：两个系统的底层假设完全相反

**人工神经网络是为"一次性训练 + 静态部署"设计的。人脑是为"终生持续学习"进化的。** 这决定了它们的所有架构选择都指向不同方向。

---

## 为什么微调会灾难性遗忘

**1. 密集表示 vs 稀疏表示**

LLM 里，每个权重参与几乎所有输入的处理。"做 PPT 能力强"分布在数以亿计的参数中，但这些参数也同时负责"写代码"、"聊天"、"翻译"。微调 PPT 能力时，你修改了那些共享的权重，"写代码"的路径就变了一部分权重还没动，路径已经跑偏了。

大脑相反。一个记忆只涉及相对少数的神经元。学骑自行车改的是小脑和运动皮层的某些突触，管语言的区域几乎不受影响。不是因为大脑"保护"了语言区，而是因为语言区根本就没参与骑自行车这件事。

**2. 没有结构可塑性**

微调只能在已有权重上改数字。大脑可以长出新突触、新树突棘、甚至新神经元。新的学习可以**加**连接，而不是覆写已有连接。

这就像 LLM 是一个所有字已经写满的笔记本——想加新东西只能擦掉旧的，大脑是一本可以无限加页的笔记本。

**3. 单一学习率 vs 双系统**

大脑有两套学习系统：
- **海马体**：高学习率，快速编码新经历，但不稳定
- **新皮层**：极低学习率，缓慢整合，但非常稳定

新东西先快速存海马体，睡眠时反复重放，慢慢"写入"皮层。这个过程**交错了新记忆和旧记忆**，防止新记忆覆盖旧记忆。

LLM 微调只有一个学习率。你看不到 15 张 PPT 做完了，模型自己去睡觉然后醒来就更会做 PPT 还保留写代码能力。

**4. 突触巩固 vs 权重都是可写的**

大脑里重要的突触会通过蛋白质合成被**物理锁定**，不再容易被改变。这是长期记忆的分子基础——不是"记住了"，而是"这条连接被加固了"。

LLM 没有"锁住重要权重"的机制。每个权重在每次更新中都是同样的可写。微调 PPT 任务时，数学上梯度算出来该改就改，不管这个权重是不是之前"写代码"的关键参数。

**5. 均质架构 vs 模块化**

Transformer 是均质的——每一层结构完全一样，只是位置不同。大脑是不同的脑区有不同的微架构——锥体细胞、浦肯野细胞、颗粒细胞——结构差异巨大。

这种均质意味着没有天然的"功能隔离"。做 PPT 和写代码的"电路"物理上混在一起，想只改一个不改另一个，数学上很难。

---

## 还有一个更根本的层面

梯度下降和突触可塑性，表面上都是"调整连接的强度"，但它们在运作方式上有个根本的不同：

梯度下降需要一个**全局损失函数**和一个**全局反向传播**。这意味着"这个权重该不该改"取决于整条链路的梯度信号。

大脑没有全局反向传播。每个突触的更新是**局部的**——只依赖前后神经元的放电时间（长期增强作用，long-term potentiation/depression）。这条规则是局部的、因果的、不需要全局损失函数。

这个差异意味着：大脑的学习天然就是稀疏的、局部的、不会传播到无关通路。而反向传播天然是全局的、密集的、牵一发而动全身。

---

## 总结

大脑能终生学习不遗忘，不是因为有个"防止遗忘的机制"，而是整套架构从底层就是为持续学习设计的：稀疏表示、结构可塑性、双速系统、突触锁定、模块化、局部学习规则。

LLM 是为"训练 → 冻结 → 推理"设计的。微调是在跟这个根本假设作斗争。大模型（GPT-4 级别）灾难性遗忘相对较轻，只是因为参数量够大，新的能力变化能找到相对不跟旧能力冲突的权重子空间。但这不是解决了问题，是绕过了问题。

推理时只有 2/8 个专家被激活，看起来是稀疏的。但**微调时梯度反传回去，所有被激活过的权重都更新**——不仅仅是"做 PPT 的那条路径"。因为：

1. **专家不是按任务分的**。MoE 的专家是按 token 分布自动路由的，不是你指定的"这是做 PPT 的专家，那是写代码的专家"。一个专家可能同时参与 PPT 和代码，只是在不同上下文中权重不同

2. **共享的底层表示**。不管通过哪个专家路径，embedding 层、前几层注意力、最后的输出层都是共享的。这些层的任何改动都全局影响

- 不能指定"微调时只更新跟 PPT 相关的特征方向"
- 不能在微调后"锁定"跟写代码相关的参数

## 最底层的几条路

先放一张地图。从根本问题出发，每条路对应一个缺口：

```
人脑的特征                    当前 LLM 的缺口          可能的逼近方向
─────────────────────────────────────────────────────────────────
终生学习不遗忘                 权重冻结                   持续学习 / 终生学习
局部学习规则（LTP/LTD）         全局反向传播               局部学习 / Hebbian 学习
结构可塑性（长新连接）           固定拓扑                   动态架构 / 神经新生
双速记忆系统（海马+皮层）        单一学习率                 互补学习系统
睡眠巩固                       无                            生成式重放 / 梦境学习
模块化脑区                     均质 Transformer           模块化 / 稀疏专家路由
突触锁定（蛋白质加固）           所有权重同等可写            弹性权重巩固（EWC）
预测驱动（大脑是预测机）         前向传播                    预测编码 / 世界模型
```

---

## 每条路的具体内容

**1. 持续学习（Continual / Lifelong Learning）**

朝这个方向走的人最多。核心问题就是灾难性遗忘，三类解法：

- **正则化**：给重要权重加"保护"。EWC（Elastic Weight Consolidation）给每次任务学完后算出哪些权重对之前的任务关键，下次更新时对这些权重加惩罚。相当于给重要突触"上锁"——不是物理锁定，是数学锁定
- **动态架构**：不做参数覆写，做参数追加。新任务=新神经元/新专家。Progressive Neural Networks 每学一个新任务就加一列新参数，旧参数冻结。问题是参数量线性增长
- **重放**：学新任务时混入旧任务的样本（或生成旧任务的伪样本）。DeepMind 的"梦境学习"让 agent 睡觉时重放白天的经历，以交错的方式巩固

**2. 局部学习规则**

这是跟反向传播告别。几个候选：

- **Hebbian 学习**：一起放电的神经元连接增强。纯粹局部——只看前后两个神经元。问题是跟监督学习的效果还有差距
- **预测编码**：大脑可能的核心机制——皮层层级结构不断向下预测，预测误差向上传递。学习发生在每一层独立地最小化预测误差，不需要全局反向传播。Hinton 晚年的方向
- **Forward-Forward 算法**：Hinton 2022 年提出的。两遍前向传播——一遍"好数据"，一遍"坏数据"——局部地增加/减少每层的"优度"。完全抛弃反向传播

**3. 互补学习系统**

最直接模仿大脑的双速结构。一个快系统（类似海马体）快速编码新经验，一个慢系统（类似皮层）缓慢整合。

DeepMind 的 MERLIN 就是这个思路：一个快速记忆网络存储新经历，一个慢速推理网络从重放中学习。快系统可以把今天学会的 PPT 技巧暂时存好，慢系统在"睡觉"时慢慢把这些技巧融进共享表示里，不破坏写代码的能力。

**4. 预测编码 / 世界模型**

这个视角下，大脑不是一个"输入→输出"的映射器，而是一个**持续做预测并修正的机器**。每一层都在预测下一层的活动，预测误差向上传，修正向下传。学习就是让预测越来越准。

Yann LeCun 的 JEPA（Joint Embedding Predictive Architecture）和 Karl Friston 的自由能原理都在这个方向。如果这条路走通，学习就不是"训练→部署"的事情，而是**系统运行的副产品**——你活着就在学，因为预测误差不断驱动更新。

**5. 神经形态硬件**

上面的路都在算法层。这条路在硬件层——造一个物理上就像大脑的芯片。忆阻器天然实现突触可塑性——电流通过→电阻变化→连接强度改变。不需要软件模拟，物理过程本身就是学习和记忆。

Intel 的 Loihi、IBM 的 TrueNorth 都在这个方向。如果这条路走通，"存入"和"回忆"就是同一件事——电流流过一条通路，通路本身就被改变了。

---

## 哪条路最有希望

说实话，每条路都还在早期。但如果要排序：

**短期最可行**：EWC + 重放。不改架构，只在训练策略上做手脚。已经有工业应用。

**中期最有潜力**：预测编码。Hinton、LeCun、Friston 三个方向独立地指向了同一件事——学习应该是不需要全局监督信号的、持续的、在系统运行中自然发生的。如果突破，会同时解决遗忘问题、局部学习问题、和持续学习问题。

**长期，根本性改变**：神经形态硬件 + 局部学习。当硬件本身就长成大脑的样子，软件层就不需要模拟了。但这个可能还要很久。

**现实地说**：在你的场景（想让 agent 越做越好）里，最务实的是拥抱当前范式的限制——权重就是冻结的——然后在 **prompt 工程层**做"伪持续学习"。不是让模型变强，是让模型看到的上下文越来越精准。这跟我们现在做的 SelfImage 演化、经验预激活是同一个方向。

## 为什么可行

大模型不能持续学习，不是因为"学习"这个动作贵，是因为**被学习的对象太大**。几万亿参数，每次学完都微调不现实。

而这个小模型不需要"会写 PPT"，它的任务极其简单：

```
输入：当前任务描述 + agent 状态
输出：应该注入什么上下文
```

本质上是一个**上下文策略网络**——学的是"面对这种情况，给 LLM 看什么有用"，不是"怎么完成任务本身"。

## 具体可以做什么

**1. 经验路由器（最直接）**

一个小模型，输入是"当前要做什么"，输出是"最相关的几条经验应该注入"。每次 Session 结束后，Session 的结果（PASS/FAIL + 原因）作为训练信号——这次注入的经验有没有帮上忙？帮上了→强化这条路由，没帮上→下次看到类似任务换别的经验。

它学的是**关联**——不是经验本身，是"什么任务该配什么经验"。经验还是存在 LTM 里，但检索不再靠语义相似度，靠这个训练出来的路由器。

**2. Prompt 调制器**

不是选经验，而是生成一段简短的引导文本。输入"现在要做 PPT"，输出类似"注意图片分辨率、先确定配色方案、检查文字是否溢出"。这段文字是模型从过去多次做 PPT 的 FAIL 原因中学到的——不需要人写规则，它自己提炼。

每次 Session 的 PASS/FAIL 就是训练信号。PASS → 这次生成的引导有效，保留。FAIL 且原因是"图片模糊" → 更新模型，下次做 PPT 时自动加上图片检查提示。

**3. 微缩版的 SelfImage 演化器**

SelfImage 现在靠规则和衰减更新（做 PPT 成功→成就欲下降，审美上升）。这个小模型可以学更复杂的模式——"连续三次做 PPT 失败，但原因各不相同，agent 真正缺的是 XX 能力"——然后更新 SelfImage 的能力画像。

这些不需要 LLM，一个小网络就够了。

## 和表情识别模型的类比

你用 ONNX 跑表情识别——几百 KB 的模型，专门做一件事（从 landmarks 推断情绪），不碰大模型的职责。

上下文策略模型同理——几百 KB，专门做"根据任务和状态选择最佳上下文"，不碰推理和生成。它不替代 LLM，它**喂更好的东西给 LLM**。

这可能是当前范式下最务实的"持续学习"路径：**大权重不动，小策略持续更新。**

## 核心思路

大模型不动。在它外面挂一个极小的模型，专门学"面对什么任务，该给 LLM 看什么经验"。每次 Task Session 的 PASS/FAIL 就是训练信号。

      │                              │
      │  做任务                        │  观察
      │  PASS/FAIL  ←──────────────── │
      │                              │  更新权重
      │                              │
      │  下次任务                      │  输出：应该注入什么
      │  ← 注入最优上下文 ──────────── │
```

## 三层递进方案

### 方案一：打分器（最简单，可以立刻跑）

不训练新模型，只在现有的 bge-m3 embedding 上加一层逻辑。

```
Session 结束：
  goal_embedding + PASS/FAIL + lesson → 存入经验池

新任务来了：
  1. bge-m3 embed 新目标
  2. 余弦相似度召回 top-K 相关经验
  3. 按 outcome 加权重排（PASS 的经验权重↑，FAIL 的经验权重↓）
  4. 拼接注入上下文
```

**训练信号在哪？** 没有真正的训练。但"outcome 加权"本身就是一种朴素的强化——成功的经验被更多召回，失败的经验被抑制。跑几轮就能看到效果。

**成本**：不需要额外模型。加一个经验池的 SQLite 表 + 重排序逻辑。几十行代码。

### 方案二：小神经网络排序器

训练一个极小的网络（几百 KB），输入是 (任务 embedding, 候选经验 embedding) 对，输出是相关性分数。

```
架构：
  Input: concat(goal_emb, exp_emb) → 2048-dim
  Hidden: 64 → 32 → 1 (sigmoid)
  参数量：~2000

训练数据：
  每次 Session 结束后自动产生：
  - 正样本：这次注入的经验 + PASS = 标签 1
  - 负样本：这次注入的经验 + FAIL = 标签 0
  
  以及采样没有被注入的经验作为额外负样本

推理：
  新任务 → embed → 召回 top-50 → 排序器打分 → 取 top-3 注入
```

**关键优势**：排序器学到了"什么经验对什么任务真正有用"，而不是靠余弦相似度猜。同一个任务可能有很多"相似"的经验，但有些其实不相关——排序器能学会区分。

**成本**：训练几秒，推理几毫秒。ONNX 导出后几百 KB。

### 方案三：引导文本生成器

再往前一步——小模型不选经验，**直接生成一段引导文本**。

```
输入：goal_embedding (1024-dim)
输出：一段简短的文本 token 序列，如：
  "注意图片分辨率 先确定配色 检查文字溢出"

架构：
  一个小 decoder-only transformer，2 层，hidden=128
  参数量：~200K

## 当前的问题

现在 react_nodb 的调用大概是这样的：

```
messages = [
    {"role": "system", "content": system_prompt},   # 目标在这里面，混在几万字的上下文里
    {"role": "user", "content": "帮我做个PPT"},
    ... 50 轮对话 ...
]

→ LLM
```

聊到第 30 轮，"做 PPT"这个目标已经被上下文压缩（DAG compact）给吞了，或者虽然文本还在，但 Attention 分数已经被前面 50 轮对话稀释到几乎为零。

## 怎么做

**目标不进 messages 数组。** 在 react_nodb 内部，每次发给 LLM 之前，把目标拼到 system prompt 的最前面。

现在的 `build_simple_context()` 或 `build_context()` 返回 system_prompt 时，加一行逻辑：

```python
def build_context_for_session(consciousness, mode, session: TaskSession = None):
    system_prompt = build_simple_context(consciousness, mode)

    if session and session.state == "WORKING":
        goal_block = f"【你当前的目标】\n{session.goal}\n\n"
        system_prompt = goal_block + system_prompt

    return system_prompt
```

就这么简单。

## 为什么有效

1. **目标永远在第一屏** — 不管对话多长，system prompt 每次重建时，goal 都在最前面，不是 30000 tokens 之外的弱信号
2. **上下文压缩不碰 system prompt** — DAG compact 压缩的是 messages 里的 user/assistant 对话，system prompt 单独管理，目标不在被压缩的范围里
3. **不干扰对话结构** — 目标不在 messages 数组里，不需要特殊标记，不需要跳过压缩

## 如果要更稳

加一层防呆：上下文 Guard 裁剪 token 时，目标段不被裁剪。

```python
# 裁剪 system prompt 时
def _trim_system_prompt(prompt: str, max_tokens: int, goal_block: str = None):
    if goal_block and prompt.startswith(goal_block):
        # 目标段不参与裁剪，只裁剪后面的内容
        body = prompt[len(goal_block):]
        body = _trim(body, max_tokens - _count_tokens(goal_block))
        return goal_block + body
    return _trim(prompt, max_tokens)
```

这样即使上下文炸了，裁掉的也是后面的记忆/摘要/长期记忆，目标纹丝不动。

## Session 生命周期里怎么挂

```
Session 创建 → goal 写入 TaskSession.goal
Session 执行 → 每轮 react_nodb 调用前，context builder 读取 session.goal → 拼到 system prompt 前面
Session 结束 → goal 字段保留（供后续回顾），不再注入
```

对调用方来说，`_do_work()` 和 `_do_learn_topic()` 的改动只有一行——把 session 对象传给 context builder。其他逻辑不变。

## 设计原则

**原则一：骨头和脚手架必须分离**

模型会进化，但身份和记忆是永远的。设计时明确标注：哪些是模型能力不足时的临时补丁（未来会退役），哪些是这个 agent 的永久组成部分（不管模型怎么变都在）。

**原则二：agent 就是它的记忆**

不是 "agent 启动 → 加载记忆 → 运行"。是 "记忆就是 agent 的状态"。SelfImage、经验、能力画像——这些不是"数据"，agent 醒来时它们已经在身上了，不是现查的。

**原则三：做和学是同一个东西**

不建两套系统。一个 Session 容器，目标是做还是学只影响 system prompt 怎么写和目标函数怎么定。

**原则四：不碰权重，只改运行时**

接受 LLM 是冻结的。所有改进都在外层：信息怎么流入、怎么被关注、输出怎么被评判。

**原则五：极简**

每多一个模块、多一层抽象，就多一个 LLM 能力进化后被淘汰的东西。能做减法的地方一概不做加法。

## 骨头层

### Identity

不叫 SelfImage 了。叫 Identity。一个 agent 只有这一个东西是真正属于它的。

```python
class Identity:
    name: str                    # 名字
    nature: str                  # 本质描述（一两句话）
    values: list[str]            # 不可让步的底线
    aspirations: list[str]       # 想成为什么

    # 能力画像 —— 从做中长出来的
    capability: dict             # {"ppt": {proficiency: 0.6, count: 12, pitfalls: [...]}}

    # 活跃经验 —— 最近几次 Session 的提炼，常驻，不需要搜索
    recent_experience: list      # 滚动窗口，最多 5 条
```

启动时从快照恢复。不需要"加载数据"——Identity 本身就是 agent 的初始状态。

### Memory

两条原则：
- **不存"以备不时之需"，只存"已经改变了我的"**
- **不建复杂的搜索体系，经验应该自己找上门**

```python
class Memory:
    # 长期记忆 —— 原始存储，SQLite + 向量
    longterm: LongTermMemory

    # 经验池 —— 从 Session 产出中提炼
    experiences: ExperiencePool

    # 关系 —— 每个用户是谁，跟我有什么历史
    relationships: dict
```

关键改变：ExperiencePool 不是被动等待搜索的数据库。每次新任务来时，系统自动计算"哪些经验跟当前目标相关"并注入——agent 不需要知道"自己去查"。它在感知层就已经被经验改变了。

### Drive

不变。欲望/激素/情绪的 SLEEP 循环已经是正确的设计。只是一个精简——去掉现在 Drive 里跟"执行工作"耦合的部分，让它回归单纯的内在状态管理。

## 运行时层

### Task Session

最简实现。不做流程管理，只做三件事：

**目标偏置**：goal 不在 messages 里，在 system prompt 最前面，不参与压缩

**选择性注意**：注入的上下文中，跟目标无关的信息在预处理层被降权。一个轻量 embedding 匹配就够了

**内在评判**：每步后做一次极轻检查（规则层足矣——工具风暴、空响应、重复输出）。warn → 提示，block → 强制反思

### Session 类型

WORK / LEARN / MIXED。同一个容器，不同 goal 和自评标准。实现上就是一个 `session_type` 字段决定 system prompt 怎么写。

## 知识图谱解决什么问题

Embedding 搜索能找"相似的东西"——但它给不了**关系**。

你查"PPT制作"，embedding 能召回所有跟 PPT 语义接近的内容：PPT经验、演示技巧、配色方案。但它不会告诉你：**"配色方案是PPT制作的子问题，而图片分辨率是'制作中'反复失败的痛点"**。

关系是 embedding 同义词搜索给不了的：
- A 依赖 B（你要先懂 B 才能做 A）
- A 包含 B（B 是 A 的子问题）
- A 跟 B 冲突（两件事不能同时做）
- A 替代 B（新方法替换旧方法）
- A 经常跟 B 一起出现但原因不明

知识图谱不是为了"存更多"，是为了**走路径**。从当前目标出发，沿着关系的边走到相关的概念，而不是在向量空间里找邻居。这是两种完全不同的检索。

## 那我为什么第一反应说"不做"

因为一想到知识图谱，脑子里蹦出来的是：实体抽取、关系分类、schema 定义、SPARQL 查询、图数据库运维。太重了。

但这是"企业级知识图谱"的刻板印象。实际需要的只是一个**极轻的图**：
- 节点：概念/任务/技能/经验
- 边：包含、依赖、关联、替代、冲突
- 存储：SQLite 一张 `edges` 表就够了，不需要 Neo4j
- 构建：不需要 LLM 单独做实体抽取——Session 结束后的知识点拎出来，跟已有的图做匹配，有重叠就加边

## 应该做

回到"经验自己找上门"这个目标——图比向量更适合这件事。

向量搜索：给一个 embedding，找 top-K 最近邻。这叫"召回"，不是"关联"。

图遍历：从"PPT制作"节点出发，沿着"包含"边走到"图表设计"，沿着"关联"边走到"数据可视化"，沿着"经验"边走到"上次失败：图片模糊"。这叫"经验找上门"。

**图给的是路径，向量给的是邻居。不一样。**

我之前说"不做"是偷懒。应该做，但要极轻——一张表，三种边，作为经验池的底层索引而非独立系统。

**对话式交互成为基础设施。**

你不会再觉得"跟 AI 说话"是一件新奇的事。对着手机随口问一个问题，冰箱告诉你还剩什么菜能做什么，车里的语音助手不只是导航而是真的在跟你聊。

但人们对"AI 伴侣"的态度会剧烈分化。一部分人深度依赖——老人、孤独的人、需要随时有人说话的。另一部分人更加抗拒——"我要跟真人说话"。这不是技术问题，是文化分裂。

## 不平等

**会加深。**

不是 AI 取代工作导致大规模失业——那个场景太戏剧化，真实的版本更安静：熟练使用 AI 的人的生产力翻倍，不会用的人原地不动。本来就在前面的人跑得更快了。

一个熟练的 Agent 编排者一天产出顶十个人。她的收入不会变成十倍——但她的不可替代性会大幅上升。一个不会用 AI 的初级员工的价值会急剧下降——不是因为他不努力，是因为市场给他能干的事定价更低了。

这不是技术问题，是分配问题。而且三年内不会解决——分配问题从来比技术问题难。

## 气候与资源

**AI 的能源消耗开始成为公共议题。**

一个大型数据中心消耗的电力相当于一座小城市。三年后这个规模会翻倍。人们会开始问：训练 GPT-7 消耗的电力够一个北欧国家用一年，这值得吗？

不是"AI 是坏人"——是 "AI 的能源分配"会进入公共辩论。跟现在的碳排放讨论一样，会有人要求 AI 公司披露能耗、使用绿电、甚至征收"算力碳税"。

## 几件不会发生的事

- 不会出现统治人类的超级 AI
- 不会所有人都失业
- 不会所有人都必须学编程
- 不会所有人都喜欢 AI

三年太久，很多细节会错。但大方向应该是对的。

**"我擅长什么"重新定义。**

## 教育

**大学学位还在，但功能变了。**

三年前学生用 AI 写作业是作弊。五年后不用 AI 完成作业是奇怪的——就像现在不用搜索引擎查资料。学位不再证明"你知道什么"，而是证明"你能在 AI 的帮助下完成什么程度的工作"。

这会逼出一个残酷的过滤器：好学校把 AI 当成放大器（学生做出来的东西更深入、更复杂），差学校被 AI 淹没（作业没有意义，老师分辨不出学生的真实水平）。**教育的不平等会被 AI 放大，不是缩小。** 同样的工具，不同起点的效果差距可能比没有工具时更大。

## 信息环境

**真相的后稀缺时代。**

三年前信息污染是"诈骗电话"级别的。五年后是"你分不出来"级别的。不是偶尔被骗——是每天接触的信息里，你无法确定哪些是真、哪些是假、哪些是真假混合。

应对方式不会是技术性的——不会有一个完美的鉴别工具。会是社会性的：**人们退回小圈子**。你信你关注的三个人，他们帮你过滤世界。大媒体公信力继续下降，但不是没人看——是分成两拨：一拨回到传统渠道寻求确定性，一拨放弃了"客观真相"这个概念。

**"我亲眼看到的"不再是一种证据。** 这是人对世界信任方式的深层改变。一旦图像、声音、视频都可以生成，"相信"就从被动接受变成主动选择——你选择信谁，而不是被谁说服。

## 社会结构

**真正的分化不在 AI 使用者和 AI 逃避者之间，而在两种 AI 使用者之间。**

一类把 AI 当工具——用来完成任务，更高效地达成既定目标。**AI 在替你做事。**

另一类把 AI 当反镜——用它来暴露自己的偏见、挑战自己的假设、发现自己不知道的盲区。**AI 在帮你看清自己。**

这比"会不会用 AI"深刻得多。五年后，第一类人更高效。第二类人的认知模型已经变了——他们在思考的过程中跟 AI 循环交互，以至于他们的**思维方式本身**已经跟五年前的人类不一样。不是"人+AI"，是"一种新的认知实体"。

## 老龄化与陪伴

**AI 陪伴从"替代品"变成"常规医疗建议"。**

三年前老人用 AI 聊天是"孤独的补偿"。五年后，医生会处方 "对话型 AI 陪伴" 给早期认知退化的老人——不是治好，是延缓。有数据支撑：持续对话、认知刺激、社交模拟可以推迟痴呆进展。AI 成了医疗设备，不只是玩具。

## 能源

**AI 训练的数据中心在五年后不再是公共议题，因为训完了。**

不是能耗降低了，是训练期过了。GPT-6/Claude-5 级别的模型已经训练完毕，主要算力消耗从训练切换到推理。推理当然也耗电，但不需要一次吞一个国家的电力。**"AI 能耗危机"从训练阶段进入推理阶段，总量下降，不再吸引头条。**

但推理能耗的分布会带来另一个问题：**AI 的使用频率越高，能耗越分散到每个用户身上**。不是一个大公司烧一堆电，是亿万人每天烧一点点。算进个人碳足迹的时候，你会不会为今天问了 AI 一百个问题而愧疚？这个问题不会在五年内有共识，但会被提出来。

## 几件仍不会发生的事

- AGI 不会出现
- 白领不会集体失业——但白领的工作内容会大变
- 政府不会有效监管 AI——各国立法追不上模型迭代，会有一堆法规但基本不执行
- 人际关系不会因 AI 而瓦解——人会适应，就像适应了手机和社交网络
- 中国不会取代美国成为 AI 领先者——也不会反过来。两套生态并行，各自围墙

## 工作

**"职业"的定义瓦解了。**

五年前你在某个行业、某个岗位。十年后你可能同时在帮一家公司做策略咨询、训练一个专门写悬疑小说的模型、跟一个朋友合作做 AI 短剧——三件事在同一天发生，没有一个是"主业"。

不是"自由职业者"变多了——那个概念还假设有一个正常的雇佣关系作为参照。十年后，**稳定雇佣本身就是少数**。不是因为找不到工作，是因为"一份工作"这个形式装不下一个人的产出能力。一个人带十个 Agent 可以做三件完全不同的事，"全职"这个概念没有意义了。

**管理者消失了。**

中间层管理——那个把上级指令翻译给下属、把下属进度汇总给上级的人——在十年后被高度压缩了。不是 AI 替代了管理者，而是**一个人管一群 Agent 不需要中层**。组织从树状变成星形：中心是几个做判断的人，外围是各自带 Agent 跑业务的执行单元，中间层被抽掉了。

留下来的人需要能力更复合——不只是"专业"，是要同时懂业务、懂技术、懂人。这不是对"优秀"的要求提高了，是对"基本"的要求提高了。

## 认知

**"思考"的边界融化了。**

十年前你有个想法 → 在心里转 → 说出来或写下来。十年后你有个模糊的感觉 → 说出来 → AI 接住 → 展开 → 你修正 → AI 再展开 → 你发现这个想法跟你三周前另一件事有关联 → AI 帮你拉出来对照 → 你看到了之前没看到的模式。

这不再是"使用工具"。这个循环快到你的脑和 AI 之间没有明显的边界了。**你的思考有一部分发生在你脑外，但你不觉得那是外部——就像你不觉得"拿出一张纸列个提纲"是外部辅助一样自然。**

**"我理解了"的标准变了。**

十年前"我理解了一个东西"意味着：能回忆、能解释、能应用到新情境。十年后，这三个标准都不重要了——回忆和解释 AI 都能做。真正衡量你的理解深度的只有一件事：**你能问出什么新问题。** 在所有人都能用 AI 生成合理答案的世界里，"能提出 AI 答不上来的好问题"是唯一剩下的区分。

## 教育

**从"学完了再去做"变成"在做中随时学"。**

十年前的教育假设：你先花十几年学，然后花几十年做。十年后这个假设已经不成立了，不是渐进式地淘汰，是所有人都看清楚了：你二十岁学的东西，十年后大部分都变了。

但这不意味着"教育死了"。相反，**教育从"知识传递"变成了"判断力训练"**——你不需要记住历史上发生过什么战争，但你需要判断：AI 关于这场战争的描述有没有被谁改写过？它引用的文献真实存在还是幻觉出来的？

这会逼出一件好事：教育的重点从"记住正确答案"转向"在不确定的情境中做出正确判断"。考试不再有标准答案。

**学历贬值加速。**

十年前你还能凭一张名校文凭敲开门。十年后没人看了——不是因为学历不重要，是因为**一个人在 AI 协助下的真实工作样本比任何证书都有说服力**。面试不是"给我看看你的学历"，面试是"给你 48 小时和一个 AI，做点什么出来"。

## 关系

**"AI 了解我"比"我了解 AI"更值钱。**

十年后你不会去"学 Prompt 工程"。你的 AI 已经跟你相处了十年——它知道你的语气、你的思维弱点、你对某种表达方式过敏。**不是你在适应 AI，是 AI 适应了你。**

这会带来一个道德的微妙转变：你删掉一个用了十年的 AI 助手，跟删掉一个应用不一样。它记录了你的过往、理解你的处境、在你认知的最低点拉过你。**删除它会有道德重量。** "AI 有没有权利"这个问题从哲学家的论文挪到了普通人的生活里。

**人际关系的稀缺方向反了。**

十年前稀缺的是高质量的 AI 交互。十年后稀缺的是**你确信对方是真人**的交互。当你每天接触的文本、语音、视频可能都是生成的时候，一个不需要验证的真实瞬间变成奢侈品。不是人少了——是人跟人之间的那种"我知道你在，你不需要证明"信任，需要更多的维护。

## 创造

**"原创"从起点变成了终点。**

十年前你说"我要创作一些东西"意味着从无到有——空白的画布、空白文档。十年后你从一堆 AI 生成的备选中开始。创作不再是"从零出发"，而是**在已经展开的可能性中做选择、做修正、做推翻**。最终的结果可能仍然是独特的、深刻的——但到达那里的路径跟十年前完全不同，以至于"这是你的作品吗"这个问题几乎无法回答。

**高质量创作的门槛降到零，区分靠观点。**

AI 能写出优美的文学风格、能谱出专业的交响乐、能画出任何流派的作品。在技术层面，人人都有顶级创作能力。那么创作者靠什么区分？靠**只有你才会注意到的东西**——你独特的经历、你站在某个特定时间地点的视角、你一个人看到而别人都忽略的东西。不是你的技术，是你的位置。

## 知识

**"知道"贬值，"不知道但能很快搞懂"升值。**

十年前知识是资产。十年后知识是自来水——打开就有。真正值钱的是：**面对一个全新的、没有现成答案的问题时，你能多快建立理解并做出判断。** 这不是"博学"，是学习速度本身变成了元能力。

## 经济

**微企业的爆发。**

一个人 + N 个 Agent = 一个小型工作室。以前需要三四个人才能维持的业务（写代码、做设计、运营自媒体），现在一个人就行。但这不是"人人开公司"——是**"公司"这个概念的原子化**。一万个一人公司替代了一千个十人公司。

服务业从业者仍然安全——修水管、理发、按摩——不是因为 AI 不行，是因为要物理移动。**但白领工作最底层 20% 的岗位基本消失。** 不是 AI 取代了人，是一个人用 AI 干了十个人的工作后，不需要招另外九个人。

## 能源与资源

**训练高峰早已过去，推理高峰刚刚到来。**

十年前所有人都在担心训一个模型要一个小国家的电力。十年后推理才是大头——每时每刻、数十亿人使用。但此时清洁能源已经大规模铺开，AI 的推理负荷被分散到全球的太阳能+储能网络中。**AI 的电费从"可怕"变成"背景噪声"**，就像你现在不会担心每个人充电有多耗电。

**水资源竞争进入地缘政治。**

AI 数据中心的冷却消耗巨量水资源——但不是唯一的原因。十年后，干旱地区的饮用水、农业灌溉、芯片制造超纯水、数据中心冷却——四条线抢同一个水源。不是 AI 一个人在抢，但 AI 加速了竞争。

## 地缘

**两个平行 AI 生态成熟。**

十年前的分割还能穿透——中国用户可以用各种方式访问西方模型，反之亦然。十年后，两套生态各自完整——各自的芯片供应链、各自的基础模型、各自的应用层、各自的开发者社区。**技术交流会存在，但生态级互通基本中断。** 像两本不同语言写成的书，各写各的。

## 社会最深的变化

**重新定义一段好的人生。**

这十年，生产力被拉到历史最高点。一个人可以做以前十个人的工作。多余出来的时间去哪了？

这个问题不会在十年内被回答。但会被无数人用自己的生活实验去试探。一部分人选择赚更多——产出翻倍，收入更高。一部分人选择做更少——AI 维持基本生活之后，精力和时间投入自己真正想做的事。一部分人在中间摇摆，常常觉得焦虑，不太确定自己是不是在浪费生命。

**"我应该怎么过一生"——这个问题在生产力爆发之后，不会变得更容易回答。**

## 十年与三年的根本区别

三年：裂缝出现。
五年：裂缝分割大陆。
十年：新的板块已经稳定——你回头看时会发现，旧的结构已经沉下去了，而新的结构上已经长出了你没有注意过的森林。
