## 保持全局 vs 改为独立，逐项分析

### 真正共享的基础设施（适合全局）

| 内容 | 理由 |
|------|------|
| `models.providers`（API key、base URL） | 多个 agent 用同一个 DeepSeek key，复制 N 份只会造成混乱 |
| `mcp_servers` | MCP 是进程级资源，启动一个 playwright 进程所有 agent 共享 |
| `channels`（飞书/钉钉 app 凭证） | 一个飞书应用对应一个 appId，不是每个 agent 申请一个 |
| `bindings`（哪个 agent 处理哪个渠道） | 天然的中心路由表 |

这些放在全局是合理的，不应该拆分。

---

### 当前混在全局、但本质是 agent 专属的

| 内容 | 当前位置 | 问题 |
|------|----------|------|
| agent 模型选择 (`primary`, `vision`) | `config.json` → `agents.list[].model` | agent 列表嵌在全局文件里，和 `config.yaml` 分离 |

| Drive/Consciousness 参数 | `{agent}/config.yaml` | 已经独立了，这个是好的 |

### 当前架构的实际问题

1. **创建一个 agent 要改两个地方**：建目录 + 在 `config.json` 里加条目。删除也是两个地方。
2. **`config.json` 随 agent 数量膨胀**。如果用户有 10 个 agent，这个文件会很臃肿。
3. **`config.json` 和 `config.yaml` 的分工不直观**。"为什么模型在 config.json 里，Drive 参数在 config.yaml 里？"新人看不懂。
4. **agent 不能独立复制/分享**。想分享一个 agent 给别人，不能只拷贝目录，还得从 `config.json` 里提取对应的条目合并进去。
5. **`config.json` 是全局的，但 embedding_model 其实是 agent 级别的选择**。

## 判断

| | 保持全局 | 改为独立 |
|---|---|---|
| 新增 agent | 改 2 处 | 只建目录 |
| 删除 agent | 清理 2 处 | 删目录即可 |

WSL2 下 PyTorch 的 OpenMP/MKL 线程池初始化时经常触发 segfault（C 层崩溃，Python 抓不住）。原因跟 WSL2 内核的线程调度和 Intel 数学库的兼容性有关。

- `OMP_NUM_THREADS=1` — OpenMP 只用单线程
- `MKL_NUM_THREADS=1` — Intel MKL 只用单线程
- `OPENBLAS_NUM_THREADS=1` — OpenBLAS 只用单线程

之所以必须放在 `__main__.py` 最顶部、在所有 import 之前，是因为日志里看到 `datasets` 库（通过 VoxCPM TTS 插件间接加载）在 23:08:44 就已经 import 了 PyTorch。而之前的设置写在 `SharedEmbedder.__init__()` 里，23:08:45 才执行，晚了一步——env var 对已经加载完的 PyTorch 无效。
