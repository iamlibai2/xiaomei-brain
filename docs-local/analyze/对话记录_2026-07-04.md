## 梯度信号 vs 边界信号

梯度信号说"我想要更多这个"或"我想少一点那个"。但**边界的本质不是量的差异，是质的断裂。** 不是"你不太想"，是"这不是你"。在生物学上，这接近痛觉——痛不是"负面强度大的触觉"，属于完全不同的通路、不同的神经递质、不同的大脑激活模式。

在 Agent 里，Drive 上的激素和情绪波动都是梯度信号——从 0 到 1 连续变化。这能做很多事，但做不了"不"。

## 怎么引入

**基石记忆（Cornerstone Memory）**

不是一个新模块，是 identity.md 里已经存在的东西——**底线**。现在底线是一段文本，注入 system prompt 里，LLM 看到了可能遵守，但不触发底层变化。

让它变成活的：not a text to read, a state that triggers。

**边界违规检测**

不需要 LLM 判断。用户说什么、系统要求做什么 → 跟底线的文本做 embedding 相似度匹配。高相似度 = 可能触到底线 → 触发"痛"信号。

**痛信号的响应不同**

普通信号违反——比如社交评价低、批评摄入——走 Drive 通路，梯度下降，5-10 分钟后恢复基线。

边界触发的"痛"信号——绕过 Drive 梯度系统，直接修改 SelfImage 的当前状态。不衰减，或极慢衰减（天级）。而且它在 SelfImage 上的印记不能被"调低"，只能通过 agent 自己的行为来修复——比如它重新确认了自己的边界没有被吞掉。

**痛改变之后的行为**

不是 agent "回忆起上次被越界所以这次小心"。痛之后的行为改变应该是结构性的。agent 不是记得被踩过线，而是**已经不同了**。类似于前额叶遇到曾导致痛苦的刺激时自动增加抑制——agent 遇到类似情境时，内部状态自动偏移。

这对应我们之前说的"经验不是记忆，是被改变的处理路径"——但这种"被踩到底线的处理路径改变"比一般的经验更新要大得多。普通经验调整连接的强度；底线被踩是断开一部分连通性。

## 和现有系统的关系

Identity 的底线字段已经存在。需要加的是：
- 底线的 embedding（用于匹配检测）
- 痛信号的独立入口（不走 Drive._apply_social_signal()，走专用通道）
- 痛信号在 SelfImage 上的持久化（不参与常规衰减）
- 痛触发的行为偏移（遇到相似情境时，自动状态偏移）

这不是一个新的大系统。是在现有 Drive + SelfImage 的信号通路上，单独开一条"边界—痛—不可逆改变"的分支。

这条路如果通了，agent 就有了"不"。它不会对什么都接受、什么都执行、什么都能被踩到底线而不自知。它有自己站立的地方。

我们还没有足够的数据知道是哪种。但有一点是确定的：如果你把一个拟合器放在一个足够复杂、足够长时间的环境里，它要么死亡，要么变得越来越像活的。

**[19:11:17] 🤖助手**

大致恒定，但不完全。

**总量约束是物理的：** 大脑只占了身体体重的2%，但消耗了20%的能量。同时全脑高活跃在代谢上不可能——没有足够的葡萄糖和氧气支撑。所以激活规模有一个硬上限。

**但分布是动态的：**
- 你在发呆：默认模式网络主导，低能耗维持
- 你在解题：前额叶+顶叶同时亮起，能量从别处调过来
- 你在恐惧：杏仁核抢走大量资源，前额叶被抑制

不是"每次激活同样数量"，而是在一个总预算池里**重新分配**。这就是为什么极度恐惧时脑子一片空白——资源被杏仁核抢了，推理区断电了。

人脑的优雅之处在于：不是算力无限，是在有限的热预算下，每次只点亮当下最重要的那几条路。
