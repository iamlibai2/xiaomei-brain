分析一下当前的问题：

**现状：** `infer_execution_result` 的调用链是通的，但 `injected_procedure_ids` 永远是空的，因为根本没有记录"本次注入了哪些 procedure"。

**调用链：**
```
用户输入 → match() 返回 top-k → inject_context() 注入 context → LLM 推理
                                                              ↓
L2 tick → infer_execution_result(conversation_history, injected_ids) ← 永远是 []
```

**核心问题是"注入"没有记录。** `ContextAssembler` 注入了 procedure hint，但没有任何地方记下"本次注入了哪几条"。

**方案讨论：**

**方案 A：在 `ProcedureMemory` 里存"最后一次注入"**
- `match()` 时同时记 `self._last_injected_ids = [p['id']`
- `infer_execution_result()` 内部直接用这个字段，不用外部传参
- 缺点：不是线程安全的（并发 chat 时会被覆盖）

**方案 B：回传 IDs，由调用方保管**
- `ProcedureMatcher.match()` 返回 `(ids, procedures)`
- `ContextAssembler` 把 IDs 返回给 `ConsciousLiving`
- `ConsciousLiving` 把 IDs 存到 `AgentInstance`（比如 `agent._last_injected_procedure_ids`）
- L2 tick 时从 `AgentInstance` 读取
- 缺点：需要改接口，改的地方多

**方案 C：infer 改为"从对话内容推断"而非"从注入记录推断"**
- `infer_execution_result` 完全不看 injected_ids，而是从 conversation_history 推断：
  - 如果 LLM 回复中出现了 `<PROC>xxx</PROC>`，说明用了某条 procedure
  - 如果没出现但匹配了 context 中的 steps，可以推断没用
- 这样完全不需要 tracking，逻辑更简单
- 缺点：不如有 injected_ids 时准确

**方案 D：轻量 tracking，只记录最近一次 chat 的注入**
- 在 `agent/core.py` 的 `stream()` 里，`procedure_memory` 注入 context 的同时记下 ids（存到一个实例变量）

### 语义召回：按相关性注入

核心思想：找到和**当前对话语义相近**的 NARR。

#### 两种实现路径

**路径A：关键词召回（当前阶段适用，< 100条 NARR）**

```
用户消息 → 提取关键词
    ↓
从 NARR 的 scene_tags / category / feels_like 匹配
    ↓
打分排序：命中 tag 越多分数越高
    ↓
取 top-5
```

- 不需要 LLM，不需要向量模型
- 基于已有字段（scene_tags、category）做过滤和排序
- `scene_tags` 命中 ≥ 1 个 → 入选；命中越多分数越高

**路径B：向量召回（后期，NARR > 100条）**

```
用户消息 → bge-m3 embedding (1024维)
    ↓
NARR 全部预编码存好（content embedding）
    ↓
向量相似度计算（余弦相似度）
    ↓
取 top-5
```

- 语义泛化能力强（聊"被需要"能找到"你暖和"这条，即使没有共同关键词）
- 复用 longterm memory 同一套 embedding 模型
- 缺点：需要额外的存储和预计算

### 具体逻辑（关键词召回）

```python
def get_relevant_narratives(user_input: str, top_k: int = 5) -> list[dict]:
    # 1. 从用户消息提取关键词（简单分词）
    keywords = extract_keywords(user_input)  # ["工作", "压力", "累"]
    
    # 2. 查所有 active NARR
    all_narrs = ltm.get_narrative_memories(status="active", limit=100)
    
    # 3. 打分
    scored = []
    for n in all_narrs:
        score = 0
        tags = n.get("scene_tags", [])
        content = n.get("content", "")
        feels = n.get("feels_like", "")
        
        # scene_tags 命中
        for kw in keywords:
            if any(kw in tag for tag in tags):
                score += 2
            if kw in content:
                score += 1
            if kw in feels:
                score += 1
        
        if score > 0:
            scored.append((score, n))
    
    # 4. 排序取 top-k
    scored.sort(key=lambda x: x[0], reverse=True)
    return [n for _, n in scored[:top_k]]
```
