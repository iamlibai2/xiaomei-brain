# 三层技术栈商业应用方向设计

## 背景

小美拥有三层技术栈的组合优势：

| 层级 | 能力 | 现有实现 |
|------|------|----------|
| 传统 CV | FACS 动作单元、人脸关键点、轻量表情分类 | dlib 人脸识别、cv2 摄像头 |
| 多模态 LLM | 深度语义理解、场景描述、情绪 nuanced 分析 | Eyes.see()、VisionUnderstanding |
| 记忆/意识 | 长期记忆、情绪追踪、基线偏离、自我认知 | Drive、LongTermMemory、SelfImage、InnerVoice |

现有商业产品都在单层内竞争：传统 CV 做到 95%+ 准确率的情绪标签分类，多模态 LLM 能做单次深度分析，但它们彼此割裂。

**核心命题**：当这三层形成闭环时，能否从"瞬时的状态识别"升级到"长期的变化追踪和基线偏离检测"？

结论：三层闭环不是 60→80 分的改善，而是从 0→1 的质变。

---

## 核心创新原理

### 从拍照到拍电影

```
单层产品（现有）：
  CV 摄像头 → "现在：开心 85%"
  LLM 多模态 → "画面中的人看起来很开心，笑容自然"
  → 输出即结束，下一次对话从零开始

三层闭环（小美）：
  CV 持续采样 → 建立个人情绪基线
  记忆系统 → 跨会话对比，检测偏离
  多模态 LLM → 深度理解"为什么变了"
  → 输出的是变化轨迹和趋势预警
```

### 三层分工

```
传统 CV（System 1）：
  - 高频采样（每秒 N 次），低成本
  - 追踪 FACS AU 单元、眨眼率、目光接触、头部姿态
  - 输出：结构化时间序列数据，不是标签

多模态 LLM（System 2）：
  - 低频唤醒（显著变化时），高成本
  - 深度语义理解："这个笑容和上次的不一样"
  - 输出：自然语言分析 + 上下文解读

记忆/意识（整合层）：
  - 存储个人基线、历史轨迹、模式记忆
  - 对比"今天 vs 他的正常状态"而非"今天 vs 人群平均"
  - 输出：偏离度、趋势、预警
```

### 关键洞察

**个人基线 > 人群模型**。现有产品全部基于人群训练的通用模型（"开心 = AU6+AU12"），但：

- 有人 resting face 就是嘴角下垂（被误判为悲伤）
- 有人社交微笑幅度天生小（被误判为不真诚）
- 有人紧张时反而笑得更多（被误判为积极）

建立个人基线需要**记忆系统**——持续追踪、累计数据、建立个人正常范围。这正是小美独有的。

## 商业应用场景

### 1. 远程医疗 — 心理健康早期预警

#### 痛点

抑郁症、焦虑症患者的复诊依赖自述，但：
- 抑郁症典型症状就是难以表达和自评
- 患者常报告"还行""还好"，但行为数据已经出问题
- 医生只能靠 15 分钟面诊判断，缺乏跨周的数据

#### 方案

```
每次视频复诊：
  传统 CV（轻量，低延迟）：
    - 记录 FACS AU 分布：AU1+AU4（悲伤）、AU12（微笑）、AU7（眼睑收紧）
    - 追踪目光接触时长、眨眼频率、面部活动度
    - 提取语音特征：能量、语速、停顿时长
    输出：结构化时间序列 → 存入个人基线

  记忆系统：
    - 横向对比：今天 vs 他的正常周五 3pm 状态
    - 纵向对比：过去 4 周的微笑频率变化曲线
    - 模式匹配：和确诊时的行为模式相似度

  多模态 LLM（偏离 > 阈值时唤醒）：
    - 综合微表情 + 语音 + 对话内容做深度分析
    - 生成临床可用的情绪变化摘要

  输出：医生的辅助报告
    - 客观数据："第 4 周微笑频率较基线下降 40%"
    - 趋势："AU1+AU4 频次连续 3 周上升"
    - 置信区间："偏离幅度处于本人历史 90 分位数以上"
```

#### 为什么现有方案做不到

| 环节 | 现有方案 | 三层闭环 |
|------|----------|----------|
| 数据采集 | 问卷/自评（PHQ-9） | 非侵入式行为追踪 |
| 对比基准 | 人群统计模型 | 个人历史基线 |
| 输出 | "可能抑郁" 标签 | "偏离本人正常状态 2.3σ，上升趋势" |
| 时间维度 | 单次 = 独立快照 | 跨周/月趋势跟踪 |

#### 市场规模

全球远程心理健康市场约 900 亿美元（2024），年增长 25%+。每家平台都需要客观的患者 engagement 和进展量化工具。

### 2. 工业安全 — 预测性疲劳管理

#### 痛点

化工厂控制室、电网调度中心、空管等岗位：
- 12 小时轮班制，慢性疲劳积累
- 现有 DMS（驾驶员监控系统）只检测"眼睛闭上了"，是事后
- 通用阈值（PERCLOS > 0.8）不管个体差异

**认知疲劳的微表情前兆比眼皮闭合早 30-90 分钟出现。**

#### 方案

```
连续监控（轮班 12 小时）：
  传统 CV：
    - 眨眼频率 + PERCLOS + 微眨眼时长
    - 头部微动模式 + 姿势僵化度
    - 面部 AU 活动范围（僵化 = 前额叶疲劳）
    - 目光扫视频率（注意力广度）
    采样：每 30 秒一次，轻量 CPU 推理

  记忆系统：
    - 建立个人疲劳曲线：通常几点出现第一次注意力下降
    - 个人临界值：PERCLOS 到了多少时开始出现操作失误
    - 跨天对比：今天比昨天的第 3 小时状态更差

  多模态 LLM（异常偏离时）：
    - 综合分析当前状态 + 近期操作记录 + 排班历史
    - 输出可执行建议："建议 15 分钟休息，高风险窗口将在 45 分钟后到来"

  输出：
    - 实时：个人疲劳指数 + 偏离基线幅度
    - 预警：预测未来 2 小时内的疲劳趋势
    - 管理：班次疲劳曲线对比、排班优化建议
```

#### 为什么现有方案做不到

| 现状 | 三层闭环 |
|------|----------|
| PERCLOS > 0.8 → 报警 | 个人基线 PERCLOS 0.3 就预警（预防） |
| 报警时已瞌睡 | 30-90 分钟前检测到微表情僵化前兆 |
| 单点报警，孤立 | 连续曲线 + 趋势预测 |
| 通用模型 | 个人化模型 + 跨天跨周对比 |

#### 市场规模

工业安全市场约 80 亿美元，疲劳相关事故年损失 300 亿美元+。不仅是 DMS 市场，而是全工业安全生产的事前预防体系。

### 3. B2B 高端服务 — 客户关系健康度量化

#### 痛点

理财顾问、律所合伙人、大客户经理：
- 年收入依赖 20-50 个高净值客户的持续关系
- 客户不会主动说"我对你的信任在下降"
- 当客户说"我考虑一下"时，通常内部已经决策离开

**微表情可以比客户本人更早知道信任在流失。**

#### 方案

```
每次视频/面对面会议：
  传统 CV：
    - 杜乡微笑（AU6+AU12，真心）vs 社交微笑（AU12 only，客气）
    - 对特定话题的情绪反应（提到"收益"时嘴角收紧 = 隐忧）
    - 微表情：厌恶（AU9，15-100ms）、蔑视（AU10）、恐惧（AU20）
    - 参与度：目光接触时长、点头频率、前倾度

  记忆系统：
    - 跨会话情绪轨迹："过去 3 次会面，积极情绪从 78% 降到 45%"
    - 杜乡微笑比例变化曲线（礼貌还在，真心没了）
    - 话题-情绪关联："每次提到费用，AU9 出现"

  输出：客户健康度仪表盘
    - 关系健康度指数（情绪效价 + 参与度 + 杜乡比例复合）
    - 趋势变化："3 个月下降了 40%，仍在下降"
    - 预警："当前轨迹下，预计 2 个月内进入高流失风险"
    - 对比："同类客户的正常波动范围"
```

#### 为什么现有方案做不到

- CRM 系统：只有交易数据，没有行为数据
- 单次视频分析工具：只有瞬时快照，没有轨迹
- 传统 CV：只有标签分类，没有长期对比

#### 市场规模

- 财富管理 AUM 全球 1.5 万亿美元，客户留存率每提升 1% 价值数十亿
- B2B 高端服务的客户留存预判是蓝海

### 4. 招聘/面试 — 候选人长期适配量化

#### 痛点

大型企业年面试数万人：
- 面试官主观判断不可靠（第一印象偏差、确认偏误）
- 结构化评分丢了"感觉"
- 无法建立面试行为 → 入职绩效的闭环反馈

#### 方案

```
面试阶段：
  传统 CV：
    - 问题类型 × 情绪反应的精细化映射
    - 真诚度指标（微表情一致性）
    - 舒适区间识别（放松 vs 紧张的自然切换点）

  入职后闭环：
    面试时的行为模式 → 入职 6 个月绩效
    → 系统学习"什么面试行为预测高绩效"
    → 持续迭代预测模型

  输出：
    - 客观行为数据补充（不是替代面试官，是补充）
    - "该候选人面对压力问题的反应模式，
       与你团队前 10% 员工的面试模式相似度 85%"
    - "温馨提示：面试官对此类型候选人的评分
       历史上有系统性偏差（偏高 15%），请注意"
```

#### 市场

HR Tech 全球市场 240 亿美元，招聘 AI 是增长最快的赛道之一。

## 技术架构差异

### 现有产品架构

```
摄像头 → CNN 分类器 → "开心 85%" → 展示 → 丢弃
```

### 三层闭环架构

```
摄像头 → 传统 CV（高频采样）
   │
   ├──→ 结构化时间序列 → 记忆系统（基线/趋势/模式）
   │                              │
   │                              ├──→ 偏离检测引擎
   │                              │      │
   │                              │      ├── 偏离 < 阈值：仅记录，静默
   │                              │      └── 偏离 > 阈值：唤醒 System 2
   │                              │             │
   │                              │             └──→ 多模态 LLM 深度分析
   │                              │                      │
   │                              │                      └──→ 报告/预警
   │                              │
   │                              └──→ 个人基线数据库
   │                                     ├── 情绪基线（AU 分布、频率）
   │                                     ├── 疲劳曲线（个人临界值）
   │                                     └── 关系轨迹（跨会话趋势）
   │
   └──→ 实时展示（低延迟标签）
```

### 三层各司其职

| 层 | 频率 | 延迟 | 成本 | 输出 |
|----|------|------|------|------|
| 传统 CV | 每秒 N 次 | <10ms | 零 | 结构化序列 |
| 记忆/基线 | 每次 CV 采样后 | <1ms | 零 | 偏离度、趋势 |
| 多模态 LLM | 偏离 > 阈值时 | 500ms-2s | API 费用 | 自然语言分析 |

## 竞争壁垒

### 数据飞轮

```
每次使用 → 个人基线更准确 → 偏离检测更精准 → 价值更高 → 更多使用
         → 人群模式更丰富 → 冷启动更快 → 新用户更快到有效基线
```

### 先发优势

- 个人基线需要数据积累（通常 2-4 周建立有效基线）
- 先部署者先积累，后来者的模型再强也缺个人数据
- 切换成本：个人基线数据是不可移植的

### 闭环强化

- 不同于传统产品"分析 → 输出 → 结束"
- 三层架构天然支持"分析 → 输出 → 观察用户反应 → 调整分析 → ..."
- 每次交互都在提高系统精度

## 实施路径

### Phase 1：打通闭环（小美内部）

- 在 `Eyes.see()` 中集成传统 CV 情绪分析（轻量 CNN 或 dlib AU 检测）
- 将情绪时间序列写入 BodyState
- Drive 系统监听情绪信号（类似现有的 apply_social_signal）
- 验证"看见 → 影响内部状态 → 记忆"的链路

### Phase 2：基线系统

- 用户个人情绪基线数据结构和存储
- 偏离检测算法（均值 + 标准差 + 趋势斜率）
- L2 引擎感知情绪偏离，触发深度分析

### Phase 3：商业场景封装

- 将核心能力（采样 + 基线 + 偏离 + 分析）封装为 SDK
- 针对 1-2 个场景做深度适配（建议先做心理健康或工业安全）
- API 化，支持集成到第三方视频会议/监控系统

## 与现有产品的关系

小美不是要和现有情绪 AI 产品（Affectiva、Kairos、Hume AI）做同一件事然后竞争。

- 它们做的是**第一层**：单次快照的情绪分类（且做得很好）
- 小美做的是**第二层**：时序分析 + 基线偏离 + 模式识别
- 第一层是小美的**数据来源**，不是竞品。可以集成它们，然后做它们做不到的事

类比：现有产品是"心率计"（测量当前数值），小美是"心电图趋势分析仪"（告诉你你的心率什么时候异常了，趋势在变好吗，和上次发作时的模式对不对得上）。
