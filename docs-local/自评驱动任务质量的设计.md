# 自评驱动的任务质量闭环

## 问题

Agent 做任务（PPT、代码、文档等）只有"做完"没有"做好"。做完就交，没有自我检查，没有不满意感，没有迭代完善的内在动力。

人类不同——看到自己做的简陋PPT，会不舒服，会改，改到满意为止。这个"不满意"的感觉，是人在做事过程中的核心驱动力。

## 核心模型

```
自我形象（talent.md："我是一个出品质量高的人"）
  ↓ 设定标准
  ↓
任务完成 → 自评（对照 talent.md 检查产出）
  ↓
│ 达标 → 信号恢复 → 结束
│ 未达标 → 审美↓ 一致性↓ → 继续迭代 → 最多3轮
│           3轮仍不达标 → 能力问题 → 记入学习队列
```

## 信号设计

四个工作质量相关的 SLEEP 信号：

| 信号 | 增加 | 减少 | 衰减 |
|------|------|------|------|
| 审美 | 自评达标 +0.1，创造行为 +0.05 | 自评未达标 -0.15/-0.08/-0.04 | -0.02/h |
| 一致性 | 自评达标 +0.05，goal完成 +0.1 | 自评未达标 -0.1/-0.05/-0.03 | -0.01/h |
| 控制感 | 主动选意图 +0.1，拒绝不合理 +0.05 | 被动执行 -0.05 | -0.05/h |
| 新奇感 | learn完成 +0.15，新互动 +0.08 | — | -0.03/h |

### 信号 → L2 意图映射

```
审美 < 0.4    → 建议 refine/express（不满意，想做出好东西）
一致性 < 0.6  → 建议 reflect（言行不一，重新审视）
控制感 < 0.5  → 建议 express/talk（重新确认主动权）
新奇感 < 0.3  → 建议 learn（需要新鲜刺激）
```

## 自评机制

### 触发时机

任何一个"任务"完成后触发。任务定义：使用了生产类工具的对话轮次。

```
生产类工具：shell, write_file, edit_file, generate_music, generate_image, mcp__chrome_devtools, ...
通信类工具：send_message, speak, check_inbox, memory_search（不触发）
```

两个入口：

| 入口 | 触发方式 | 自评时机 |
|------|---------|---------|
| L2意图（work/learn） | 心跳主动触发 | ActionDispatcher 行为完成后 |
| 聊天指令（"做个PPT"） | 用户直接要求 | 对话轮次结束后检测工具使用 |

### 自评 prompt（极轻量）

```
你刚用以下工具完成了一个任务：
工具：{tools_used}
任务：{goal_description}
产出摘要：{result_preview}

你的自我形象是：{extract from talent.md - 身份/底线/追求}

对照你的自我形象，评估这个产出：
- 达标了 → 回答：PASS
- 还差一些 → 回答：FAIL：<具体哪里不够>
```

不做开放式回答，限制为PASS/FAIL + 一句原因。

### 迭代衰减（防无限循环）

```
第1轮自评 FAIL → 审美 -0.15, 一致性 -0.1 → 重新执行
第2轮自评 FAIL → 审美 -0.08, 一致性 -0.05 → 重新执行
第3轮自评 FAIL → 审美 -0.04, 一致性 -0.03 → 结束，不继续
```

### 能力 vs 态度

3轮还不达标，不触发 refine，改为：
- 记入 SelfImage："{领域}能力未达标，第{count}次遇到此问题"
- 认知欲 +0.1（"我需要学习这个"）
- L2 后续可能选择 learn 意图

这是"凡人锚点"——有些事你就是现在做不到最好，尽全力就够了。

## 和已有系统的关系

- **Drive SLEEP**：四信号复用现有的信号→衰减→阈值→意图→行为闭环
- **talent.md**：作为自评的固定参照锚点，而非让LLM自己瞎判断
- **L2 Engine**：hint注入已有雏形，扩展即可
- **ActionDispatcher**：`_satisfy_intent_desire()` 已是行为完成回调，在此注入自评

## 不做的事

- 不做实时检查（只在任务完成后检查）
- 不做复杂评估（不做多轮LLM辩论，就一次PASS/FAIL）
- 不做跨任务比较（不对比"上次PPT做的好还是这次好"）
- 不做自动修复（FAIL后不是AI自动改，而是agent自己重新执行）
