## 3. 发现的问题

### 3.1 确认的 Bug

| 测试 | 严重度 | 描述 |
|------|--------|------|
| `test_complete_goal_twice_double_boost` | 中 | `complete_goal()` 无状态守卫，已完成的目标可重复完成，重复触发 Drive 钩子（多巴胺/血清素双重提升、皮质醇双重下降） |
| `test_intent_priority_from_rule_not_buffer` | 低 | intent buffer 中携带 `priority` 字段，但 `tick()` 完全忽略，使用规则硬编码优先级。存在冗余数据字段 |
| `test_intent_buffer_not_consumed_by_tick` | 低 | `tick()` 只读取 intent_buffer 不消费，同一 intent 每次 tick 都重新触发，依赖外部调用者显式管理生命周期 |

### 3.2 架构设计待确认

| 测试 | 描述 |
|------|------|
| `test_determine_mode_reflect_before_active_goal` | REFLECT 意图的优先级高于活跃目标（reflect > task）。如果同时存在梦境意图和活跃目标，走 reflect 而非 task——需确认是否符合设计意图 |

### 3.3 测试盲区

以下区域当前测试未覆盖，建议后续补充：

1. **`build_context()` SelfImage 完整路径**：当前只有 mock 版本，未验证 inject_consciousness 实际渲染内容
2. **记忆提取 dedup 逻辑**：`recall()` + score>0.85 → UPDATE 的路径未测（需要 mock `_vector_recall` 返回高分结果）
3. **行格式解析路径**（`_execute_line_actions`）：JSON 解析失败后的 fallback 未覆盖
4. **ActionDispatcher 紧急意图冷却绕过**：`urgent_intents` 集合逻辑未测试
5. **Multi-agent 场景**：跨 agent 记忆隔离、同时运行的竞态条件
6. **文件系统失败路径**：LanceDB 不可用、SQLite 写入失败等
7. **PACE 任务执行 + Drive 反馈**：完整的 task→执行→反馈闭环

## 5. 结论

当前测试体系覆盖了核心模块的 happy path 和关键边界条件。4 个集成测试文件验证了：

- **Drive ↔ Purpose**：目标管理的完整激素/情绪/欲望响应链
- **Intent → ActionDispatcher**：9 种 intent 类型的规则匹配、冷却、能量门控
- **MemoryExtractor 全链路**：ADD/UPDATE/MERGE/DELETE 全部 action 类型 + 错误处理
- **Context 组装管线**：模式判定优先级顺序、上下文组装、token 裁剪

发现的 3 个问题中，`complete_goal_twice_double_boost` 是值得修复的真实 bug。其余属于数据一致性和架构清晰度改进项。
