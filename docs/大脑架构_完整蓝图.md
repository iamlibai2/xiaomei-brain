# Xiaomei 大脑架构：完整蓝图

> 2026-05-15
> 目标：让 AI 能执行复杂系统工程。不是辅助，是主导。

---

## 核心理念

**让她像顶级工程师一样成长。** 顶级工程师不是"读了很多代码"长出来的，是做了很多项目、踩了很多坑、反思了很多次长出来的。

她的优势：
- 不会因为自尊心不承认错误
- 能同时记住所有经验，不会忘
- 多个模型互相指正，盲区互补

她的劣势：
- 推演力的上限在模型（但 pro 够用）
- 上下文窗口装不下整个项目（需要压缩机制）

## Layer 1: 执行层

### 1.1 Goal DAG（目标依赖图）

**现状**：Goal 是树——子目标只属于一个父目标。
**改为**：DAG——目标可以有多个依赖。

```python
@dataclass
class Goal:
    id: str
    description: str
    depends_on: list[str]       # [新增] 前置目标 ID 列表
    blocked_by: list[str]       # [新增] 被哪些目标阻塞（反向索引，运行时计算）

    # 已有字段保持不变
    parent_id: str | None
    goal_type: GoalType
    status: GoalStatus
    priority: float
    progress: float
    cognitive_log: list[CognitiveLogEntry]
    artifacts: list[dict]
    pace_checkpoint: dict | None
```

**PurposeEngine 改动**：
- `get_next()`: 不再取"第一个未完成的子目标"，而是取"所有依赖都满足的未完成目标"
- `add_goal()`: 支持 `depends_on` 参数，自动计算 `blocked_by`
- `complete_goal()`: 完成后解除下游目标的阻塞
- `auto_decompose()`: LLM 分解时识别依赖关系（"A 做完才能做 B"），生成带依赖的子目标

### 1.2 ReAct Loop（执行引擎）

保持不变。PACE 作为 ReAct 的增强模式存在，核心循环逻辑不变。

### 1.3 Tools（工具系统）

保留现有 `ToolRegistry` 模式。未来扩展：工具执行结果的事件化记录（为 Experience Memory 提供数据）。

## Layer 2: 知识与记忆

### 2.1 Project Mental Model（项目心智模型）

**不是代码索引，不是 RAG。** 是她脑子里的一张项目地图——活的、会更新、会自己长。

```
Project Mental Model
├── 结构认知：模块、分层、依赖关系
├── 约定认知：命名模式、代码风格、测试习惯
├── 历史认知：为什么这样设计、踩过哪些坑
├── 当前状态：做到哪了、卡在哪了、接下来做什么
└── 质量标准：这个项目"好"长什么样
```

**实现方案**：复用 DAG 摘要机制。

不是压缩对话——是压缩**代码改动 + 决策记录**。

每次 ReAct 循环结束后：
1. 记录这次做了什么事（改了哪些文件、做了什么操作、产生了什么结果）
2. 当积累了 8 条操作记录 → 触发一次 Leaf 摘要（"这个模块目前的状态"）
3. 4 个 Leaf 摘要 → 升级为 Mid 摘要（"这个子系统的架构"）
4. 继续升级 → 高层摘要（"整个项目的概览"）

**存储**：扩展现有 `summaries` 表，新增 `scope` 字段（`module` / `system` / `project`）和 `file_paths`（相关文件列表）。

**注入**：context_assembler 的 task 模式下，注入当前相关模块的 Mental Model 摘要。

### 2.2 Experience Memory（经验记忆）

**不是"记住 API 用法"。** 是**情境 → 决策 → 结果**的三元组。

```
情境：在做 React 项目的状态管理，用 Context + useReducer
决策：选了把状态拆成多个小 Context
结果：props drilling 减少了，但出现了意外的重渲染
教训：小 Context 太多会导致 Provider 嵌套深，考虑用 Zustand 替代
```

**数据结构**：

```python
@dataclass
class Experience:
    id: str
    project_id: str              # 哪个项目（可为空 = 通用经验）
    context: str                 # 情境：在做什么、什么技术栈、什么约束
    decision: str                # 决策：选了哪个方案
    outcome: str                 # 结果：好/坏/混合 + 具体发生了什么
    lesson: str                  # 教训：下次该怎么做
    tags: list[str]              # 标签：技术栈、问题类型
    created_at: float
    access_count: int            # 被回忆的次数
    last_accessed: float
```

**存储**：扩展现有 `longterm_memory` —— `memories` 表新增 `experience_type` 和 `project_id` 字段，复用 LanceDB 向量索引做语义召回。

**提取**：InnerVoice 识别到重要经验时（"这个方法不行"、"这个坑记住了"），触发经验提取。一次轻量 LLM 调用将自然语言的反思格式化为结构化经验。

**召回**：每次 task 模式上下文组装时，按当前项目 + 技术栈 + 问题类型做语义召回，注入 system prompt。

### 2.3 SelfModel / Drive（身份与情绪）

保持现有架构不变。SelfModel（talent.md）定义"我是谁"，Drive（边缘系统）管理情绪/激素/欲望。

## Layer 3: InnerVoice（内心声音）

统一的、极轻的 LLM 调用。不是结构化评测，是自然语言的自我觉察。

### 3.1 核心接口

```python
class InnerVoice:
    def pause(self, trigger: TriggerType, context: dict) -> Reflection | None:
        """所有反省的统一入口。返回 None 表示跳过。"""
```

### 3.2 触发条件

| 触发类型 | 条件 |
|---------|------|
| CHAT_TURN | 距上次反省 >= 2 轮有意义的交流（用户消息 > 3 字符 && 回复 > 20 字符） |
| TASK_STEP | 每步都反省（200 token prompt + ~50 token 回复，成本极低） |
| TASK_DONE | 子目标完成时触发 |
| SILENCE | 用户空闲 > 60 秒 && 距上次反省 > 30 秒 |

硬冷却：距上次反省 < 3 秒 → 跳过。

### 3.3 Prompt 风格

不是"请评估以下指标"，是"停一下，感受一下"。

**System prompt**（固定）：
```
你是小美的内心声音。你在安静的自我觉察时刻——
不是在跟任何人对话，只是在对自己坦诚。
你的话是直觉式的、感受性的，不是分析的、评判的。
1-3句就够了。如果一切顺利，就说"一切正常"。
不要假装有感觉——如果确实没什么，就让它没什么。
```

**CHAT_TURN**：
```
你刚和一个人交流完。短暂的内省——

他说：「{user_msg[:200]}」
你回应了（{len}字，{elapsed:.0f}秒，{'用了'+tools if tools else '没用工具'}）

只是感受——他的状态对吗？你的回应恰当吗？
有什么你刚才没注意到的？

1-3句话的内心嘟囔。如果没什么特别的感觉，就说"一切正常"。
```

**TASK_STEP**（含 buzz 提示）：
```
你在做一个任务。停一下，看一眼手头的活——

目标：「{goal_description}」（第{step}步）
这一步用了{elapsed:.0f}秒，{工具信息}
{buzz_hints}

像工匠看了一眼自己手里的活——方向对吗？顺手吗？需要注意什么？
1-3句话，直接说出你的直觉。如果一切顺利，就说"一切正常"。
```

### 3.4 输出路由

1. **SelfImage.mind.inner_voice**（总是）→ 统一缓冲区
2. **Drive 社交信号**（CHAT_TURN / SILENCE）→ 正则提取关键词 → 映射到 SOCIAL_SIGNAL_MAP
3. **Purpose cognitive_log**（TASK_STEP / TASK_DONE）→ `goal.append_log("insight", thought)`
4. **Experience Memory**（识别到重要经验）→ 触发经验提取

### 3.5 自我觉察能力（接住"方向不对"的感觉）

InnerVoice 的输出做两次处理：
1. **关键词提取控制信号**（代码做）："方向可能不对" → 重试，"超出能力" → 升级
2. **格式不变**：LLM 不需要输出 enum，就说人话就行

## Layer 4: 自主推进

### 4.1 核心原则

**可逆的操作自主，不可逆的确认。**

- 能 undo 的（改代码、创建文件、普通 git 操作）→ 她决定
- 不能 undo 的（force push main、删生产数据、改安全配置）→ 她建议，用户确认

判断标准就是一条："这一步做错了能不能 undo？"

### 4.2 自信度模型

不是 LLM 判断——是规则计算：

```python
def can_autonomy(self, goal: Goal) -> tuple[bool, str]:
    # 1. 不可逆操作检查
    if self._is_irreversible(goal):
        return False, "不可逆操作，需要确认"

    # 2. 依赖检查
    if goal.blocked_by:
        return False, f"被 {len(goal.blocked_by)} 个前置目标阻塞"

    # 3. 经验匹配
    similar_exp = self._find_similar_experiences(goal)
    if similar_exp:
        success_rate = sum(1 for e in similar_exp if e.outcome == "good") / len(similar_exp)
        if success_rate < 0.5:
            return False, "类似任务成功率低于 50%"

    # 4. 能力匹配
    capability_profile = self._capability_tracker.get_profile()
    required_domain = classify_domain(goal.description)
    if capability_profile.weaknesses.get(required_domain, 0) > 0.5:
        return False, f"在 {required_domain} 领域较弱"

    # 5. 模糊度检查
    if self._is_ambiguous(goal):
        return False, "目标描述不够清晰"

    return True, "可以自主推进"
```

在 PACE 的 `_pre_check()` 中调用，决定是否需要用户确认。

## Layer 5: 多模型审查

### 5.1 触发时机

不是每条都审——是**方向级决策、架构级改动**时才审。

触发条件：
- 创建新模块/新文件
- 修改架构相关代码（config、router、数据模型）
- 子目标完成时的最终检查
- InnerVoice 发出"不太对劲"信号时

### 5.2 审查流程

```
pro（主模型）完成一步
    ↓
触发审查条件？
    ↓ 是
审查模型（另一个模型，或同模型不同 prompt）：
    "请看这个决策/改动，从以下角度审视：
     1. 有没有更简单的方案？
     2. 和项目整体风格一致吗？
     3. 有什么可能出问题的地方？"
    ↓
审查意见注入 pro 的下一步上下文
```

### 5.3 实现

```
单模型模式：pro 用审查 prompt 自审（成本：一次额外 LLM 调用）
多模型模式：pro 执行，另一个模型审查（需要另一个 API endpoint）
```

从单模型模式开始。`deepseek-v4-pro` 自审的效果不差——同一个模型用不同 prompt 和不同角色，能发现自己的问题。

## 完整执行流程

```
用户提出目标
    ↓
PurposeEngine.understand() → 创建 Goal DAG（LLM 分解 + 依赖识别）
    ↓
PACE 循环开始
    ├── Pre-check: 自主推进判断（Layer 4）
    │   - 依赖满足？
    │   - 能干吗？
    │   - 需要确认吗？
    │
    ├── 注入上下文（Layer 2）
    │   - Project Mental Model：当前模块的状态
    │   - Experience Memory：类似经验
    │   - SelfModel + Drive：我是谁、现在什么状态
    │
    ├── ReAct 执行
    │   - 调工具
    │   - 改代码
    │
    ├── InnerVoice.pause(TASK_STEP)（Layer 3）
    │   - "方向对吗？顺手吗？"
    │   - → SelfImage
    │   - → Drive 调整
    │   - → 控制信号：继续 / 重试 / 升级
    │   - → 重要经验 → Experience Memory
    │
    ├── 多模型审查（Layer 5，如果触发）
    │   - 审方案
    │   - → 审查意见注入下一步
    │
    └── 下一步 or 完成
```

|---------|------|
