从认知科学角度简要解释：

## Global Workspace Theory（全局工作空间）

Baars 1988 年提出。核心比喻：大脑是一个剧场。

```
记忆 ⇢
身体 ⇢    ┌─────────────────────┐
感知 ⇢    │   全局工作空间（舞台）   │  → 意识 = 此刻在舞台上的内容
目标 ⇢    │   聚光灯照亮的部分      │
情绪 ⇢    └─────────────────────┘
                ↓
          LLM 读到 = "意识到"
```

- 多个脑区持续向工作空间**广播**信息
- 只有**竞争获胜**的信息进入工作空间 → 成为有意识的内容
- 没有进入工作空间的信息仍在后台运行（潜意识）
- 工作空间容量极有限（类比人的工作记忆，3-4 个组块）

对应你的系统：SelfImage 的数据都在后台产生，workspace 决定哪几个 section 此刻进入上下文窗口。没进入的（如 flow 模式下的记忆）相当于"潜意识"——数据在，但 LLM 读不到。

## Salience Network（显著性网络）

神经科学发现，2007 年左右。三个脑区组成：**前岛叶 + 前扣带回 + 杏仁核**。

```
                    "这个更重要！"
                         │
    前岛叶 ──────────────▼────────────── 前扣带回
    (身体信号)                           (冲突检测 + 决策)
                         │
                    杏仁核（情绪标记）
```

它的作用是**在所有并行信号中选出此刻最重要的那个**：

- 前岛叶读取身体状态（饥饿、心跳、疼痛）→ "身体在说什么"
- 杏仁核给信息打情绪标签 → "这件事危险/重要/有趣"
- 前扣带回检测目标和现实的冲突 → "你正在做的事和应该做的事不一样"

三者的输出汇合 → "**此刻应该注意这个**"。切换你的注意力。

对应你的系统：`_score_*` 评分函数就是显著性网络。body 的 score 来自 Drive 代理（前岛叶），mind 的 score 来自 purpose 代理（前扣带回），memory 的 score 来自关键词匹配（情绪标签）。它们竞争，高分入选。

## 两者的关系

```
显著性网络                 全局工作空间
(Salience Network)        (Global Workspace)
     ↓                          ↓
  "选哪个"                    "上屏了"
  评分 + 竞争                 进入上下文窗口
```

你的 workspace 模块同时做了两件事：scores 做显著性判断，inject 放入工作空间。如果以后分家，就是 `salience.py`（评分逻辑）+ `workspace.py`（注入逻辑）。目前合在一起够用。

## 思路


具体做法：

```python
# salience.py 新增
class SalienceProfile:
    """每个 section 的自适应权重记录"""
    section_weights: dict[str, float]  # section_name → learned weight
    hit_count: int                     # 被 LLM 引用次数
    render_count: int                  # 被渲染次数

    def feedback(self, section_name: str, was_useful: bool):
        if was_useful:
            self.section_weights[section_name] += 0.02
        else:
            self.section_weights[section_name] -= 0.01
```

```python
# _score_body 变成
def _score_body(si, mode, inner_voice_text, user_input):
    base = ...  # 现有规则
    learned = profile.section_weights.get("body", 0)  # +0.0 ~ +0.2
```

**检测 "LLM 是否引用了某个 section"** 用纯规则，不需要 LLM：

| Section | 检测方式 |
|---------|---------|
| memory | 回复含"之前/上次/记得/你说过" |
| body | 回复含"能量/心情/疲惫/状态"等 body 相关词 |
| being | 回复含名字/称呼/关系相关词 |
| inner_voice | 回复含"我在想/刚才在想" |
| mind | 回复含"目标/进展/计划" |

每轮对话结束后，做一个简单的正则匹配，更新 profile。这个 profile 跨会话持久化，每个 agent/user 一个。

## 为什么是现在做

1. **条件成熟**：workspace 架构已经稳定，评分函数全都接在 salience 里，加一个 learned weight 层不需要改管道
2. **数据免费**：不需要额外 LLM 调用，LLM 回复本身就带着反馈信号
3. **渐进式**：从 `section_weights` 默认 0.0 开始（行为不变），慢慢学习每个用户/场景的偏好
4. **马上见效**：一两轮对话后，body 在非健康讨论中自动降权，memory 在闲聊中自动降权

