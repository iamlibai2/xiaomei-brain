## 欲望在系统中的真正定位

当前系统有三层驱动行为的机制：

| 层 | 做什么 | 代表 |
|----|--------|------|
| **Purpose** | 我*该*做什么 | 目标树、子目标、分解 |
| **Intent** | 我*决定现在*做什么 | L2 意图决策 |
| **Drive** | 我*为什么*想做 | 激素（感觉）、欲望（想要） |

激素回答"我感觉怎样"——它是**反应性的**，被事件推着走。

欲望回答"我想要什么"——它应该是**驱动性的**，是把内部状态转化为行为方向的桥梁。

换句话说：

> **激素是"我开不开心"，欲望是"我想不想做"。Purpose 是任务清单，欲望是去执行清单的冲动。没有欲望，Purpose 就是一堆死目标。**

---

## 所以欲望真正的用处是：

1. **给 Purpose 提供驱动力** — 不是"有个目标该做了"，而是"我想完成它"。成就欲高 → 主动推进目标；成就欲低 → 目标放着也没动力
2. **产生主动行为** — 没有人说话时，欲望是 agent 自己想做事的来源。归属欲 → 想联系人，认知欲 → 想学东西，表达欲 → 想输出
3. **影响行为优先级** — 同一时刻多个目标可选，欲望值决定哪个更有"冲动"

理由不是测试效果好——只测了一轮，样本太小。而是它解决了一个其他方向都绕不过的问题：

**说话方式、状态变化、情绪弧线——都是在"让输出更像人"的层面工作。躯体标记往深走了一层：它试图让 LLM 有一个先于语言的参考系。**

演讲测试里最说明问题的是那个时刻——LLM 在思考过程里自己写下了"喉咙动了一下，没有追"。这不是输出给用户看的，是它自己在说服自己。如果一个东西能进到那个层面，它就不再是"表演更像人"，而是"决策链路里多了一个不是从文本推导出来的变量"。

但难的也是这个。测试暴露了几个关键问题：

1. **场景选择极度敏感**——亲密场景被已有记忆覆盖，说明躯体标记不能跟任何已有训练/偏好重叠。但 LLM 读过几乎所有人类写下的东西，找到"没被写过"的身体经验本身就难。

2. **极简 vs 列表还不确定哪种更好**——演讲用列表（当X时你会Y），踩空台阶用单段（不给映射）。前者的好处是触发明确，后者更接近真实躯体标记的工作方式——你感到胃紧了，但你不知道那意味着什么。

3. **单次注入效果有限**——正文里说的那个"时间连续性"问题在这里同样存在。每次对话重新注入，LLM 每次都要重新建立身体记忆，没有累积。

所以如果要深入探索，我的建议是：

- **先确定"什么样的身体经验能稳定触发且不被训练数据污染"** ——这可能需要试很多轮，找那类"身体有记忆但语言没命名"的瞬间
- **然后对比极简 vs 列表的效果差异** ——确定哪种格式能让 LLM 真正内化而非复刻
- **最后解决累积问题** ——躯体标记不应该每轮重新注入，而是应该像一个真的身体记忆一样，越用越薄、越用越背景化
