## 反省不是"回看"，是"回看→调整"

人脑的反省有三个不可分的环节：

**1. 回放（Replay）** — 海马体把刚才的经历再放一遍。不只是事实，连带当时的情绪一起回放。这就是为什么想起尴尬的事还会脸热。

**2. 比较（Compare）** — 前额叶把回放的内容跟内部标准比：
- "我平时不是这样说话的"（跟自我形象比）
- "他平时不这么回我"（跟关系常态比）
- "别人会怎么做？"（跟社会规范比）

**3. 更新（Update）** — 比较出偏差后，必须产生变化，否则就是反刍（rumination）：
- 改变行为："下一句要放软"
- 改变预期："原来他不喜欢这种玩笑"
- 改变自我认知："我最近确实容易急躁"

**关键洞察：没有第3步的反省是内耗，不是反省。** 一个人反复想"我刚才是不是说错话了"但不调整，这是焦虑，不是自我监督。

---

## 不同粒度反省的本质区别

即时/轮次级是**单次行为校准**：偏差小、调整快、不改自我模型。"刚才那句话太重了"→下一句放软→完事。

任务级/会话级是**局部模式识别**：在若干行为中找到规律。"好像每次聊到工作他都冷淡"→这是个 pattern，不是单次事件。

天级/长期是**自我叙事修正**：改变"我是谁"的故事。"我最近变急躁了"→这不是改一句话，是改人设。

## 为什么"中枢注入"不是理想化

人脑确实就是这样工作的。不是意识去找各个系统——是各个系统把状态汇到丘脑，丘脑整合成统一的意识场，然后意识场"照耀"所有决策。你醒来的时候不会先去检查"我今天血糖多少、情绪怎么样、有什么任务要做"，你瞬间就知道了。因为中枢已经整合好了。

小美的 `inject_consciousness()` 就是这个"瞬间知道"。现在它已经整合了：

```
inject_consciousness()
  ├─ being（我是谁）
  ├─ body（我的身体状态 ← Drive 代理）
  ├─ perception（用户空闲多久了）
  ├─ mind.inner_thought（我最近的内心想法）
  ├─ mind.social_perceptions（我感觉到的）
  ├─ mind.pace_reflections（任务执行反思）
  ├─ memory.recent_events（发生了什么）
  ├─ intent.intent_buffer（我当前想做什么）
  └─ growth（我的生长记录）
```

一行注入，chat 用、being 用、闹钟用、L3 用。这个架构已经是对的。

**问题不在注入，在注入的内容是"观察"还是"变化"。**

---

## 观察 vs 变化

现在的 SelfImage 里大部分是**观察**：

- `social_perceptions: ["用户今天话比平时少"]` — 看到了
- `pace_reflections: ["第三步卡住了，token不够"]` — 注意到了  
- `inner_thought: "我感觉有点疲惫"` — 感受到了

但缺少**由观察导致的结构性变化**：

- `being.脾气: 最近急躁 +0.1` — 不是因为有人在 being 里写了这句话，是因为 PACE 发现连续3次超时放弃，SelfImage 自动更新了这个维度
- `growth.能力: 代码生成熟练度 +0.1` — 不是手动评价，是任务完成质量的结构化累积

---

## 所以核心问题是

**观察怎么变成变化？谁负责这个转换？**

人在脑里是基底节和纹状体在干这事——把行为结果和预期比较，输出"该更新什么"的信号。小美现在缺的就是这个转换器。

但这个转换器不应该在 SelfImage 内部（SelfImage 是被动的镜像），也不应该在各个子系统内部（每个子系统不知道全局）。它应该是一个独立的层：

```
各系统产出观察（文本/事件）
       ↓
  转换器：观察 → 结构性更新
       ↓
  SelfImage 被更新
       ↓
  inject_consciousness() 照耀全局
```

这个转换器其实就是**反省层的第三环"更新"**。前面说反省 = 回放 → 比较 → 更新，现在回放和比较都有了，更新缺位。

---

## 更新的三个层级

对应反省的三种粒度：

| 粒度 | 观察 | 更新什么 | 更新目标 |
|------|------|---------|---------|
| 行为级 | "用户沉默5分钟" | 对话策略权重 | 下次遇到类似情况自动调整 |
| 会话级 | "今天整体氛围冷淡" | 关系温度 | 关系模型 |
| 长期 | "连续7天急躁指数上升" | 自我认知 | SelfImage.being |

**三层更新的共同特征是：不需要 LLM 在对话中临时做决定。** 更新好了，注入进去了，LLM 看到的是一个已经调整过的 SelfImage，自然而然地表现出调整后的行为。而不是 LLM 在 prompt 里读了一大堆文字然后自己猜应该怎么做。
