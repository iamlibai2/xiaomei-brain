已改为 1800 秒。craving 恢复机制：

```
tick_minute() 每分钟:
  pleasure_value  -= 0.2        # 快感快速衰减
  craving = expected - actual   # 渴望 = 预期 - 实际
  expected_pleasure -= 0.01     # 预期缓慢回落，地板 0.5
```

**为什么很快就回到 0.5：**

按压后：`pleasure=0.65, craving=0, expected=0.52`

| 分钟 | pleasure | expected | craving |
|------|----------|----------|---------|
| 0 | 0.65 | 0.52 | 0.00 |
| 1 | 0.45 | 0.52 | 0.07 |
| 2 | 0.25 | 0.52 | 0.27 |
| 3 | 0.05 | 0.52 | 0.47 |
| 4 | 0.00 | 0.52 | **0.52** ← 再次触发 |

也就是说按压后 **4 分钟 craving 就回到 0.5 以上**。但因为现在冷却 30 分钟，实际触发频率被限制在每 30 分钟一次。

**根本问题**：`expected_pleasure` 地板值是 0.5，而 `pleasure_value` 每分钟 -0.2 快速归零，所以 `craving = 0.5 - 0 = 0.5` 几乎永远满足。只要冷却一到，立刻又会触发。这不影响正常运行——30 分钟一次 mini chat 还好，但逻辑上 craving 是个"伪循环"。是否要调整？
