InnerVoice 目前有 **4 种触发类型**，分布在 2 个入口：

## 1. CHAT_TURN — 每次对话后触发（最频繁）

```
用户发消息 → LLM 回复 → task_orchestrator._invoke_inner_voice_chat_turn()
                              → inner_voice.on_chat_turn()
```

**触发点在 `task_orchestrator.py`**，`_run_react()` 方法中，每次用户对话 LLM 回复完就调用。这是最常见的触发路径，也是 salience 反馈闭环挂载的地方。

## 2. TASK_STEP — PACE 任务执行中每步触发

```
PACE loop → metacognition/runner.py:_invoke_inner_voice_task_step()
                → inner_voice.on_task_step()
```

任务执行过程中每个子步骤完成时触发，用于内省"这一步做得对不对、方向是否偏了"。

## 3. TASK_DONE — 子目标完成时触发

两个调用点：
- `metacognition/runner.py` → `inner_voice.on_task_done()` — PACE 任务完成
- `action_dispatcher.py` → `cl._inner_voice.on_task_done()` — 主动行为完成

## 4. SILENCE — 用户沉默时触发

用户在安静周期（如 5 分钟没说话）触发，用于生成内在思考。

## 统一入口：`pause()`

所有 4 种类型最终调用 `InnerVoice.pause(trigger_type)`，这是统一入口：

```
pause(trigger_type)
  → LLM 调用（轻量 prompt，让 LLM 内省当前状态）
  → 5 路输出：
     1. thought    → SelfImage.mind.inner_voice（文本，给下次 salience 评分用）
     2. EVENTS     → Drive（即时更新激素/情绪）
     3. SIGNAL     → Drive + RelationshipEngine（社交信号）
     4. GAPS       → LearningQueue（知识缺口）
     5. INSERT     → task_orchestrator（新增任务）
```

结构很简单：

```python
# consciousness/round_scheduler.py

class RoundScheduler:
    def __init__(self):
        self._round = 0
        self._handlers: dict[int, list[Callable]] = {}

    def every(self, interval: int, handler: Callable):
        """注册每 N 轮执行的行为"""

    def tick(self, **ctx):
        """每轮调用，按频率分发到对应 handler"""
        self._round += 1
        for interval, handlers in self._handlers.items():
            if self._round % interval == 0:
                for h in handlers:
                    h(**ctx)
```

| 特点 | 说明 |
|------|------|
| **独立线程** | 与 Living 主循环（处理消息）并行，互不阻塞 |

### 二、问题：主循环本质是"聊天服务器"
