## 实现方案

在 `assess_only()` 里，规则检测跑完后，追加一次轻量 LLM 调用：

```python
def _llm_surprise_reflect(self, user_msg: str, agent_output: str, 
                          tool_calls: list[str], elapsed: float) -> dict:
    """轻量 LLM 反射 — 一问一答，不带工具"""
    
    prompt = f"""你刚才完成了一轮对话。快速反省：

【用户说了】{user_msg[:200]}
【你的操作】调用了 {", ".join(tool_calls) or "无"} 工具，花了 {elapsed:.0f} 秒
【你的回复】{agent_output[:300]}

【请判断以下哪种情况最接近你的感受】：
1. 一切正常
2. 比预期耗时更长（简单的事搞了很久）
3. 用户的语气或措辞有不对劲（可能有言外之意）
4. 你的方法或思路可能不对
5. 你在重复自己（内容和之前相似）

只输出一行：
SURPRISE: <序号>
NOTE: <可选，一句话说明哪里不对劲>"""

    llm = self._agent_provider.llm
    resp = llm.chat(
        messages=[{"role": "user", "content": prompt}],
        tools=None,           # 不发工具
        max_tokens=100,       # 极短输出
    )
    return self._parse_surprise_reflect(resp.content)
```

## 关键：为什么 LLM 能判断"语气不对"

LLM 看到的不是孤立的一句话，而是**完整的对话上下文**。比如：

```
【用户说了】随便吧，你看着办
【你的操作】调用了 shell 工具，花了 12 秒
【你的回复】好的，那我帮你写一个 REST API...

→ LLM 反射: SURPRISE: 3  NOTE: "用户说'随便'、'你看着办'，可能有不耐烦或不满"
```

```
【用户说了】这个函数帮我改一下
【你的操作】调用了 read_file, shell, write_file 工具，花了 120 秒  
【你的回复】已完成修改...

→ LLM 反射: SURPRISE: 2  NOTE: "改一个函数不应该花2分钟，我可能绕了弯路"
```

这是因为 LLM 有常识——它知道"随便吧"通常不是真的随便，知道改一个函数不应该花 2 分钟。
