        "timestamp": time.time(),

    }

只收集本地可算的数据。不再每秒查 `conversation_db`、`ltm.count()`、`ltm.get_recent(5)`。

**2. 用户活跃 → 事件驱动，不轮询**

用户发消息时就已经知道了。`_handle_message()` 里已经调了 `drive.on_user_active()`，在那同时设 `last_user_activity_time`：

```python

```
`user_idle_duration` 变成惰性计算：`time.time() - last_user_activity_time`，不需要 `_sense()` 维护。

**3. 记忆数量 → 事件驱动**

记忆增加/删除时更新，不是每秒数一遍。在 `LongTermMemory.store()` 成功后调 `mind.update_memory_count()`。L1 的 `memory_loss` 异常检测从 `memory_count_history` 读，不需要每 tick 追加。

### 性能问题

**1. `to_dict()` 每秒被调用**

```python

def tick(self, perception):
    self.flame.last_cycle_state = self.to_dict()  # 序列化所有7模块
    self.update_from_perception(perception)
    changes = self._diff(self.flame.last_cycle_state)  # 只比较6个字段
```

`to_dict()` 序列化 40+ 字段，还触发 `body._sync_fallback()` 和 `mind._sync_fallback()` 各读 14 个属性。但 `_diff()` 只用了 6 个字段：`consciousness_age`、`agent_state`、`user_idle_duration`、`energy`、`memory_count`、`goal_progress`。每秒全量序列化只为比较 6 个值。

**2. `_save_snapshot()` 每 60s 无条件写盘**

即使 SelfImage 没有任何变化，仍然 `json.dump` 写 `latest.json`。

---

### 设计问题

**3. body/mind 的双重身份混乱**

同一个字段有两套存储：`_drive` 实时值和 `_fallback` 序列化值。getter 读 Drive，setter 写 fallback。`to_dict()` 前必须先 `_sync_fallback()`。这个模式让人困惑——看代码时不确定读到的是哪个值。

**4. 序列化格式不一致**

identity/relation/perception/growth 直接平铺在 `to_dict()` 顶层，但 body 和 mind 是嵌套对象：
```python
"identity": self.identity.identity,    # 平铺
"body": self.body.to_dict(),           # 嵌套
"mind": self.mind.to_dict(),           # 嵌套
```

**5. `from_dict()` 有大量"旧格式兼容"代码**

`SelfImage.from_dict()` 和 `SelfMind.from_dict()` 都有 `else: # 旧格式兼容` 分支。重构残留的技术债。

---

### 数据膨胀

**6. 内部列表无限增长**

`agent_state_history`、`accumulated_changes`、`goal_progress_history`、`memory_count_history` 只在序列化时截断（`[-10:]` / `[-20:]`），内存中永远不 trim。

---

### 逻辑问题

**7. `interpret_changes()` 的中文字段映射分散在两处**

```python
# interpret_changes
field_map = {"空闲": "user_idle_duration", "关系深度": "relationship_depth", ...}

# _get_field_value
path_map = {"user_idle_duration": self.perception.user_idle_duration, ...}
```
新增一个字段要改两个地方。

---

### 优化建议

| 问题 | 改法 |
|------|------|
| `to_dict()` 每秒 | `_diff()` 直接读6个属性，不调 `to_dict()` |
| 60s 无条件写盘 | 加 dirty flag，有变化才写 |
| body/mind 双重身份 | 去掉 fallback，Drive/Purpose 断开时直接抛异常或返回默认值 |
| 内部列表膨胀 | `collections.deque` maxlen 或定期 trim |
| 序列化不一致 | 要么全平铺，要么全嵌套 |
| 旧格式兼容 | 清理，不再需要 |
