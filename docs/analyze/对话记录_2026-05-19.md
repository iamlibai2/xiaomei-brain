对，人类是发完消息就挂，对方处理完再回。WebSocket 是两个人在线一直连着，不符合你说的"打电话说完就挂"。

**HTTP 就够了**：

- Agent A 发 assign → HTTP POST 到 Agent B → 返回 ack → 断开
- Agent B 自己循环中处理，做完 → HTTP POST 回报到 Agent A → 断开

每个 agent 只需要开一个轻量 HTTP server（收消息），发消息时作为 HTTP client 拨过去就行。Python 内置库就够，不需要额外依赖。

WebSocket 适合的场景是实时流式对话（比如飞书渠道），那才需要长连接。agent 间协作不需要。

**结论**：agent 间通讯走 HTTP + JSON，每个 agent 起一个小 HTTP server 收消息，发消息用 `requests` 直连。

   - Communication medium: HTTP + JSON (not WebSocket for agent-to-agent; WS reserved for human streaming)
   - Topology: P2P direct connection (like phone calls), no central hub
   - Address book: `agent_directory.yaml` for looking up other agents' addresses
   - Message types: chat, assign, query, report
   - Each agent starts its own HTTP server on a unique port to receive messages from other agents
   - `send_message` tool registered so LLM can call it to message other agents
   - Ports auto-assign from 18765 decrementing; configurable via config.json per-agent
   - Agent-to-agent replies MUST use `send_message` tool, not in-conversation reply
   - Fix duplicate message delivery bug (message received via both callback and inbox polling)

2. Key Technical Concepts:
   - HTTP direct communication (POST /message endpoint), stateless
   - Python stdlib `http.server` for lightweight HTTP server (no FastAPI dependency for comms)
   - `queue.Queue` thread-safe message passing between HTTP handler thread and living loop
   - SQLite `agent_inbox` table for durability (survives restart)
   - `agent_directory.yaml` as phonebook for agent address resolution
   - `@tool` decorator pattern from existing tools system
   - Setter pattern for tool context injection (`set_context()`)
   - Per-agent port auto-assignment with OSError fallback
   - Config priority: CLI `--port` > `config.json` > auto-assign (18765→18764→...)

## Agent 间通讯 — 完整总结

### 架构

**直连 P2P，无中心 Hub。** 像打电话：每个 agent 启动自己的 HTTP 服务器，通过通讯录查对方地址，直接 POST 消息过去。

### 模块清单

| 文件 | 作用 |
|------|------|
| `comms/protocol.py` | `AgentMessage` 数据类 + `MsgType` 枚举（chat/assign/query/report） |
| `comms/directory.py` | 通讯录，读写 `~/.xiaomei-brain/agent_directory.yaml`，注册/解析 agent 地址 |
| `comms/server.py` | 轻量 HTTP 服务器（stdlib `http.server`），`POST /message` 接收消息 |
| `comms/client.py` | HTTP 客户端（`urllib.request`），向其他 agent 发送消息 |
| `comms/inbox.py` | SQLite 收件箱，`agent_inbox` 表持久化，防重启丢失 |
| `tools/builtin/send_message.py` | LLM 可调用的 `send_message` 工具 |

### 消息流

```
Agent A                          Agent B
  │                                │
  │  LLM 调用 send_message()       │
  │       ↓                        │
  │  client.send_message()         │
  │  POST /message ─────────────→  │  server 收到
  │       ↓                        │       ↓
  │                                │  inbox.store(processed=0)
  │                                │       ↓
  │                                │  on_receive 回调 → _on_agent_message()
  │                                │       ↓
  │                                │  put_message() → 注入对话队列
  │                                │       ↓
  │                                │  mark_processed()（防止重复投递）
  │                                │       ↓
  │                                │  _check_inbox() 轮询（安全网）
  │                                │       ↓
  │                                │  LLM 看到 "[Agent消息 — 来自 A]"
  │                                │  + system prompt 顶部强制规则
  │                                │       ↓
  │                                │  LLM 调用 send_message() 回复
  │  ←─ 收到回复 ────────────────  │
```

### 关键设计决策

1. **HTTP 而非 WebSocket** — Agent 间消息低频、短小，HTTP 无状态更简单。WS 保留给人机流式对话
2. **端口自动分配** — 从 18765 递减，配置优先级：`CLI --port > config.json > 自动分配`
3. **消息即存** — 收到即写入 SQLite（`processed=0`），回调立即标记 `processed=1`，`_check_inbox()` 轮询兜底（处理 HTTP 回调遗漏的积压）
4. **system prompt 双重约束** — `_render_being()` 中永久规则 + `build_context()` 顶部 `!!!` 强制规则（检测到 agent 消息时注入）

### 已知问题

LLM 有时仍然无视 `send_message` 指令直接对话回复。根因是"用工具替代对话回复"属于**替代行为**（vs memory 指令的附加行为），模型本质上更倾向于生成文本。已通过 system prompt 顶部强制规则优化，但非 100% 可靠。

         try:
             term_w = shutil.get_terminal_size().columns
         except Exception:
             term_w = 80
         width = max(term_w - 4 - indent, 40)
         prefix = " " * indent
         paragraphs = content.split("\n\n")
         wrapped_lines: list[str] = []
         for para in paragraphs:
             para = para.strip()
             if not para:
