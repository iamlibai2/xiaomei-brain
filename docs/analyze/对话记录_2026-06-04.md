---

**[00:28:13] 🤖助手**

专用工具（Read/Edit/Grep/Glob）相比 shell 命令有几个优势：

**1. 透明性** — 用户能清楚看到我读/改/搜了哪些文件，diff 是什么，而不是一坨 shell 输出

**2. 权限控制更细** — 工具调用走独立的权限检查，不会让一个 `cat` 之类的无害命令给整个 Bash 放行

**3. 大文件更安全** — Read 限制行数、Grep 限制 head_limit，避免 `cat` 整个大文件或 `grep` 爆掉上下文

**4. 上下文效率** — 专用工具的输出格式更适合 LLM 处理，shell 输出可能带 ANSI 码/分页符

**5. 审查体验** — 工具调用在 UI 里单独展示，可以单独批准/拒绝，比翻 Bash 输出找关键行友好

简单说：**专用工具更适合人机协作的交互模型**，shell 命令更适合纯脚本场景。

## 日志诊断

**根因**：DeepSeek 账号 `Insufficient Balance`（HTTP 402）。代码逻辑正确——402 不在 `RETRYABLE_STATUS_CODES`（只有 429/5xx），所以**不重试、立刻抛 LLMError**。这不是 bug。

**问题是 LLM 客户端没"断路器"**：

| 现象 | 后果 |
|------|------|
| 每 10 分钟 L2 tick 仍照常发请求 | 反复 402，浪费 IO + 噪音日志 |
| 没记"最近 N 分钟已失败过" | 下一个调用点又走完整网络栈才发现 402 |
| 失败时 `_record_call(... is_error=True)` 仍触发 | memory erosion/wear 持续累加（第28→29次），把"欠费"误判为"我状态在衰退" |
| 内部叙事拿不到 → `🎯 意图: (未识别)` | 用户看到 ghost 状态 |

每周期 5 次 LLM 调（L2 inner voice / L2 procedure / L2 ReAct / 主对话 / 内部 7-信号 attention）—— **L2 一次 tick 大约 5 个 402**。`00:44 → 00:54 → 01:04` 三轮 L2 tick = 15 次无效请求。

# 2. 或切换 provider：~/.xiaomei-brain/config.json 改 default_provider
# 3. 或临时禁用 L2：conscious_living 关掉 L2 engine

## 代码侧建议（优先级排序）

1. **断路器**：在 `LLMClient` 加 `_circuit_breaker`——收到 402/401 时标记 provider 不可用 N 分钟，期间所有调用快速失败（不联网）。**这是真正的修复**。

# base/llm.py 加一个（不动 LLMClient 本身）
def safe_chat(llm_client, messages, *, fallback="", caller=""):
    """统一包装：catch + 分级日志 + 返回降级值。
    
    行为：
    - LLMError(retryable=False, 402/401) → ERROR 一次，然后 fallback（不重）
    - LLMError(retryable=True) → WARNING + fallback
    - 其他 Exception → ERROR + fallback
    - caller 用于日志标签，区别"内心独白"和"ReAct 意图"等
    """
    try:
        return llm_client.chat(messages)
    except LLMError as e:
        if not e.retryable:
            logger.error("[%s] LLM 不可用: %s", caller, e)
        else:
            logger.warning("[%s] LLM 临时失败: %s", caller, e)
        return fallback
    except Exception as e:
        logger.error("[%s] LLM 异常: %s", caller, e, exc_info=True)
        return fallback
```

```

1. 5 个调用点收敛到一个 helper
2. 错误日志格式统一
3. `exc_info=True` 可选，**避免每个调用点都喷完整栈**
4. 未来加断路器（前面说的 S+1）只在 helper 里改一处
