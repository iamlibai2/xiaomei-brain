# 配置文件系统技术文档

## 概述

xiaomei-brain 配置文件系统采用 OpenClaw 兼容的 JSON 格式，支持 CLI 直接访问和 Gateway RPC 动态更新。

## 配置文件格式

### 文件位置

| 文件 | 说明 |
|------|------|
| `config.json` | 主配置文件（项目根目录或 `~/.xiaomei-brain/`） |
| `config.json.bak` | 旧版备份 |
| `agents/` | Agent 目录（每个 Agent 有独立子目录） |

### 配置结构（OpenClaw 格式）

```json
{
  "meta": { "version": "1.0.0" },

  "models": {
    "mode": "merge",
    "providers": {
      "minimax": { "baseUrl": "...", "apiKey": "...", "models": [...] },
      "zhipu": { ... },
      "volcengine": { ... }
    }
  },

  "agents": {
    "defaults": { "model": { "primary": "minimax/MiniMax-M2.7" } },
    "list": [
      { "id": "xiaomei", "name": "小美", "talent": "...", "tools": {...} },
      { "id": "xiaoming", "name": "小明", ... }
    ]
  },

  "channels": {
    "feishu": { "enabled": true, "connectionMode": "websocket", "accounts": {...} }
  },

  "bindings": [
    { "agentId": "xiaomei", "match": { "channel": "feishu", "accountId": "xiaomei" } }
  ],

  "xiaomei_brain": {
    "agent": { "max_steps": 10, "context": {...} },
    "memory": { "similarity_threshold": 0.3 },
    "dream": { "idle_threshold": 300 },
    "tts": { "enabled": true, "voice_id": "female-tianmei" },
    ...
  }
}
```

## CLI 配置命令

### 命令列表

```bash
# 获取配置
python -m xiaomei_brain.cli config get <path>
python -m xiaomei_brain.cli config get                      # 获取完整配置

# 设置配置
python -m xiaomei_brain.cli config set <path> <value>      # 设置配置值（JSON 或字符串）

# 配置文件
python -m xiaomei_brain.cli config file                    # 打印配置文件路径
python -m xiaomei_brain.cli config validate                 # 验证配置格式
```

### 使用示例

```bash
# 获取配置值
python -m xiaomei_brain.cli config get xiaomei_brain.tts.speed
# {"value": 1.5, "hash": "8298c381583df8a8"}

# 设置配置值
python -m xiaomei_brain.cli config set xiaomei_brain.tts.speed 1.5
# {"success": true, "hash": "8298c381583df8a8"}

# 查看配置文件路径
python -m xiaomei_brain.cli config file
# /path/to/config.json

# 验证配置
python -m xiaomei_brain.cli config validate
# {"valid": true, "errors": []}
```

### 技术原理

#### 1. CLI 直接访问（离线模式）

```
┌──────────────┐         ┌────────────────┐
│  CLI         │────────→│  config.json   │
│              │  直接   │  (文件系统)    │
└──────────────┘  读写   └────────────────┘
```

**特点：**
- 不需要 Gateway 运行
- 直接读取/修改配置文件
- 无原子性保证（直接覆盖文件）
- 无热重载（修改后需手动重启）

**适用场景：**
- Gateway 未运行时修改配置
- 离线查看配置
- `validate` 和 `file` 命令

#### 2. Gateway RPC 访问（在线模式）

```
┌──────────────┐         ┌─────────────────┐         ┌──────────────────┐
│  CLI         │────────→│   Gateway       │────────→│  Config          │
│              │  WS     │   WebSocket     │  RPC    │  Provider        │
└──────────────┘         └─────────────────┘         └──────────────────┘
```

**Gateway RPC 方法：**
```bash
# 获取配置
openclaw gateway call config.get --params '{}'

# 完整替换配置
openclaw gateway call config.apply --params '{
  "raw": "{ ... }",
  "baseHash": "<hash>",
  "restartDelayMs": 2000
}'

# 部分更新
openclaw gateway call config.patch --params '{
  "raw": "{ channels: { feishu: { enabled: true } } }",
  "baseHash": "<hash>"
}'
```

**特点：**
- 需要 Gateway 运行
- WebSocket + JSON-RPC 通信
- Hash 冲突检测（防止并发修改）
- 热重载（配置变更自动通知订阅者）

#### 3. 两种方式对比

| 特性 | CLI 直接访问 | Gateway RPC |
|------|-------------|-------------|
| 是否需要 Gateway | 否 | 是 |
| 通信方式 | 直接文件系统读写 | WebSocket + JSON-RPC |
| 原子性 | 无 | 有（hash 冲突检测） |
| 热重载 | 无 | 有（patch 后自动 restart） |
| 权限控制 | 文件系统权限 | Gateway token 鉴权 |
| 适用场景 | 离线修改、查看 | 运行时动态更新 |

## 核心组件

### 1. ConfigProvider（配置中心）

```python
# src/xiaomei_brain/config_provider.py

class ConflictError(Exception):
    """配置冲突：baseHash 不匹配"""
    pass

class ConfigProvider:
    def __init__(self, config_path: str):
        self.config_path = config_path
        self._config: dict = {}
        self._hash: str = ""
        self._lock = threading.RLock()
        self._change_handlers: list[Callable] = []
        self._load()

    @property
    def hash(self) -> str:
        """返回当前配置的 hash"""
        return self._hash

    def get(self, path: str = "") -> Any:
        """获取配置，支持 dot notation 路径"""
        ...

    def apply(self, new_config: dict, base_hash: str = "") -> None:
        """完整替换配置"""
        if base_hash and base_hash != self._hash:
            raise ConflictError("Config has been modified")
        ...

    def patch(self, partial: dict, base_hash: str = "") -> None:
        """部分更新配置（JSON Merge Patch）"""
        merged = self._deep_merge(self._config, partial)
        self.apply(merged, base_hash)

    def subscribe(self, handler: Callable) -> None:
        """订阅配置变更"""
        self._change_handlers.append(handler)
```

### 3. GatewayRPCServer（RPC 服务器）

```python
# src/xiaomei_brain/gateway_rpc_server.py

class RPCHandler:
    """RPC 请求处理器"""

    async def handle(self, method: str, params: dict) -> dict:
        if method == "config.get":
            return await self._config_get(params)
        elif method == "config.patch":
            return await self._config_patch(params)
        elif method == "config.apply":
            return await self._config_apply(params)

class GatewayRPCServer:
    """Gateway RPC WebSocket 服务器"""

    def __init__(self, config_path=None, host="127.0.0.1", port=18789):
        self.host = host
        self.port = port
        self.provider = get_provider(config_path)
        self.rpc_handler = RPCHandler(self.provider)

        """停止 RPC 服务器"""

    """打印配置文件路径"""
```

### 5. 全局单例

```python
_provider: ConfigProvider | None = None

def get_provider(config_path: str | None = None) -> ConfigProvider:
    """获取全局 ConfigProvider 单例"""
    global _provider
    if _provider is None:
        _provider = ConfigProvider(config_path)
    return _provider
```

## Gateway RPC 实现原理

### config.get

```bash
CLI                              Gateway
  │                                  │
  │──── config.get ────────────────→│
  │                                  │ configProvider.get(path)
  │                                  │
  │←─────────────────────────────────│
  │   { value, hash }                │
```

### config.patch

```bash
CLI                              Gateway
  │                                  │
  │──── config.get ────────────────→│ 获取 hash
  │←─────────────────────────────────│
  │   { hash: "v1" }                │
  │                                  │
  │──── config.patch ──────────────→│
  │   { raw, baseHash: "v1" }      │
  │                                  │ 1. 校验 hash
  │                                  │ 2. JSON Merge Patch
  │                                  │ 3. 写入文件
  │                                  │ 4. 广播 config.changed
  │                                  │ 5. 触发 restart
  │←─────────────────────────────────│
  │   { success: true }             │
```

### Hash 冲突检测

```
Client A                    Client B
    │                            │
    ├─ config.get ── hash="v1" ──┼─ config.get ── hash="v1"
    │                            │ 修改配置
    │                            ├─ config.patch(hash="v1") → OK
    ├─ config.patch(hash="v1") ─→│ 409 Conflict!
    │   (基于旧配置)              │  (配置已过期)
```

## 热重载模式

| 模式 | 行为 |
|------|------|
| `off` | 不重载 |
| `hot` | 仅热安全变更（如 TTS 参数） |
| `restart` | 需要变更时完全重启 |
| `hybrid` (默认) | 热应用安全变更，需要时重启 |

## 迁移记录

### 旧版配置（已废弃）

| 旧文件 | 废弃原因 | 替代方案 |
|--------|---------|---------|
| `config.yaml` | YAML 格式不兼容 OpenClaw | `config.json` |
| `config.example.yaml` | 模板文件 | `config.json` 示例 |
| `~/.xiaomei-brain/agents.json` | Agent 注册表 | `config.json` 的 `agents.list` |

### 迁移步骤

1. `config.yaml` → `config.json`
   - 将 YAML 结构转换为 JSON
   - Provider 配置映射到 `models.providers`
   - Agent 定义移到 `agents.list`

2. `agents.json` → `config.json`
   - Agent 配置合并到 `agents.list`
   - 每个 Agent 的 `talent.md` 保留在 `agents/{agent_id}/` 目录

## 注意事项

1. **Hash 冲突**：使用 RPC 修改配置时，必须携带 `baseHash`，否则会报错
2. **热重载**：非安全变更（如 channel 配置）会触发 Gateway 重启
3. **离线修改**：CLI 直接修改文件后，需要手动重启 Gateway 才能生效
4. **Agent talent**：`talent.md` 文件独立于 `config.json`，可单独编辑
