## 完整架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户输入                                  │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Agent（爬行动物脑）                                              │
│  - 接收用户输入                                                  │
│  - 调用 LLM                                                      │
│  - 执行工具                                                      │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Metacognition（元认知）                                         │
│  - check()：执行前检查                                          │
│  - review()：执行后复盘                                         │
│  - 输出：Adjustment                                             │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Purpose（意图理解）                                              │
│  - understand()：理解用户意图                                    │
│  - 分解目标：Phase Goal → Executable Goals                      │
│  - 输出：Goal 列表                                               │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Basal Ganglia（行为选择）                                       │
│  - select_action()：从候选中选择最优                            │
│  - 习惯匹配：context → action（0 LLM）                          │
│  - 价值评估：expected_reward - cost                             │
│  - 输出：                                                        │
│      → 确定性 action → 直接执行（不需要 LLM）                  │
│      → 放入上下文 → 交给 LLM 处理                              │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Memory（记忆）                                                  │
│                                                                  │
│  被调用：                                                         │
│    ← Basal Ganglia 查习惯                                       │
│    ← Metacognition 查经验                                        │
│    ← Agent 查偏好/知识                                          │
│                                                                  │
│  主动更新（后台线程）：                                          │
│    → 定期梳理（每 N 分钟）                                      │
│    → 梦境整合（idle/午夜）                                      │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Drive（动力）                                                    │
│  - on_event()：事件驱动更新状态                                 │
│  - 输出：DriveSignals（dopamine, serotonin, stress...）         │
│  - 影响 Basal Ganglia 的价值评估                                 │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Tool 层                                                          │
│  - 工具注册表                                                    │
│  - 工具执行                                                      │
└─────────────────────────────────────────────────────────────────┘
```

---

## 各层 LLM 调用情况

| 层 | 需要 LLM | 说明 |
|---|---------|------|
| Agent | ✅ | LLM 是核心决策者 |
| Metacognition | ✅ | 判断理解是否正确 |
| Purpose | ✅ | 理解意图、分解目标 |
| Basal Ganglia | ⚠️ | 复杂情况才用，简单用规则 |
| Drive | ❌ | 纯规则/算法 |
| Memory | ⚠️ | 生成摘要时才用，存/取不用 |
| Tool | ❌ | 执行工具 |

## 流程示例：ERP 系统

### 用户输入

```
用户："帮我做一个ERP系统"
```

### Step 1: Agent 接收

```
Agent 收到消息
    ↓
记录到对话日志
    ↓
触发 Memory 主动备份检查
    ↓
传递给上层处理
```

### Step 2: Metacognition 检查

```
Metacognition.check()
  输入：用户意图（模糊的）

  分析：
    - "ERP系统"范围很大
    - 需要澄清：完整还是某个模块？
    - 需要澄清：用什么技术栈？

  输出：
    Adjustment(
      type=REQUEST_CLARIFICATION,
      reason="ERP系统范围太大，需要确认具体需求"
    )
```

### Step 3: 用户澄清

```
xiaomei："你想做完整的ERP系统还是某个模块？用什么技术栈？"

用户回答：
  "先做订单和库存模块，用 Python FastAPI"
```

### Step 4: Purpose 层理解

```
Purpose.understand()
  输入：用户意图（明确的）

  理解：
    - 目标：帮用户开发订单和库存模块
    - 技术：Python FastAPI

  分解目标：
    Phase Goal："完成订单和库存模块"

    Executable Goals：
      1. 确认数据库表结构
      2. 实现订单管理 CRUD
      3. 实现库存管理 CRUD
      4. 实现订单-库存联动
      5. 编写测试
```

### Step 5: Basal Ganglia 选择

```
当前候选：5个目标

Basal Ganglia.select_action()
  检查习惯：
    - 没有"做ERP"的习惯

  价值评估：
    Goal 1 "确认数据库表结构"
      - expected_reward: 高（后面都依赖它）
      - cost: 低（就问几个问题）
      - value: 高 ← 选择

  输出：
    selected_goal = "确认数据库表结构"

  判断：
    - 这需要和用户交互
    - 不是确定性 action
    - 放入上下文，交给 LLM
```

### Step 6: Memory 被调用

```
Memory.recall()
  查询："数据库设计"、"ERP"、"FastAPI"

  返回相关记忆：
    - 用户偏好：喜欢简洁的代码
    - 之前项目：没有相关经验
    - 知识：FastAPI 如何设计 REST API
```

### Step 7: Drive 影响

```
Drive.on_event("new_goal")
  更新状态：
    - dopamine += 0.2（期待奖励）
    - motivation_level 正常

  影响 Basal Ganglia：
    - motivation_level 正常，不调整成本
```

### Step 8: LLM 处理

```
上下文组装：
  - 系统提示
  - 用户输入
  - Memory 召回的相关记忆
  - Basal Ganglia 结果："先确认数据库表结构"
  - 最近对话

LLM 生成回复：
  "好的，我来帮你做订单和库存模块。
   首先确认一下数据库设计..."
```

### Step 9: Agent 执行

```
LLM 返回文本
    ↓
Agent 展示给用户

用户回答数据库设计问题
    ↓
进入下一个循环
```

## 多步流程管理

### 目标状态存储

```
Working Memory 中：

current_workflow = {
  phase_goal: "完成订单和库存模块",
  sub_goals: [
    {id: 1, name: "确认数据库表结构", status: "completed"},
    {id: 2, name: "实现订单CRUD", status: "in_progress"},
    {id: 3, name: "实现库存CRUD", status: "pending"},
    {id: 4, name: "实现联动", status: "pending"},
    {id: 5, name: "编写测试", status: "pending"}
  ],
  current_step: 2
}
```

### 每个消息的循环

```
用户每发一条消息：

1. 检查 Working Memory 有没有进行中的 goal
   ├── 有 → 继续当前 goal
   └── 没有 → 新对话

2. Basal Ganglia 选择当前 goal

3. 执行直到完成

4. 更新 goal 状态

5. 选择下一个 goal
```

### 状态持久化

```
对话结束：
  - Working Memory 持久化到磁盘

下次对话开始：
  - 读取 Working Memory
  - "上次你说到实现订单CRUD，现在继续？"
  - 继续从当前 step 开始
```

## 后台任务执行

### 场景

```
用户："帮我做ERP系统"
xiaomei："好的，开始处理，完成后通知你"
    ↓
用户去做其他事
    ↓
xiaomei 在后台持续工作
    ↓
可能需要用户确认某些步骤（主动发消息）
    ↓
完成时通知用户
```

### 实现方式

```
用户发起任务
    ↓
创建后台任务
    ↓
任务调度器持续执行：
    while not completed:
        执行下一步
        if 需要确认:
            主动发消息给用户
            等待用户回复
        if 时间到:
            定时发消息给用户（进度报告）
        sleep(N分钟)
    ↓
完成后通知用户
```

### 触发场景

| 触发方 | 场景 | 实现 |
|--------|------|------|
| 用户主动 | 用户发消息 | 正常流程 |
| xiaomei 主动 | 用户回来了、定时任务到期、后台完成 | proactive engine |
| xiaomei 主动 | 后台任务需要确认 | 任务调度器 |
| xiaomei 主动 | 后台任务完成 | 任务调度器 |

## 主动备份机制

### 时间间隔触发

```
后台线程（每 N 分钟）：
  → 读取最近 N 分钟的对话
  → LLM 梳理提取重要信息
  → 写入记忆
```

### 规则触发

```
检测到重要信号时立即备份：
  - 包含"记住"
  - 包含"我以前"
  - 包含"喜欢/要/不要"
  - 重复出现

符合规则 → 立即备份（简单处理）
```

### 梦境模式

```
idle / 午夜触发：
  → 深度整合
  → 权重更新
  → 清理低权重记忆
```

## 关键设计决策

| 问题 | 决策 |
|------|------|
| LLM 参与 | 大多数判断都需要 LLM |
| Basal Ganglia 结果 | 确定性 action 直接执行，否则放入上下文 |
| Agent 角色 | Agent = LLM + 工具执行 |
| Memory 触发 | 规则 + 定期后台 + 梦境 |
| 多步流程 | Working Memory 存储状态，持久化 |
| 后台任务 | 后台线程执行，主动通知用户 |
