# xiaomei-brain 项目演进历史

> 整理自 docs/analyze/对话记录_2026-04-11 至 2026-04-24（共13天）

---

## 核心理念

**AI Agent的三个根本问题**：
1. **记忆** — Agent如何保存和检索经验？
2. **生命** — Agent如何从"响应工具"变成"持续运行的个体"？
3. **自我** — Agent的"自我意识"是什么？

这13天的开发始终围绕这三个问题，每个阶段都是对这三维的深化。

---

## 阶段一：从零到可运行Agent（04-12）

**起点**：用户想开发OpenClaw插件（连接消息平台和AI模型），但随后转向构建自己的Agent框架。

**ReAct循环**：`agent.stream()` — 用户输入 → 组装messages → LLM调用 → 工具执行（如果有）→ 返回响应

**API切换**：OpenAI → 火山引擎 → **智谱GLM-5.1**（国产，长上下文）

## 阶段二：获得记忆能力（04-13）

**问题**：每次对话Agent都是从零开始，没有"记忆"。

### 增加了什么

**语义搜索记忆系统**：
```
用户输入 → embed() → cosine相似度搜索 → 相关记忆 → 注入context → LLM
```

**核心模块**：
- `memory/search.py`：本地sentence-transformers all-MiniLM-L6-v2（384维）
- `memory/indexer.py`：MemoryIndexer管理MEMORY.md索引
- `memory/store.py`：统一入口（搜索/保存/列表）
- `memory/conversation.py`：对话记录（JSONL，按日期）
- `memory/dream.py`：梦境处理器
- `memory/scheduler.py`：梦境调度器（空闲5分钟或午夜触发）

**8项优化**（一次性全部实现）：

| # | 优化项 | 实现 |
|---|---|---|
| 4 | 相似度阈值过滤 | `MIN_SCORE=0.3`，低分不注入 |
| 2 | 持久化向量索引 | npz + json双文件缓存 |
| 7 | 增量索引更新 | 检测新增/修改文件 |
| 3 | 段落级chunk分片 | 按##标题分片 |
| 6 | 时间衰减 | `score * exp(-0.005 * days)` |
| 8 | 容量控制 | `max_topics=100` + LRU淘汰 |
| 1 | 自动提取记忆 | 梦境机制（每晚/空闲触发） |
| 5 | 语义去重合并 | `similarity >= 0.75`时合并 |

**性能优化**：每条消息触发2-3个LLM调用（慢4-10秒）→ 改启发式关键词匹配 → 27.9s → 5.2s

### 完善了什么

- Embedding懒加载（首次使用时才加载模型）
- API重试（429/5xx/网络错误指数退避）
- 启动时预加载embedding（避免首次请求阻塞）
- Surrogate字符过滤（JSON序列化崩溃）

## 阶段三：发现"压缩丢失"问题（04-15）

**问题**：引入对话日志后，发现了一个根本矛盾——所有记忆系统的设计倾向都是**压缩**（摘要、精华、丢细节），但用户的核心需求是**一字不差保存**。

### 增加了什么

**对话日志层**：一字不差存入SQLite，永久保存

### 完善了什么

**上下文组装**的认知深化：
- 当前只有"滑动窗口"，没有"永久记忆"
- 需要区分：短期（当前对话）、长期（跨对话）

### 剔除/修复了什么

**发现问题**：摘要丢掉的是最有价值的——语气、情绪、具体细节
- 这不是Bug，是**架构层面的设计问题**

## 阶段四：DAG分层摘要架构（04-18~04-19）

**借鉴**：Lossless-claw的DAG方案

**核心洞察**：
- 对话天然是多主题交叉的
- 树结构必须归到一个父节点下
- DAG可以同时链接到多个摘要节点

### 增加了什么

**新记忆架构**：
```
SelfModel（启动加载，极小）

DAG摘要图谱（分层压缩 + 按需展开）
  叶子节点：8条 → LLM摘要（约118 tokens）
  高层节点：多个叶子 → 再次压缩

对话日志（SQLite，一字不差，FTS5搜索）
```

**新模块**：
- `memory/conversation_db.py`：SQLite对话日志，FTS5搜索
- `memory/dag.py`：分层摘要图谱，双向链接可展开
- `memory/longterm.py`：向量记忆，user_id隔离
- `memory/extractor.py`：提取器（immediate/periodic/dream三模式）
- `memory/context_assembler.py`：上下文组装（daily/flow/reflect三模式）

**记忆提取三模式**：
| 模式 | 时机 | 内容 |
|------|------|------|
| immediate | 对话中关键词触发 | 快速存入 |
| periodic | 固定周期 | 定期整理 |
| dream | 梦境阶段 | 深度提取 |

**上下文组装三模式**：
| 模式 | 场景 | 内容 |
|------|------|------|
| flow | 心流/快速对话 | identity + 最近4条 |
| daily | 日常对话 | SelfModel + DAG + 长期记忆 + 最近6条 |
| reflect | 反省/深度 | 完整SelfModel + 更多摘要 + 更多记忆 |

### 完善了什么

- **记忆冲突检测**：ADD/UPDATE/MERGE/DELETE/NOOP决策逻辑
- **BUG修复**：UPDATE时多调一次LLM提炼核心事实（避免"确认"越来越臃肿）

### 技术约束

**MiniMax只允许1个system message**：DAG摘要+长期记忆都合并到SelfModel system prompt，不单独创建system message

## 阶段五：从工具到生命体（04-20）

**问题**：之前的Agent只是"响应工具"，用户发一句它回一句，没有生命周期概念——不会主动做事，不会等待，不会休息。

### 增加了什么

**AgentLiving状态机**：
```
DORMANT → WAKING → AWAKE
                      ↓
              SLEEPING ↔ DREAMING
```

**状态说明**：
| 状态 | 行为 |
|------|------|
| DORMANT | 初始状态 |
| WAKING | 苏醒中 |
| AWAKE | 处理消息，超时→SLEEPING |
| SLEEPING | 等待消息，超时→DREAMING |
| DREAMING | 梦境强化，超时→SLEEPING |

**ProactiveOutput**：四种主动触发
- WAKE：进程启动
- REMINDER：定时提醒
- RECALL：记忆召回
- IDLE：长时间空闲

### 完善了什么

- 从"单轮对话"变成"持续运行的进程"
- 有空闲检测（30分钟无消息→SLEEPING）
- 有梦境周期（5分钟SLEEPING→DREAMING）
- 有主动行为（问候、提醒）

### 剔除/修复了什么

- **ProactiveEngine废弃**：独立线程轮询 → AgentLiving状态驱动
- `proactive.py`移入`src/bak/`
- asyncio事件循环不适用（chat/embedding/LLM都是同步阻塞）
- `queue.Queue`替代`asyncio.Queue`

## 阶段六：上下文爆炸Bug修复（04-22）

**问题**：messages在ReAct循环中不断累积，每轮对话都追加，导致上下文越来越大。

### 增加了什么

**认知**：正常ReAct应该让messages自然累积，但我们的架构里context是每次从外部组装的（通过ContextAssembler），所以messages在ReAct内部累积是错的。

### 完善了什么

- self.messages放到AgentInstance上，贯穿整个生命周期
- fresh_tail从SQLite读取 → 改为不读SQLite
- 每轮对话开始时清空self.messages

### 剔除/修复了什么

**BUG**：fresh_tail读出了主动问候消息，但用户没有回复，导致上下文只有assistant的消息
- **BUG**：MiniMax不遵循system message指令 → MEMORY_DECISION_PROMPT改为追加到user message

## 阶段七：三层集成——有内在世界的自主体（04-23~04-24）

**问题**：Agent有了记忆、有了生命状态，但它有"自我"吗？它的内在世界是什么？

### 增加了什么

**Drive层（边缘系统）**：
```
情绪/欲望/动机，状态机+事件驱动

Emotion（情绪）     — 分钟级衰减，快速响应
Hormone（激素）     — 小时级衰减，多巴胺/血清素/皮质醇/催产素
Motivation（激励）  — 奖励预测误差（RPE机制）
Desire（欲望）      — 归属/认知/成就/表达，驱动目标
```

**Purpose层（前额叶）**：
```
三层目标层级：

Meaning（存在意义）→ Phase Goals（阶段目标）→ Executable Goals（执行目标）

目标管理：添加/分解/完成/优先级计算
```

**Consciousness层（意识）**：
```
核心理念：意识如火焰，代码维护骨架，LLM加柴涌现

分层心跳：
  L0 — tick_L0()  每秒，火焰骨架维护（age++）
  L1 — tick_L1()  每分钟，异常检测，解读accumulated_changes
  L2 — tick_L2()  动态，LLM轻度加柴，产出Intent
  L3 — tick_L3()  DREAMING，LLM深度燃烧，梦境报告

动态加柴时机：
  用户空闲 > 5分钟
  累积变化 > 10条
  上次加柴 > 10分钟
```

**三层集成关系**：
```
Consciousness（协调层）
  ├── LLM加柴 → 产出Intent（行动意图）
  ├── 内在感知 → 规则生成（状态观察，不调LLM）
  └── 读取Drive/Purpose状态 → 注入LLM上下文

Drive（边缘系统）
  ├── 事件驱动（分析对话提取表扬/批评/进展）
  └── 欲望行为（归属欲>0.7→问候，认知欲>0.8→学习）

Purpose（前额叶）
  ├── 目标完成 → 通知Drive激素变化
  └── 学习主题 → 来自当前目标
```

### 完善了什么

**Self系统模块化**（受大脑分区启发）：
```
SelfIdentity    # 前额叶 — 身份/价值观（静态）
SelfState     # 下丘脑 — 实时监测（实时）
SelfRelation  # 边缘系统 — 关系深度/信任
SelfPerception # 感官输入 — 用户状态/环境
SelfMemory    # 海马体 — 记忆存取
SelfGrowth    # 皮层 — 累积经验/模式识别
```

**内在感知设计**：
- 最初：LLM调用产生想法（和L2加柴重复）
- 04-23讨论决定：规则生成，不调LLM
- 规则示例：`能量<0.4 + 空闲>600秒 → "我有些疲惫，用户离开很久了..."`

### 剔除/修复了什么

- **撤销错误修复**：之前的sync workaround（`time.time()`重置）被删除
- **内在感知彻底分离**：加柴调LLM（产出Intent），内在感知用规则（产出状态描述）
