# 记忆系统升级补充设计：情绪记忆层

## 背景

在设计三层记忆系统（碎片化+场景标签+关系权重+梦境加固）时，遗漏了一个关键维度：**情绪**。

前期分析人类记忆原理时曾明确指出——"情绪是记忆的胶水，可给记忆打情绪强度标签，高情绪强度检索优先级更高"。但写设计文档时未将其纳入。

本设计为该缺失维度的补充方案。

---

## 1. 设计思路

### 1.1 为什么需要情绪

检索记忆时，同等相关度的候选中：
- 情绪强度高的记忆 → 更优先浮现
- 情绪基调匹配当前状态的记忆 → 更容易被唤起

没有情绪维度的记忆系统，检索结果是纯事实排序——像在读一本没有标注重点的笔记。

### 1.2 情绪在记忆中的三个作用

| 作用 | 说明 |
|------|------|
| **锚定** | 情绪强度越高的记忆，被巩固的倾向越强，衰减越慢 |
| **过滤** | 检索时情绪强度参与排序，让"有感觉的记忆"优先于"平淡的事实" |
| **氛围关联** | 相同情绪基调的记忆之间建立弱关联——当你在开心时，更容易想起其他开心的事 |

## 2. 设计方案

### 2.1 存储层扩展

每条记忆碎片上，新增两个字段：

#### 2.1.1 情绪标签 (emotion_label)

可选值：一组基础情绪标签，可多选。

**基础情绪标签集**（灵感来源于Ekman基础情绪理论，结合场景需要）：

| 类别 | 标签 |
|------|------|
| 正向 | joy（喜悦）, love（爱）, peace（平静）, hope（希望）, gratitude（感激） |
| 负向 | sadness（悲伤）, anger（愤怒）, fear（恐惧）, disgust（厌恶）, shame（羞耻） |
| 中性 | surprise（惊讶）, curiosity（好奇）, nostalgia（怀旧）, confusion（困惑） |

情绪标签可在运行时由LLM在记忆写入时自动标注，无需人工预设。

**设计决策**：不限定死标签集，基础标签提供默认值，允许LLM根据内容扩展或组合。存储时保留原始标签字符串。

#### 2.1.2 情绪强度 (emotion_intensity)

取值范围：0.0 ~ 1.0

| 值域 | 含义 | 示例场景 |
|------|------|---------|
| 0.0~0.2 | 近乎无感 | "今天天气不错" |
| 0.3~0.5 | 轻微感受 | "收到了一个意料中的回复" |
| 0.6~0.8 | 强烈感受 | "被误会了，有点难受" |
| 0.9~1.0 | 极致体验 | 某次刻骨铭心的对话或事件 |

**强度来源**：由LLM在写入记忆时，根据内容情绪浓度自动评估。

### 2.2 场景情绪氛围 (scene_emotion_vibe)

场景标签（如"家里""工作"）本身也可附带一个累积的情绪氛围值。

**用途**：当检索命中某个场景时，该场景的"情绪底色"会影响该场景下所有记忆的检索排序。

例如："家"这个场景积累了很多"peace""love"标签——那么当你处于放松状态时，"家"相关的记忆更容易被唤起。

**存储方式**：场景标签表新增字段 `emotion_vibe: map[string]float`，记录每种情绪标签在该场景下的累积强度。

### 2.3 检索层扩展

在现有的检索流程（场景匹配 → 关键词命中 → 关联扩散 → 现场组合）基础上，增加情绪相关步骤：

#### 2.3.1 情绪上下文注入

检索时，LLM可以显式传入当前的情绪上下文，例如：
- "当前情绪基调：curiosity+0.3"
- 或者不传，默认使用中性

#### 2.3.2 情绪加权排序

检索命中的候选记忆碎片，最终得分 = 原有相关度 × (1 + emotion_intensity × 0.5)

即：情绪强度满分的记忆，相关性权重提升50%；0强度的记忆，不增减。

#### 2.3.3 情绪氛围匹配加分

如果候选记忆的情绪标签与当前情绪基调匹配（如当前joy + 记忆joy），额外加分系数1.2。

### 2.4 梦境加固层扩展

#### 2.4.1 情绪衰减

现有梦境系统中的记忆强度（importance）会随时间衰减。情绪强度也应当衰减——让时间冲淡感受。

- 每次梦境运行时，对未触发的记忆：`emotion_intensity *= 0.95`
- 情绪强度低于 0.1 时，标记为"情绪中性"（保留标签但强度归零）

#### 2.4.2 情绪强化

当某条记忆在梦境中被加固（ReinforceJob触发时），其情绪强度不被衰减反而轻微回升：
- `emotion_intensity = min(1.0, emotion_intensity + 0.05)`
- 情绪标签不变

#### 2.4.3 氛围衰减与更新

场景的 `emotion_vibe` 也随梦境衰减，但每当该场景下有新记忆写入时，氛围会被重新计算（取该场景下所有未衰减记忆的情绪标签均值）。

## 3. 数据结构变更

### 3.1 记忆碎片表（memory_fragments）新增字段

```sql
-- 新增字段
emotion_labels    TEXT         -- JSON数组，如 '["joy","gratitude"]'
emotion_intensity REAL DEFAULT 0.0  -- 0.0 ~ 1.0
```

### 3.2 场景标签表（scene_tags）新增字段

```sql
-- 新增字段
emotion_vibe      TEXT         -- JSON对象，如 '{"joy":0.6,"peace":0.3}'
```

### 3.3 检索接口调整

```python
def retrieve_fragments(
    scene: str | None = None,
    keywords: list[str] | None = None,
    emotion_context: dict | None = None,  # 新增，如 {"label": "joy", "intensity": 0.3}
    top_k: int = 10
) -> list[Fragment]:
    ...
```

## 4. 与其他三层的关系

本层不替代已有三层，而是在其基础上叠加：

| 层 | 作用 | 与情绪层的关系 |
|----|------|--------------|
| 碎片化存储 | 原子信息单元 | 每条碎片附加情绪字段 |
| 场景标签 | 索引维度 | 场景增加情绪氛围 |
| 关系权重 | 碎片间关联强弱 | 同基调碎片增加弱关联 |
| 梦境加固 | 离线巩固与衰减 | 情绪强度衰减与加固 |

简单说：**情绪是第四维度，贯穿前三层。**

---

## 5. 边界情况

1. **无情绪标注的记忆**：emotion_intensity 默认为0.0，不参与情绪加权，不影响检索
2. **情绪标签冲突**：一条记忆可以有多个情绪标签（如"又气又笑"），所有标签均保留，排序时取最高强度
3. **场景氛围空值**：新场景无氛围数据时，检索不触发氛围匹配加分
4. **情绪上下文为空**：检索时不传情绪上下文，则不触发情绪匹配加分，只保留强度加权
