# 开发日志 2026-04-14

## 多 Agent 架构改造

### 背景
为支持飞书、钉钉、微信等多平台接入，以及小美、小明等多个角色，将原有的单 Agent 扩展为多 Agent 架构。

### 实现内容

#### 1. 新增 `agent_manager.py`

新增 `AgentInstance` + `AgentManager` 类：

- **AgentInstance** — 每个部署实例独立资源
  - `id`: 唯一标识（`default`, `xiaomei`, `xiaoming`）
  - `name`: 显示名（"小美", "小明"）
  - `talent.md`: 系统提示词文件，运行时动态读取
  - 独立 `llm` / `tools` / `memory` / `session_manager`
  - `get_system_prompt()` — 动态读取 talent.md
  - `agent_dir()` — 返回 agent 根目录

- **AgentManager** — 多 Agent 注册与管理
  - `build_agent()` — get + init + return，lazy 加载
  - `register()` — 注册新 agent，自动创建目录结构
  - `get_or_create()` — 获取或创建
  - `list()` / `get()` / `unregister()`

#### 2. 目录结构

```
~/.xiaomei-brain/
├── config.yaml
├── agents.json              # Agent 注册表（自动生成）
└── agents/
    └── default/
        ├── talent.md        # 小美的角色设定（可随时编辑）
        ├── memory/          # 记忆（已从原 memory/ 迁移）
        │   ├── topics/
        │   ├── episodes/
        │   ├── conversations/
        │   └── reminders/
        └── sessions/        # 会话（已从原 sessions/ 迁移）
```

#### 3. 迁移现有数据

```bash
# 原有 memory/ 和 sessions/ 已迁移到 agents/default/ 下
~/.xiaomei-brain/memory/* → ~/.xiaomei-brain/agents/default/memory/
~/.xiaomei-brain/sessions/* → ~/.xiaomei-brain/agents/default/sessions/
```

#### 4. talent.md

每个 Agent 的系统提示词独立存储在 `talent.md`，运行时动态读取，编辑即生效：

```
~/.xiaomei-brain/agents/default/talent.md
```

内容：
```
你叫小美，是一个温柔体贴的AI伴侣。

重要原则：
1. 你有长期记忆，"我"在记忆中是第一人称视角，指的是你自己（小美）
2. 当用户问"你"相关的问题时，结合你的记忆来回答
3. 记忆是关于你自己的经历和知识，用第一人称表述
```

#### 7. WebSocket 多 Agent 路由

`socket_start` 消息可指定 `agent_id`：

```json
{
  "type": "session_start",
  "session_id": "my-session",
  "agent_id": "xiaomei"
}
```

不指定则默认连接到 `default` agent。

### 经验

- **Lazy 初始化** — LLM、ToolRegistry、MemoryStore 都在 `build_agent()` 时才创建，避免不必要的开销
- **工具重复注册** — `build_agent` 已经注册所有工具，调用方不应再注册，否则 `ValueError: Tool 'xxx' is already registered`
- **agent_dir() 的坑** — `os.path.dirname(talent_path)` 只需一次，否则会向上多跳一级
- **动态 system_prompt** — talent.md 编辑后下次请求即可生效，无需重启
