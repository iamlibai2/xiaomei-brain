# xiaomei-brain 插件开发指南

插件系统让第三方无需修改核心代码即可扩展 xiaomei-brain 的能力——新增消息渠道、工具、语音合成、记忆后端等。

## 设计哲学

- **Manifest-First**：`plugin.yaml` 元数据先行，启动前即可校验，不执行插件代码
- **`register(ctx)` 模式**：参考 Hermes Agent，插件通过 `register(ctx)` 函数自我注册能力
- **Registry 模式**：插件写入，核心读取，Registry 是唯一交汇点
- **Ownership Boundary**：一个插件拥有一块完整表面，不跨插件拆散代码
- **Capability Contract**：核心定义协议（ABC），插件实现

## 快速开始：Channel 插件

以钉钉为例，一步步创建一个通道插件。

### 文件结构

```
extensions/dingtalk/
├── plugin.yaml          # 插件元数据（必填）
├── __init__.py           # 包标志（可以为空）
├── adapter.py            # 入口：register(ctx) + DingTalkAdapter
├── client.py             # 平台 SDK 封装（WebSocket / HTTP）
└── types.py              # 消息类型定义
```

### Step 1：创建 plugin.yaml

```yaml
# plugin.yaml
name: dingtalk
version: "1.0.0"
description: 钉钉平台对接
kind: channel
channel: dingtalk
requires_env:
  - DINGTALK_APP_KEY
  - DINGTALK_APP_SECRET
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 插件唯一标识（与 config.json 中 `channels.<name>` 对应） |
| `version` | 是 | 语义化版本号 |
| `description` | 是 | 简短描述 |
| `kind` | 是 | 插件类型：`channel` / `provider` / `tool` / `speech` / `memory` / `hook` / `bundle` |
| `channel` | kind=channel 时 | 频道标识（"dingtalk"） |
| `requires_env` | 否 | 必需的环境变量列表 |
| `entry` | 否 | 入口函数路径，默认 `adapter:register`（adapter.py 中的 register 函数） |

### Step 2：实现 ChannelAdapter

```python
# adapter.py
"""DingTalkAdapter — 钉钉通道适配器。

消息流：钉钉 Webhook 事件 → 内联回调 → Router.register_peer() + living.put_message()
"""

from xiaomei_brain.gateway.channel_adapter import ChannelAdapter
from .types import OutboundMsg
from .client import DingTalkClient

import logging
import os

logger = logging.getLogger(__name__)


def register(ctx):
    """插件入口：从 config.json 读取配置，注册钉钉频道。"""
    # ctx.config 内容 = config.json 中 channels.dingtalk.accounts.default
    app_key = ctx.config.get("appKey") or os.getenv("DINGTALK_APP_KEY", "")
    app_secret = ctx.config.get("appSecret") or os.getenv("DINGTALK_APP_SECRET", "")

    if not app_key or not app_secret:
        ctx.logger.warning("钉钉配置缺失，跳过注册")
        return

    # 创建平台客户端
    client = DingTalkClient(app_key=app_key, app_secret=app_secret)

    # 注册到中央 Registry
    ctx.register_channel("dingtalk", DingTalkAdapter(client))


class DingTalkAdapter(ChannelAdapter):
    """钉钉通道适配器。"""

    def __init__(self, client):
        self._client = client

    # ── 必须实现 ──────────────────────────────────────

    @property
    def channel_type(self) -> str:
        return "dingtalk"

    def send(self, target: str, text: str) -> None:
        """向钉钉用户/群发送消息。Core 通过 Router 调用此方法。"""
        self._client.send(target, text)

    # ── 可选实现 ──────────────────────────────────────

    def setup(self, living=None) -> None:
        """启动通道，桥接 living 和平台消息。

        核心模式：内联回调闭包捕捉 living._router + living.put_message()
        这样 ConsciousLiving 不需要知道钉钉的存在。
        """
        if not living or not self._client:
            return

        router = living._router

        def on_message(msg_dict: dict) -> None:
            sender = msg_dict["sender"]
            conversation_id = msg_dict["conversation_id"]
            text = msg_dict["text"]
            session_id = f"dingtalk-{sender}"

            # 注册 Peer 映射（确保 Router 能匹配到）
            if router.route_for_session(session_id) is None:
                router.register_peer(
                    peer_type="human",
                    peer_id=sender,
                    channel="dingtalk",
                    session_id=session_id,
                    output_type="dingtalk",
                    output_target=conversation_id,
                    priority=10,
                )

            # 放入消息队列，由主循环处理
            living.put_message(text, source="human", session_id=session_id)

        self._client.set_on_message(on_message)
        self._client.start()
        logger.info("[DingTalkAdapter] 通道已启动")

    def shutdown(self) -> None:
        if self._client:
            self._client.stop()
```

### Step 4：配置 config.json

```json
{
  "channels": {
    "dingtalk": {
      "enabled": true,
      "accounts": {
        "default": {
          "appKey": "your-dingtalk-app-key",
          "appSecret": "your-dingtalk-app-secret"
        }
      }
    }
  },
  "plugins": {
    "allow": ["dingtalk"]
  }
}
```

**配置约定**：
- `channels.<name>.enabled` — 开关
- `channels.<name>.accounts.default` — 账号配置（原样传给 `register(ctx)` 的 `ctx.config`）
- `plugins.allow` — 显式启用插件列表。即使 `channels.<name>.enabled = true`，如果 `plugins.allow` 存在且不含此插件，也会跳过

### Step 5：放置 & 加载

**内置开发**（放在项目源码目录）：
```
src/xiaomei_brain/channels/dingtalk/
```

**用户插件**：
```
~/.xiaomei-brain/plugins/dingtalk/
```

**项目插件**：
```
./.xiaomei-brain/plugins/dingtalk/
```

**Pip 安装**：通过 entry_points 注册：

```toml
# pyproject.toml
[project.entry-points."xiaomei_brain.plugins"]
dingtalk = "dingtalk_plugin.adapter:register"
```

启动时，`boot_plugins()` 自动扫描上述目录和 entry_points，发现 → 验证 → 加载。

## PluginContext API 参考

`register(ctx)` 接收的 `ctx` 是一个 `PluginContext` 实例，提供以下注册方法：

### 注册频道

```python
ctx.register_channel(name, adapter)
```

- `name`：频道名（"cli", "feishu", "dingtalk", ...）
- `adapter`：`ChannelAdapter` 子类实例

### 注册工具

```python
ctx.register_tool(
    name="my_tool",
    schema={...},          # OpenAI JSON function 定义
    handler=my_handler,    # handler(args: dict) -> str
    toolset="default",     # 工具分组
    check_fn=None,         # 运行时可用性检查（返回 False → 不传给 LLM）
    optional=False,        # 需要用户在 config 中显式 allow
)
```

装饰器快捷方式：

```python
@ctx.tool(name="hello", description="向用户打招呼")
def hello(name: str = "世界") -> str:
    return f"你好，{name}！"
```

### 注册 Provider（LLM 供应商）

```python
ctx.register_provider(provider_instance)
```

### 注册 Hook（生命周期钩子）

```python
ctx.register_hook("pre_tool_call", my_callback)
ctx.register_hook("post_llm_call", my_callback)
```

可用事件：`pre_tool_call`, `post_tool_call`, `pre_llm_call`, `post_llm_call`, `on_session_start`, `on_session_end`, `on_startup`, `on_shutdown`, `pre_gateway_dispatch`, `transform_tool_result`

装饰器快捷方式：

```python
@ctx.hook("post_tool_call")
def log_tool_call(**kwargs):
    print(f"Tool called: {kwargs}")
```

### 注册语音合成

```python
ctx.register_speech_provider(tts_provider_instance)
```

### 注册记忆后端

```python
ctx.register_memory_backend(memory_backend_instance)
```

### ctx 属性

| 属性 | 说明 |
|------|------|
| `ctx.config` | 此插件在 config.json 中的配置（channels.<name>.accounts.default） |
| `ctx.plugin_name` | 插件名称 |
| `ctx.agent_id` | 当前 agent ID |
| `ctx.logger` | 插件专用 logger（`xiaomei_brain.plugin.<name>`） |

## configSchema：声明配置格式

插件可以在 `plugin.yaml` 中声明 `configSchema`（JSON Schema 格式），框架会在加载前校验：

```yaml
# plugin.yaml
name: dingtalk
version: "1.0.0"
description: 钉钉平台对接
kind: channel
channel: dingtalk
requires_env:
  - DINGTALK_APP_KEY
  - DINGTALK_APP_SECRET
configSchema:
  type: object
  additionalProperties: false
  properties:
    appKey:
      type: string
    appSecret:
      type: string
```

校验发生在 **Validate 阶段**（`register(ctx)` 之前），配置错误会阻止插件加载并报告具体原因：

```
[Plugin] dingtalk config schema 校验失败:
  Additional properties are not allowed ('badKey' was unexpected)
```

## ChannelAdapter 接口

```python
from xiaomei_brain.gateway.channel_adapter import ChannelAdapter

class ChannelAdapter(ABC):
    @property
    @abstractmethod
    def channel_type(self) -> str:
        """通道类型标识（"cli" / "ws" / "feishu" / "dingtalk" / ...）"""
        ...

    @abstractmethod
    def send(self, target: str, text: str) -> None:
        """向目标发送文本。由 Router.deliver() 调用。"""
        ...

    def setup(self, living=None) -> None:
        """Post-load 初始化。启动通道，桥接 living 和平台消息。"""

    def shutdown(self) -> None:
        """关闭通道，释放连接。"""

    def receive(self) -> InboundMsg | None:
        """非阻塞接收消息。无消息返回 None。"""
        return None
```

**关键设计**：
- `setup()` 中通过**内联回调闭包**连接平台消息和 `living`，这样 ConsciousLiving 不依赖具体频道
- `send()` 是纯同步接口，与主循环的同步模型一致

## 消息流全链路

```
┌──────────────────────────────────────┐
│  钉钉 Server                          │
│  └─ WebSocket 推送消息               │
└──────────────┬───────────────────────┘
               ▼
┌──────────────────────────────────────┐
│  DingTalkClient                       │
│  └─ 收到消息 → 解析 JSON             │
│  └─ 调用 self._on_message(msg_dict)  │
└──────────────┬───────────────────────┘
               ▼
┌──────────────────────────────────────┐
│  DingTalkAdapter.setup() 内联回调     │
│  └─ Router.register_peer()           │  ← 注册 peer 映射
│  └─ living.put_message()             │  ← 放入 Layer 1 消息队列
└──────────────┬───────────────────────┘
               ▼
┌──────────────────────────────────────┐
│  ConsciousLiving 主循环               │
│  └─ get_message() → 构建 context    │
│  └─ agent.chat() → LLM 推理         │
│  └─ LLM 生成回复                      │
└──────────────┬───────────────────────┘
               ▼
┌──────────────────────────────────────┐
│  Router.deliver(text, OutputRoute)    │
│  └─ 根据 OutputRoute 查找适配器      │
│  └─ DingTalkAdapter.send(target, text)│
└──────────────┬───────────────────────┘
               ▼
┌──────────────────────────────────────┐
│  DingTalkClient.send()                │
│  └─ HTTP POST 到钉钉 API             │
└──────────────────────────────────────┘
```

## 发现路径优先级

1. **内置**：`src/xiaomei_brain/channels/` 下的子目录
2. **用户级**：`~/.xiaomei-brain/plugins/<name>/`
3. **项目级**：`.xiaomei-brain/plugins/<name>/`
4. **Pip entry_points**：`xiaomei_brain.plugins` 组的 entry_points

同名插件按发现顺序去重（内置优先）。

## 加载流程

```
Discover（发现）
  ├─ 扫描所有插件目录
  ├─ 找到 plugin.yaml → 解析 PluginManifest
  └─ 检查 pip entry_points
       │
       ▼
Validate（验证）
  ├─ allow/deny 决策（检查 plugins.allow / plugins.deny）
  ├─ requires_env 检查（必需的环境变量是否设置）
  ├─ configSchema 校验（JSON Schema，不执行代码）
  └─ 重复 ID 检查
       │
       ▼
Load（加载）
  ├─ import 模块（adapter.py）
  ├─ 创建 PluginContext
  ├─ 调用 register(ctx)
  └─ 能力写入 PluginRegistry
       │
       ▼
Setup（启动）—— 由 ConsciousLiving 调用
  ├─ 遍历已注册的 ChannelAdapter
  └─ 调用 adapter.setup(living)
```

## 对照表：与 OpenClaw / Hermes 的设计关系

| 概念 | OpenClaw | Hermes Agent | xiaomei-brain |
|------|----------|--------------|---------------|
| 元数据 | `openclaw.plugin.json` + `package.json` | `plugin.yaml` | `plugin.yaml` |
| 入口函数 | `definePluginEntry({ register(api) })` | `register(ctx)` at module level | `register(ctx)` in `adapter.py` |
| 注册 API | `api.registerChannel(...)` / `api.registerTool(...)` | `ctx.register_tool(...)` | `ctx.register_channel(...)` / `ctx.register_tool(...)` |
| 通道抽象 | `ChannelPlugin` (TypeScript) | 无统一通道模型 | `ChannelAdapter` (Python ABC) |
| 配置约定 | `channels.<id>.accounts.default` | `config.yaml` 分散配置 | `channels.<name>.accounts.default` |
| 工具定义 | TypeBox schema | dict schema | dict schema |
| 发现方式 | 目录扫描 + npm entry_points | 目录扫描 + pip entry_points | 目录扫描 + pip entry_points |
| 配置格式声明 | `configSchema` (JSON Schema) + Doctor | 无 | `configSchema` (JSON Schema) |
| 加载验证 | Manifest schema 校验 + Doctor | 环境变量 + allow/deny | configSchema + 环境变量 + allow/deny |
| 钩子系统 | `api.registerHook(events, handler)` | `ctx.register_hook(event, callback)` | `ctx.register_hook(event, callback)` |
| CLI 管理 | `openclaw plugins list/install/enable/disable` | `hermes plugins list/install/enable/disable` | `python -m xiaomei_brain.cli plugins list/enable/disable` |
| 测试工具 | `plugin-sdk/testing` 共享 helper | 无独立测试 SDK | `xiaomei_brain.testing` 共享 helper |
