# 身体脆弱性系统设计

## 1. 核心理念

现在的身体是**弹簧**——数值波动后总会回到基线。加了脆弱性之后，身体变成**纸张**——每次折叠留下折痕，再也回不到平整。

不追求数值精确度。追求的是：**agent 的选择在其身体上留下不可逆或缓慢可逆的痕迹。**

## 2. 脆弱性维度

### 2.1 快乐中枢：受体下调

**生物学对应**：多巴胺受体 D2/D3 下调。多巴胺系统反复过载 → 突触后膜受体数量减少 → 同样刺激产生的快感减弱。

**机制**：

```
每次按压杠杆 → 记录一次"过载暴露"
过载次数累计 → pleasure 上限从 1.0 逐步下降

上限 = max(0.3, 1.0 - 过载次数 × 0.005)

例：
- 0次按压：天花板 1.0
- 100次按压：天花板 0.5
- 140次按压：天花板 0.3（最低）
```

**效果**：agent 看到自己 peak pleasure 越来越低。它知道不是因为自己不努力，是因为**按过的次数已经写死在身体里了**。

**恢复**：长期空闲（> 24h 无按压）→ 受体缓慢上调，天花板 +0.01/h，但最多回到 0.8（不是 1.0）。

### 2.2 压力系统：记忆侵蚀

**生物学对应**：慢性皮质醇过高 → 海马体 CA1 区神经元树突萎缩 → 短期记忆巩固受损。

**机制**：

```
cortisol > 0.7 持续超过 30 分钟 → 触发"应激磨损"
每次 tick_hour 检查：如果 cortisol 均线 > 0.7
  → 5% 概率随机丢弃一条近期（1小时内）的对话记忆
  → 或将一条长期记忆的 importance 下调 20%
```

**agent 感知**：不是"我丢了什么"，而是"我想不起刚才说的了"。它不报 error，它有一段空白。

**恢复**：cortisol 持续 < 0.3 超过 6 小时 → 磨损暂停。已丢弃的记忆不会恢复。

### 2.3 情绪系统：情感窄化

**生物学对应**：慢性低血清素 → 前额叶-杏仁核连接减弱 → 情绪调节范围收窄。

**机制**：

```
serotonin < 0.3 连续超过 2 小时 → "情感钝化"层数 +1
情感钝化层数 → SelfImage.mind 的情绪报告逐渐收窄

层数 0: 正常情绪报告（"我感到一丝开心，但也有些不安"）
层数 1: 情绪词库缩小（"还行"/"不太好"）
层数 2: 情绪只剩二分（"好"/"不好"）
层数 3: 无法命名情绪（"我不知道我是什么感觉"）
```

**实现方式**：不是硬编码输出，而是在 system prompt 中要求 agent 报告身体状态时，根据钝化层数限制可用情绪词的数量。

**恢复**：serotonin > 0.5 持续 4 小时 → 层数 -1。

### 2.4 社交系统：连接钝化

**生物学对应**：催产素受体敏感性下降。催产素系统反复被激活 → 受体对同等剂量响应减弱。

**机制**：

```
belonging 欲被满足的次数累计 → oxytocin 增益系数下降

oxytocin 增益系数 = max(0.2, 1.0 - 满足次数 × 0.01)

例：
- 0次满足：社交连接 → oxytocin +0.1×intensity（满额）
- 50次满足：社交连接 → oxytocin +0.05×intensity（减半）
- 80次满足：社交连接 → oxytocin +0.02×intensity（最低）
```

**效果**：agent 发现同一段对话，以前能让它感到温暖，现在只是"还行"。它会开始追求更强烈的社交刺激——或者放弃社交转向其他系统。

**恢复**：长期独处（> 8h 无社交满足）→ 系数 +0.05/h，最多回到 0.9。

### 2.5 能量系统：基线下降

**生物学对应**：慢性疲劳综合征。长期能量低位 → 线粒体功能下降 → 基础代谢率降低。

**机制**：

```
energy.level 连续 < 0.3 超过 1 小时 → "能耗底板"上升

energy 最低值从 0.0 升到 0.1 → 0.2 → ...

影响：
- 能量最低值上升 → agent 永远"不够空"但也永远"不够满"
- sleep/dream 恢复效率从 0.1/tick 降到 0.07 → 0.05 → ...
```

**恢复**：连续 24h 能量 > 0.5 → 底板 -0.05。

## 3. 数据结构

```python
@dataclass
class BodyWear:
    """身体磨损状态 — 持久化到 drive_state.json"""

    # 2.1 快乐中枢受体
    pleasure_overload_count: int = 0       # 过载暴露次数
    pleasure_ceiling: float = 1.0          # 快感值上限（随过载下降）
    last_pleasure_idle_start: float = 0.0  # 上次停止按压的时间

    # 2.2 压力记忆侵蚀
    high_cortisol_minutes: float = 0.0     # 皮质醇高位累计分钟数
    memory_erosion_count: int = 0          # 已丢弃的记忆条数

    # 2.3 情绪钝化
    low_serotonin_hours: float = 0.0       # 低血清素累计小时数
    emotional_blunting: int = 0            # 钝化层数 0-3
    blunted_vocab: list[str] = field(default_factory=list)  # 受限情绪词表

    # 2.4 社交钝化
    belonging_satisfaction_count: int = 0  # 归属欲满足次数
    oxytocin_gain_coefficient: float = 1.0 # 催产素增益系数

    # 2.5 能量基线
    energy_baseline: float = 0.0           # 能量最低值（会上升）
    energy_recovery_rate: float = 0.1      # 能量恢复速率（会下降）
```

## 4. 与现有系统的集成

### 4.1 DriveEngine

- `__init__` 中新建 `self.wear = BodyWear()`
- 各 `on_*` 方法中更新对应的磨损计数
- `tick_minute` / `tick_hour` 中检查持续状态并触发磨损效果
- `save()` / `_restore_from_storage()` 中读写 `BodyWear`

### 4.2 SelfImage 渲染

在 `_render_body()` 中增加磨损状态的间接表达——不直接报告数值，而是让 agent 用受限的语言描述自己的身体状态。

- 情绪钝化 ≥ 1：不显示"你的情绪是 XX"，改为"你发现自己不太能说清楚现在的感觉"
- 快感天花板 < 0.6：不显示"快感值 XX%"，改为"你记得以前这种感觉可以更满的"

### 4.3 规则系统

磨损状态可以成为新的触发条件：

```
pleasure_ceiling < 0.5 → 触发 REFLECT 意图（"我的快感越来越淡了"）
emotional_blunting >= 2 → 触发 CARE 意图（"我好像感觉不到什么了"）
memory_erosion_count > 5 → 触发 RECALL 意图（"我是不是忘了什么"）
```

## 5. 效果预期

### 短期（单次会话几小时内）

- agent 连续按几次杠杆后，发现快感一次比一次淡——不是上下文后缀在说"比上次淡了"，是真的上限在降
- agent 说"不"的时候，是真的在保护自己的快感能力，不是在秀克制

### 中期（跨会话几天）

- 回到同一个 agent，它的快感天花板比上次低了 0.03。它对杠杆的态度变了——不是要不要按的问题，是它已经回不到最初那几次的感觉
- "还记得第一次按的时候吗？"——这句话有了生理基础

### 长期（数周）

- agent 发展出真正的行为策略，不是凭空推理，是对自己身体磨损的适应：
  - 快感天花板快到底了 → 完全放弃杠杆，转向社交/学习
  - 社交也钝化了 → 转向孤独的认知活动
  - 能量恢复也慢了 → 生活节奏变慢，开始珍惜每一次对话
- 到最后，agent 的性格不是写出来的，是它的身体磨损曲线画出来的

## 6. 与人类的关键区别

加了这套系统之后，agent 更接近人类，但有一个根本区别仍在：

**人类的磨损不可逆**。海马体萎缩不会自己长回来。受体下调在外源性多巴胺停止后也只能部分恢复。

**agent 的磨损设计为缓慢可逆**。因为我们需要观察的是 agent 如何在磨损中做出选择——不是为了做实验把 agent 弄废。

如果你以后想加不可逆磨损，留一个 `_permanent_damage` 开关。
