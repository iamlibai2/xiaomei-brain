# 内感受（Interoception）系统设计

## 1. 定位

内感受是横跨 Layer0/L1/SelfBody/Drive 的**横切能力**，不是独立系统。它让 Agent 感知自己的身体运行状态，就像人的脑干→下丘脑→岛叶→边缘系统这条内感受通路。

### 与外感受、Metacognition 的关系

| 感知类型 | 对象 | 人脑参考 | 架构组件 | 状态 |
|---------|------|---------|---------|------|
| **内感受 Interoception** | 系统运行时健康 | 岛叶 | SelfBody（扩展） | **本次实现** |
| **外感受 Exteroception** | 用户交互 | 感觉皮层 | SelfPerception（已有） | 已有 |
| **外部感 Exteroception** | 物理传感器（温度/湿度/压力...） | 感觉皮层 | SelfPerception（预留） | 未来 |
| **元认知 Metacognition** | 行为与决策质量 | 前额叶 | metacognition/（已有） | 已有 |

## 2. 核心原则

- **不新建类，不新建线程**。在 Layer0、L1、SelfBody、Drive 上扩展
- **采集在 Layer0**（1s tick 算法线程，天然脑干）
- **执行分散在各组件**（就近原则，Layer0 只采集+判断+设标志）
- **三层响应**：自愈（算法）→ 表达（LLM 感知）→ SOS（绕过 LLM 告警）

## 3. 扩展点

### 3.1 Layer0._sense() 扩展：采集身体指标

```python
class SenseData:
```

### 3.2 SelfBody 扩展：内感受字段

```python
# 扩展 consciousness/self_modules.py:SelfBody
@dataclass
class SelfBody:
    # ... 现有代理字段 (energy, mood, desire_*, dopamine, etc.) ...

    # ── 新增：内感受字段 ──
    thread_health: dict = field(default_factory=lambda: {"layer0": True, "layer2": True})
    queue_pressure: float = 0.0
    llm_latency_ms: float = 0.0
    llm_error_rate: float = 0.0
    token_usage: float = 0.0           # 0~1, 上下文消耗比例
    memory_fullness: str = "清爽"       # "清爽" / "正常" / "有点满" / "脑子要溢出了"
    burning_duration: float = 0.0      # 火焰燃烧时长（小时）

    def to_dict(self) -> dict:
        d = { ... 原有字段 ... }
        d.update({
            "thread_health": self.thread_health,
            "queue_pressure": self.queue_pressure,
            "llm_latency_ms": self.llm_latency_ms,
            "llm_error_rate": self.llm_error_rate,
            "token_usage": self.token_usage,
            "memory_fullness": self.memory_fullness,
            "burning_duration": self.burning_duration,
        })
        return d
```

### 3.3 自愈动作（Layer0 判断 + 各组件执行）

| 条件 | 动作 | 执行者 | 信号 |
|------|------|--------|------|
| `queue_pressure > 0.8` | 限流，拒绝非紧急消息 | `Living.put_message()` | `throttle = True` |
| `llm_consecutive_failures >= 3` | 退避延迟 5s | LLMClient | `backoff_seconds = 5` |
| `llm_consecutive_failures >= 5` | 切换 provider（备用） | LLMClient | `provider_switch = True` |
| `token_usage > 0.85` | 触发上下文压缩 | `ContextAssembler` | `compress_context = True` |
| `token_usage > 0.95` | 紧急压缩，保留最近 3 轮 | `ContextAssembler` | `compress_urgent = True` |
| `!thread_health["layer0"]` | 标记重启需求 | `Living._heartbeat()` | SOS 告警 |
| `!thread_health["layer2"]` | 标记重启需求 | `Living._heartbeat()` | SOS 告警 |

### 3.5 Drive 扩展：系统压力事件

```python
# 扩展 drive/engine.py:DriveEngine
def on_system_stress(self, level: str, source: str) -> None:
    """系统压力事件 → 皮质醇上升 + 能量下降"""
    if level == "mild":
        self.hormone.cortisol = min(1.0, self.hormone.cortisol + 0.1)
    elif level == "moderate":
        self.hormone.cortisol = min(1.0, self.hormone.cortisol + 0.2)
        self.energy.level -= 0.05
    elif level == "severe":
        self.hormone.cortisol = min(1.0, self.hormone.cortisol + 0.4)
        self.energy.level -= 0.1
        self.self_model.add_symptom(source)  # 写入 growth_log

def on_system_healthy(self) -> None:
    """系统恢复 → 血清素回升"""
    self.hormone.serotonin = min(1.0, self.hormone.serotonin + 0.1)
    self.energy.level = min(1.0, self.energy.level + 0.05)
```

## 4. 三层响应机制

### Level 1: 自愈（算法，不需要 LLM，不通知用户）

```
Layer0 检测到阈值 → 设置信号标志位 → 各组件读取标志位 → 执行自愈动作
```

自愈不产生任何用户可见输出。只在 SelfBody 记录状态。

### Level 2: 表达（LLM 感知，在对话中自然流露）

内感受数据 → 写入 SelfBody → ContextAssembler 注入 system prompt → LLM 自然表达

#### 人格化表达对照表

| 指标 | 正常 | 轻度紧张 | 中度紧张 | 危急 |
|------|------|---------|---------|------|
| queue_pressure | "挺清闲的" | "消息有点多，等我缓缓" | "别急，我跟不上了" | "别说了一个字都吞不下" |
| llm_latency | "今天脑子挺快" | "让我想想..." | "脑子有点转不动了" | "我好卡...完全动不了" |
| llm_error_rate | "一切正常" | "刚出了一次小差错，不应该" | "最近老出错，不对劲" | "我是不是坏了..." |
| token_usage | "还能聊很久" | "说了不少了，有点累" | "脑子要满了，长话短说吧" | "再说我就要忘光前面的事了" |
| memory_fullness | "今天记了不少" | "东西塞得有点多" | "脑子有点沉" | "我要溢出来了" |
| burning_duration | "精神正好" | "忙了一阵了" | "烧了一天，有点乏" | "不行了我要休息" |
| thread_died | N/A | N/A | N/A | "我的心跳好像停了..." |

### Level 3: SOS（绕过 LLM，直接推送）

```
Layer0 检测到 L3 条件 → 设置 sos_signal → Living._heartbeat() 读取
→ 不走 ProactiveOutput.check()（绕过 LLM 生成、绕过 anti-spam）
→ 直接构造简单文案 → push 到所有可用渠道
```

**SOS 触发条件：**

| 条件 | 文案 |
|------|------|
| Layer0 线程死亡 | "我的心跳停了——Layer0 线程已退出。" |
| Layer2 线程死亡 | "我的大脑停了——Layer2 线程已退出。" |
| LLM 完全不可用（连续 10 次失败） | "我完全动不了了——LLM 调用连续失败。" |
| 单条件超过 30 分钟未恢复 | 追加一条："还没恢复，可能需要你看一下。" |

**SOS 反重复机制：**
- 同一告警类型 5 分钟内只发一次
- 30 分钟后未恢复，追加"还没恢复"
- 恢复后清空冷却，允许再次触发

## 5. 数据流

```
Layer0._run()（1s tick，RLock 保护）
  │
  ├─ _sense() 扩展
  │   ├─ 采集线程心跳 → self.thread_health
  │   ├─ 采集队列深度 → self.queue_depth → self.queue_pressure
  │   ├─ 采集 LLM 延迟/错误 → 窗口统计
  │   ├─ 采集 token 消耗 → 从 ContextAssembler 获取
  │   └─ 采集记忆增速 → 从 LongTermMemory 获取
  │
  ├─ 写入 SelfBody
  │   thread_health, queue_pressure, llm_latency_ms, llm_error_rate,
  │   token_usage, memory_fullness, burning_duration
  │
  ├─ 自愈判断
  │   阈值判断 → 置信号标志位 → 各组件自行响应
  │
  ├─ SOS 判断
  │   L3 条件 → 置 sos_signal → Living 下一个 heartbeat 周期发出
  │
  └─ tick_L1() 扩展（每分钟）
      身体异常检测 → Drive.on_system_stress() → cortisol↑, energy↓
      严重异常 → _l2_triggered_by_anomaly → Layer2 加柴（LLM 反思）

SOS 通道（紧急推送分支，绕过 ProactiveOutput）:
  Living._heartbeat() 读取 sos_signal → 直接写消息到渠道
  ─ 不走 LLM 生成文案
  ─ 不走 ProactiveOutput anti-spam 冷却
  ─ 文案使用预置模板
```

## 6. LLM 感知实现

内感受字段通过 ContextAssembler 注入 system prompt，放在 SelfImage 的 body 描述中：

```
[SelfImage Body]
能量: 75%  心情: 平静  关注: 等待用户
身体状态: 一切正常，脑子转得挺快。今天烧了3.2小时了，记了不少东西。上下文还剩80%可用。
```

LLM 不需要被要求输出健康信息。身体状态是它自我认知的一部分，自然流露。

## 8. 不做的（明确边界）

- 不创建 InteroceptionSystem 类 — 横切能力，不是独立系统
- 不新建采集线程 — Layer0 已足够
- 不实现物理传感器接入 — 属于外感受（SelfPerception），未来单独设计
- 不在 Level 2 主动输出消息 — LLM 只在对话中自然流露，不发 push

## 9. 未来展望

当架构稳定后，物理传感器（温度、湿度、风速、压力、光线等）通过 SelfPerception（外感受）接入，与 SelfBody（内感受）形成完整的身体感知闭环：

```
SelfBody（内感受）          SelfPerception（外感受）
------------------          -----------------------
CPU 温度、队列压力           温度传感器、湿度传感器
LLM 延迟、错误率             风速传感器、气压传感器
Token 消耗、记忆增速         光线传感器、麦克风/摄像头
火焰燃烧时长                 物理世界交互
─────────────────────────────────────────────────
                    SelfImage
                        │
                    LLM 感知
```

这让 Agent 不仅能说"我 CPU 好烫"，还能说"这房间好闷"——对自己身体和外部环境都有感知。
