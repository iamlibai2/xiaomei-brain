### 运行方式对比

| 方式 | L1时机 | L2时机 | 优点 | 缺点 |
|------|--------|--------|------|------|
| **同步（主线程）** | onWake时 | 梦境阶段 | 简单、无锁 | 阻塞主循环 |
| **后台线程** | onWake时 | 独立线程定时 | 不阻塞 | 线程安全复杂 |
| **混合** | onWake同步 | 梦境阶段 | 两全其美 | 需要改造 reflect_phase |

### 为什么推荐混合

**L1 同步的理由**：
- 必须在问候前完成
- 规则判断很快（<50ms）
- 结果立即被 ProactiveOutput 使用

**L2 放梦境阶段的理由**：
- 梦境阶段本来就是"后台思考"
- `_reflect_phase()` 已经预留，待实现
- 不引入新线程，避免锁问题
- 用户不感知延迟

**当前架构是纯同步**，后台线程会带来：
- `agent.messages` 线程安全问题
- `self_model` 读写需要锁
- 与主循环状态机冲突

## 意识的持续运行

人脑的意识是**连续流**，不是离散事件：
- 醒着时：持续感知、持续思考
- 睡觉时：梦境也是意识的一种形式

当前架构的问题：**事件驱动**
```
onWake → 意识检查一次 → 结束
```

应该改成：**持续运行**
```
每个状态转换、每个时刻，意识都在运行
形成意识流（stream of consciousness）
```

## 改进方案
