## 1. 意识的本质

```
意识 = 自我审视 + 意图生成

不只是"我知道我在哪"
而是"我知道我在哪，并且我想要做什么"
```

**意识的首要目的 = 意识自身的存在**

意识不是为了：
- 服务问候系统
- 服务反省层
- 服务任何其他模块

意识系统的产出首先服务于**自身**，形成"意识流"。对外的输出（system prompt、问候）只是副产品。

---

## 2. 层级关系

```
意识 > 记忆 > 上下文 > 反应
```

意识是最顶层，它：
1. 确认自身存在（我思故我在）
2. 整合下层信息（记忆、上下文）
3. 产出意识报告（对内）
4. 可选对外输出（system prompt、问候）

---

## 3. 分层心跳架构

**意识不是离散事件，而是连续流**。

```
┌─────────────────────────────────────────────┐
│              意识分层心跳                     │
├─────────────────────────────────────────────┤
│                                             │
│  L0: 感知心跳（高频，纯规则）                │
│      频率：每秒                             │
│      LLM：不调用                            │
│      内容：                                 │
│        - 时间流逝（秒数累加）               │
│        - 消息队列变化检测                   │
│        - 外部事件检测                       │
│      输出：感知信号（累加到buffer）         │
│                                             │
│  L1: 状态更新（中频，纯规则）                │
│      频率：感知信号累积触发                  │
│           或每分钟                          │
│      LLM：不调用                            │
│      内容：                                 │
│        - 更新 self_image                    │
│        - 检测状态异常                       │
│      输出：状态变化通知                     │
│                                             │
│  L2: 意图生成（低频，调LLM）                 │
│      频率：状态变化触发                     │
│           或用户触发                        │
│      LLM：调用（轻度）                      │
│      内容：                                 │
│        - 判断"我想做什么"                   │
│        - 生成 Intent                        │
│      输出：意图信号 → 驱动行为层            │
│                                             │
│  L3: 深度意识（极低频，完整LLM）             │
│      频率：梦境阶段                         │
│      LLM：完整调用                          │
│      内容：                                 │
│        - 反省                               │
│        - 意识报告生成                       │
│        - self_image 深度更新                │
│      输出：insight → 记忆层                 │
│                                             │
└─────────────────────────────────────────────┘
```

## 4. LLM调用频率

| 层级 | 频率 | 每小时LLM调用 |
|------|------|--------------|
| L0 | 每秒 | 0 |
| L1 | 每分钟 | 0 |
| L2 | 状态变化触发 | 约 5-10 次 |
| L3 | 梦境阶段 | 约 1-2 次 |

**每小时总计：约 6-12 次 LLM调用**

## 5. SelfImage 数据结构

意识自己维护的自我认知，不依赖 SelfModel。

```python
@dataclass
class SelfImage:
    """意识自己维护的自我认知"""

    # ── 基础身份（相对稳定）────────────────────
    identity: str = ""                # "小美"
    role: str = ""                    # "情感陪伴"
    core_traits: list[str] = field(default_factory=list)

    # ── 动态状态（频繁更新）────────────────────
    current_mood: str = "平静"
    energy_level: float = 0.8
    attention_focus: str = "等待用户"

    # ── 关系感知───────────────────────────────
    user_trust_level: float = 0.5
    user_emotional_state: str = "未知"
    relationship_depth: float = 0.3

    # ── 目标感知───────────────────────────────
    primary_goal: str = "建立信任"
    goal_progress: float = 0.0
    pending_intents: list[str] = field(default_factory=list)

    # ── 意识历史摘要───────────────────────────
    last_wake_summary: str = ""
    last_dream_summary: str = ""
    consciousness_age: float = 0.0

    # ── 元数据────────────────────────────────
    updated_at: float = field(default_factory=time.time)
    update_count: int = 0
```

### SelfImage vs SelfModel

| 维度 | SelfModel | SelfImage |
|------|-----------|-----------|
| 来源 | talent.md（静态） | 意识自主维护（动态） |
| 更新频率 | 手动编辑 | 意识tick自动更新 |
| 内容 | 身份、追求、底线 | 当前状态、关系感知、意图 |
| 用途 | 初始配置 | 意识运行时参考 |

**后期 SelfModel 可弃用，SelfImage 完全替代。**

## 6. 意图驱动行为

意识产出意图，行为层听从意图。

```python
@dataclass
class Intent:
    """意图：意识产出的行动倾向"""
    type: str                         # 类型
    priority: int                     # 优先级（0-100）
    content: str                      # 具体内容
    trigger_time: float               # 何时执行
    source: str = "consciousness"     # 来源
```

### 意图类型

| 类型 | 说明 | 触发行为 |
|------|------|---------|
| `wait` | 等待 | 无 |
| `greet` | 想问候 | ProactiveOutput |
| `remind` | 想提醒 | ReminderManager |
| `recall` | 想回忆 | DAG/LongTermMemory |
| `reflect` | 想反省 | 深度意识（L3） |
| `act` | 想执行动作 | 行为模块 |
| `dream` | 想进入梦境 | AgentLiving |

### 流程

```
意识 tick_L2()
    │
    ├─► 生成 Intent
    │
    ├─► 存入 intent_buffer
    │
    └─► emit_intent()
         │
         ▼
    AgentLiving 主循环
         │
         ├─► 检查 intent_buffer
         │
         ├─► 优先级仲裁
         │
         └─► 执行最高优先级意图
```

## 7. 异常检测

意识检测"状态异常"，触发深度思考。

### 异常类型

| 异常 | 触发条件 | L2意图 |
|------|---------|--------|
| 用户失联 | 超过N分钟无消息 | `greet` 或 `wait` |
| 目标偏离 | goal_progress连续下降 | `reflect` |
| 情绪突变 | user_emotional_state变化 | `act`（关心） |
| 关系退化 | user_trust_level下降 | `reflect` |
| 记忆丢失 | 记忆数量减少 | `recall` |
| 意识中断 | consciousness_age归零 | 使用梦境报告 |

## 8. 苏醒流程

**苏醒时零LLM调用**，直接使用梦境产出。

```
梦境阶段
    │
    ├─► 意识 L3 深度运行
    │    - 生成完整意识报告
    │    - 更新 self_image
    │    - 存入 last_dream_summary
    │
    ▼
AgentLiving 进入 SLEEPING
    │
    ▼
苏醒
    │
    ├─► 意识 L0/L1 快速感知
    │
    ├─► 直接使用 last_dream_summary
    │    - 不调用 LLM
    │    - self_image 直接恢复
    │
    ├─► 生成苏醒意图
    │    Intent(type="greet", priority=80)
    │
    └─► 行为层执行问候
```

## 10. 反省与意识

反省是意识的**深度模式**，不是独立模块。

```
意识 tick()
    │
    ├─► 轻度模式：快速感知 + 意图生成
    │
    ├─► 中度模式：检查异常 + 轻度反省
    │
    └─► 深度模式：完整反省
         - 输入：意识历史 + 行为记录
         - 输出：insight → 更新记忆层
```
