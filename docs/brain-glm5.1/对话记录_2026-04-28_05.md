## 1. L3 触发条件修复

### 问题：Awake 状态出现了 L3（深度燃烧）

日志显示：
```
01:37:37 [xiaomei_brain.consciousness.core] INFO [Consciousness L3] LLM深度燃烧:
```

L3 本该只在 DREAMING 状态触发，但 AWAKE 状态也出现了。

### 根因 1：L3 触发条件用 `in_dream` 标志

原代码（`core.py:822-825`）：
```python
if in_dream and self._should_l3(dream_start):
    self.tick_L3()
```

`in_dream` 是调用参数，不是实际状态。如果传了 `in_dream=True` 但 `agent_state="awake"`，L3 就会错误触发。

### 根因 2：REFLECT intent 直接调用 `tick_L3()`

`_execute_reflect_intent()` 在任何状态下都直接调用 `tick_L3()`：
```python
def _execute_reflect_intent(self, intent: Intent) -> None:
    self.consciousness.consume_intent()
    # ... 没有状态检查
    self.consciousness.tick_L3()  # ← 任何状态都调用！
```

### 修复

**1. L3 触发条件改为 `agent_state == "dreaming"`：**
```python
if agent_state == "dreaming" and self._should_l3(dream_start):
    self.tick_L3()
```

**2. `_execute_reflect_intent()` 加状态检查：**
```python
if self.state not in (LivingState.SLEEPING, LivingState.DREAMING):
    logger.info("当前状态 %s 不适合 L3 深度燃烧，跳过", self.state)
    return
```

**修复效果：** AWAKE 状态不再触发 L3，REFLECT intent 只在 SLEEPING/DREAMING 时执行深度燃烧。

## 3. DAG filter_compressed_messages 重写

### 旧逻辑（bug）

`get_unsummarized_messages(limit=100)` 当白名单，100 条以外的消息（含新用户消息）全被误删：
```python
unsummarized = get_unsummarized_messages(limit=100)
# 把不在前100条的当作"多余"删掉
```

### 新逻辑

直接查 `summaries` 表，只删除确认被压缩的消息：
```python
rows = conn.execute(
    "SELECT message_ids FROM summaries WHERE session_id = ? AND depth = 0",
    (session_id,),
).fetchall()
summarized_ids = set()
for r in rows:
    summarized_ids.update(json.loads(r["message_ids"]))
return [m for m in messages if m.get("id") and m["id"] in summarized_ids]
```

### `_auto_compact` 从 DB 查询改为内存判断

**旧逻辑**：每次 `chat()` 从 DB 查未压缩消息，不管当前上下文
**新逻辑**：从 `self.messages`（内存）找未压缩消息，加时间窗口过滤（只压缩最近 2 小时）
```python
def _unsummarized_from_messages(self, session_id: str, messages: list[dict]) -> list[dict]:
    """从 self.messages 中找出未被 DAG 摘要覆盖的消息（不查 messages 表）。"""
    conn = self.dag._get_conn()
    rows = conn.execute(
        "SELECT message_ids FROM summaries WHERE session_id = ? AND depth = 0",
        (session_id,),
    ).fetchall()
    summarized_ids = set()
    for r in rows:
        summarized_ids.update(json.loads(r["message_ids"]))
    return [m for m in messages if m.get("id") and m["id"] not in summarized_ids]
```

### `_auto_compact` 调用位置调整

从 `assemble()` 移到了 `core.py` 的 `stream()` 方法，调用时机改为对话开始时：
```python
# core.py:154-155
if self.context_assembler and self.session_id:
    self.context_assembler._auto_compact(self.session_id, 50000, self.messages)
```

## 5. Memory 作为统一叙事基础设施

### 设计背景

各层独立存储：Drive→drive_state.json, Purpose→goals.json, Consciousness→daily JSON。Memory 只存对话数据。`_sense()` 分别查各子系统拼凑感知。

### 核心思路

各层关键内部事件（情绪变化、目标创建/完成、L2/L3 意识输出）写成自然语言叙事，统一写入 LongTermMemory。`_sense()` 和上下文组装时从 Memory 直接读取统一时间线。

## 6. 目标执行闭环（Task + PurposeEngine + Drive）

### 双轨制

| 组件 | 职责 |
|------|------|
| Task | 认知过程实体，独立跟踪当前执行进度，有 cognitive_log |
| PurposeEngine | 目标树管理（Meaning → Phase Goals → Executable Goals） |
| Drive | 欲望衰减，achievement>0.6 触发 progress_goal |

### Task 与 Goal 的关系

- Task 关联一个 Goal（task_type=EXECUTION 时）
- Goal 是 PurposeEngine 的目标树节点
- 子目标是 Goal 的 children
- Task.cognitive_log 累积每次执行的产出记录

### `progress_goal` 完整实现

从残缺到完整：
```python
def progress_goal(self) -> str:
    # 1. 获取/创建 Task
    task = self.task_manager.get_current_task()
    if not task:
        goal = self.purpose.get_current()
        if not goal:
            return "没有活跃目标"
        task = self.task_manager.create_task(goal.description, task_type=TaskType.EXECUTION)

    # 2. 找当前子目标
    sub_goal = self.purpose.get_current()
    if not sub_goal:
        sub_goal = self._get_next_pending_sub_goal(task.goal_id)

    # 3. 设置意图上下文
    intent = IntentResult(
        intent_type=PurposeIntentType.TASK,
        goals=[sub_goal],
        relation=GoalRelation.MODIFIES,
        target_goal_id=sub_goal.id,
    )
    intent_context = self._build_intent_context(intent, chosen_by_user=True)
    self.agent.intent_context = intent_context

    # 4. 通知用户（不阻塞）
    self._on_proactive(f"想起来之前的「{sub_goal.description}」还没完成，要继续吗？回复'继续'我就开始。")
    return ""
```

### Consciousness vs ConsciousLiving 边界讨论

设计模仿人脑结构：
- **Consciousness**：意识/火焰，关心"我是什么状态、该有什么情绪"
- **ConsciousLiving**：身体/物理，关心"能量够不够、记忆恢复了没、欲望驱动什么行为"

**当前问题**：边界模糊，存在三层重复：
- `Consciousness.on_wake()` 写叙事 + 生成 greet intent
- `ConsciousLiving._on_wake()` 又写叙事 + 执行 intent + 恢复能量 + 加载上下文
- `Consciousness.tick_L2/L3()` 又写叙事

**建议方向**：
- Consciousness = 纯决策（返回数据，无副作用）
- ConsciousLiving = 所有副作用（写文件、发消息、驱动行为）
