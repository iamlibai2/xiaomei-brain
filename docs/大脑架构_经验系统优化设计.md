# 经验系统优化设计

> 2026-05-23
> 基于现有 Experience Memory 实现的改进方案

## 现状

Experience Memory 已完整实现，核心链路：

```
InnerVoice TASK_STEP/TASK_DONE → _has_experience_signal() → extract_from_reflection()
                                                                    ↓
                                                            LLM 提取 JSON
                                                                    ↓
                                                              store_experience()
                                                                    ↓
ContextAssembler → recall() → _render_experience() → system prompt
```

链路通畅。以下三项优化。

---

## 优化 1：Compact 渲染（省 token）

### 问题

`_render_experience()` 每条渲染完整 `content[:200]`（情境/决策/结果/教训全文），在 daily 模式下和 experience_timeline 叠加，token 负担重。LLM 日常对话不需要读完整经验故事，只需要被提醒"上次学到什么"。

### 方案

按模式分层渲染：

```python
def _render_experience(self) -> list[str]:
    """按渲染级别输出不同粒度的经验。"""
    m = self.mind
    if not m.experience:
        return []

    level = self.render_level  # "compact" | "daily" | "reflect" | "task"

    if level == "compact":
        # 一行教训摘要
        lessons = [e.get("lesson", "") for e in m.experience if e.get("lesson")]
        if not lessons:
            return []
        return [f"经验提醒：{'；'.join(lessons[:3])}"]

    if level == "daily":
        # 只输出教训 + outcome_type
        lines = ["\n****你过去的经验****"]
        for e in m.experience[-3:]:
            outcome_mark = {"good": "✓", "bad": "✗", "mixed": "~"}.get(e.get("outcome_type", ""), "")
            lesson = e.get("lesson", "") or e.get("content", "")[:100]
            lines.append(f"- [{outcome_mark}] {lesson}")
        return lines

    # reflect / task：完整 story
    lines = ["\n****你过去的类似经验****"]
    for exp in m.experience[-5:]:
        content = exp.get("content", "")
        lines.append(f"- {content[:200]}")
    return lines
```

### 改动点

- `self_image_proxy.py`：`_render_experience()` 加级别判断
- `context_assembler.py`：调用时传当前模式

## 优化 2：source="experience" 修复

### 问题

`longterm.py` 的 `VALID_SOURCES`:

```python
VALID_SOURCES = {
    "immediate", "periodic", "dream", "manual",
    "insight", "internal", "chat_turn", "task_completion",
}
```

不包含 `"experience"`。`ExperienceMemory.store_experience()` 调用 `self._ltm.store(source="experience")`，`store()` 内部有校验：

```python
if source not in self.VALID_SOURCES:
    source = "manual"  # fallback
```

所有 experience 实际以 `source="manual"` 写入，但 `recall()` 用 `sources=["experience"]` 过滤，永远召回不到。

### 修复

```python
# longterm.py
VALID_SOURCES = {
    "immediate", "periodic", "dream", "manual",
    "insight", "internal", "chat_turn", "task_completion",
    "experience",  # 经验记忆
}
```

一处改动，修复过滤失效问题。

### 影响

修复后 `recall(sources=["experience"])` 才能正确过滤。现有经验如果已以 "manual" 存储，需要迁移：

```sql
UPDATE memories SET source = 'experience' WHERE source = 'manual' AND experience_type != '';
```

## 优化 3：经验强度动态化

### 问题

当前 `importance` 是静态的（bad=0.7, good=0.5），存储后不变。一条经验不管被验证过多少次——或者每次都被忽略——重要度相同。长期积累的经验池没有"哪些更可靠"的区分。

### 方案

给 Experience 加两个字段，复用 memories 表已有列：

| 字段 | memories 列 | 作用 |
|------|------------|------|
| strength | `strength`（已有列） | 初始值通过，使用后动态调整 |
| last_strengthen | `last_strengthen`（已有列） | 上次强化时间 |

```python
class Experience:
    strength: float = 0.5       # 动态强度（新增）
    last_strengthen: float = 0  # 上次强化时间（新增）
```

### 强度变化规则

| 事件 | 规则 |
|------|------|
| 初次存储 | strength = 0.5（中性起点） |
| 任务完成 + 遵循了经验 | strength += 0.1（上限 1.0） |
| 任务完成 + 忽略了经验 + 结局差 | strength -= 0.2（惩罚 > 奖励） |
| tick_hour 衰减 | strength \*= 0.999（极慢，约 70 天从 1.0 到 0.5） |
| strength < 0.1 | 经验降级，不再注入（但保留在库里） |

### 验证判断（任务完成时）

L2 在 task completion 时做一次轻量判断，prompt 简短：

```
这次任务完成了吗？[是]
你事先拿到了这些经验提醒：[{lessons}]
你在执行过程中遵循了它们吗？
- 如果有遵循，结果是否因此更好？
- 如果有忽略，结果是因此出问题了吗？

返回 JSON：{"followed": true/false, "helped": true/false/null, "ignored": false, "ignored_and_failed": false}
```

根据判断结果更新相关经验的 strength。

### 召回排序

`recall()` 返回结果按 `strength DESC` 排序，强度高的经验优先注入。

### 改动点

- `experience.py`：`Experience` 加 `strength`/`last_strengthen`，`store_experience()` 初始化 strength，`recall()` 排序
- `extractor.py`：task completion 时触发验证判断
- `self_image_proxy.py`：注入时只选 strength > 阈值 的经验
