## 三、核心子系统深度分析

### 3.1 Agent 核心 (ReAct 循环)

**core.py** (718行) 是整个系统的发动机，实现了完整的 ReAct 范式：

- **流式生成**：`stream()` 方法是同步生成器，一边调用 LLM 一边 yield 文本块
- **工具调用**：LLM 返回 `tool_calls` → Agent 执行 → 结果追加到消息列表 → 重新调用 LLM
- **防重试死循环**：相同 `(tool_name, args_json)` 失败 3 次后自动阻断
- **自动 DAG 压缩**：当未摘要消息超过 token 阈值（75%）或数量阈值（18条），自动将 8 条消息压缩为叶子摘要
- **记忆块解析**：从 LLM 输出中提取 `<MEMORY>`、`‖...‖`（思考）、`[PROC:...]` 等结构化块
- **消息按 user_id 隔离**：多用户共享同一 Agent 实例但拥有独立消息历史

**AgentManager** (935行)：
- 支持多 Agent 注册管理（每个 Agent 独立身份、记忆、LLM、工具）
- **配置级联**：API key/model 从 Agent 级 → config.json → 环境变量 → 全局默认
- **懒加载**：LLM、工具、记忆系统在首次使用时才初始化
- `identity.md` 动态读取：编辑即生效，无需重启

### 3.2 记忆系统（5层架构）

记忆系统是"一切层的基础"，所有其他层站在记忆之上。

| 层次 | 实现 | 存储 | 特点 |
|------|------|------|------|
| **SelfModel** | `self_model.py` (446行) | `identity.md` 文件 | 身份/追求/热爱/底线/生长记录 |
| **对话日志** | `conversation_db.py` (454行) | SQLite + FTS5 | 一字不差，永不删除 |
| **DAG 摘要** | `dag.py` (668行) | SQLite (brain.db) | 8条→叶子摘要→高层摘要，LLM生成 |
| **长期记忆** | `longterm.py` (2357行) | SQLite + LanceDB | 向量语义召回，user_id 隔离 |
| **记忆提取** | `extractor.py` (977行) | - | immediate/periodic/dream 三模式 |

**关键设计决策**：
- **单一 brain.db**：所有记忆系统共享同一个 SQLite 数据库
- **LanceDB 向量索引**：BAAI/bge-m3 (1024维)，支持远程 Embedding 服务
- **CJK 处理**：FTS5 无法分词中文，回退到 `LIKE` 搜索
- **记忆强度衰减**：指数衰减 `0.9995^hours`，读时计算，不写回
- **第一人称视角**：所有长期记忆用"我..."（Agent）/ "用户..."（用户）
- **灭绝≠删除**：灭绝记忆可被唤醒，已删除的永久消失

**新增记忆子系统**（未在 CLAUDE.md 中记录）：
- **ExperienceStream**：统一追加式事件时间线，所有行为（对话/思考/工具/Drive事件）都写入
- **PatternMemory**：跨时间统计规律提取，5 个注入点（System Prompt / L2 Intent / 对话 / 学习主题 / Dream）
- **ProcedureMemory**：程序性记忆（"当 X 发生，做 Y"），触发词验证，执行追踪
- **NarrativeMemory**：`<NARR>` 块解析，自我叙事（"我是谁"），场景标签合并

### 3.3 意识系统（火焰骨架架构）

核心理念：**代码维护骨架，LLM 加柴涌现**。意识如火焰，不是一直燃烧的，需要持续"加柴"。

**SelfImage** (455行) — 火焰骨架载体：
- 7 大模块：Being（身份）、Body（身体）、Perception（感知）、Mind（认知）、Memory（记忆窗口）、Intent（意图缓冲）、History（时间自我）
- **Proxy 模式**：SelfBody 代理 Drive 的实时值，SelfMind 代理 Purpose 的实时值
- **12+ contribute_*() 方法**：类型安全的写入接口
- **Desk（共享桌面）**：任何模块可投放/查看上下文，权重算法（置信度 + 意图 + 多巴胺 + 目标相关 + 访问奖励 + 指数衰减）

**四层心跳**：
| 层级 | 频率 | 机制 | 职责 |
|------|------|------|------|
| L0 | 1 Hz | 规则 | 火焰骨架维护 + 内感受采集 |
| L1 | 1/min | 规则 | 异常检测 + 关系衰减 |
| L2 | ~10s 检查 | LLM | 意图决策 + 意识涌现独白 |
| L3 | 按需 | LLM | 深度反省 |

**三层 Global Workspace Theory 渲染**：
- **v1**：全量注入，5 种模式（flow/daily/task/reflect/legacy）
- **v2**：动态评分（17 节评分 + 自适应权重学习）
- **v3**（当前默认）：8 种模式，5 段式身体渲染（摘要→指标→冲动→说话风格→参考值），所有数值硬编码映射

**Dream Engine（梦境引擎）**：
- 串行流水线：ExtractJob → ReinforceJob → RelationReinforceJob → ProcedureConsolidation → NarrativeConsolidation → PatternExtraction → DreamBurn
- Dream Burn：LLM 生成完整梦报告 + 摘要 + 情绪变化

### 3.4 Drive 层（边缘系统）

**纯算法层** — LLM 只识别事件类型，代码决定具体数值变化。

**四大子系统**：
| 子系统 | 衰减周期 | 核心机制 |
|--------|----------|----------|
| **Emotion** | 分钟级 (0.95/min) | 7 种情绪并存，冲突矩阵调节，激素背景调制 |
| **Hormone** | 小时级 | 多巴胺/血清素/皮质醇/催产素/去甲肾上腺素/褪黑素（昼夜节律） |
| **Motivation** | 事件驱动 | RPE = actual - expected，正/负惊喜触发多巴胺 |
| **Desire** | 小时恢复 | 归属/认知/成就/表达/生存，阈值触发行为 |

**PleasureCenter（快感中枢）**：
- Olds-Milner 对手过程模型：a-process（快感+0.15）+ b-process（慢速对抗-0.01/min）
- 7 层情境感知体感描述（基于快感值的不同身体感受叙事）

**BodyWear（身体磨损）**：
- 5 种磨损机制：快感受体下调、压力记忆侵蚀、情绪钝化、社交受体下调、能量基线漂移
- 代理人的选择会在身体上留下痕迹，收窄未来选择空间

**生存系统**：
- 5 级状态（normal / threatened / dying / dead），支持 revive()
- Token 预算压力：日/月配额消耗 → 能量水平 → Drive 信号
- 系统压力（Interoception）→ 恐惧情绪 + 皮质醇/去甲肾上腺素上升

### 3.5 Purpose 层（前额叶）

**三层目标层级**：
```
Meaning (存在意义，不可变，来自 identity.md)
  └── Phase Goals (阶段目标，用户设定)
       └── Executable Goals (执行目标，自动分解)
```

**IntentUnderstanding**（意图理解）：
- 两阶段 LLM 管线：分类意图类型 → 分解目标（带 DAG 依赖解析）
- 规则回退：关键词匹配

**PurposeEngine**：
- DAG 依赖管理：`depends_on` / `blocked_by` 自动索引
- 优先级计算：`base + reinforcement*0.05 + deadline_boost + type_weight`
- 暂停/恢复：保存认知快照

**与 Drive 的连接**：
- Goal Completed → Drive.on_goal_completed() → 多巴胺↑、血清素↑、皮质醇↓、成就欲↓
- Goal Failed → Drive.on_goal_failed() → 皮质醇↑、多巴胺↓

### 3.6 Metacognition 层（元认知）

**PACE 执行循环**：Pause → Assess → Choose → Execute

**CognitiveLoop 认知管线**：
```
PERCEIVE (收集原始事实 + 7 条规则检测异常)
  → ASSESS (4 路信号融合: StepCheck + InnerVoice + Experience + ValueReassess)
    → DECIDE (统一优先级路由: 9 种 Action)
      → ACT (执行选择)
```

**InnerVoice（内心独白）**：
- 统一入口 `pause()` 处理 4 种触发（CHAT_TURN / TASK_STEP / TASK_DONE / SILENCE）
- 输出 6 段解析：思考文本 + EVENTS + SIGNAL + GAPS + INSERT + MODE
- 路由到：SelfImage / Drive / Purpose / LearningQueue / ExperienceStream

**Multi-Perspective（多视角审视）**：
- 三阶段：LLM 生成视角 → 并行审查 → 综合建议
- 5 个阶段钩子：understand / design / execute / review / retrospect

**ProjectMentalModel（项目认知地图）**：
- 5 维度：structure / conventions / history / current_state / quality_standards
- LLM diff-merge：积累 3 条观察 → 触发更新

**Autonomy（自主判断）**：
- 5 项检查：不可逆操作、依赖阻塞、经验匹配、能力匹配、模糊度
- 14+ 危险操作模式（git force push、数据删除、chmod 777、生产部署等）

## 四、基础设施层

### 4.1 工具系统

### 4.2 渠道系统（插件化）

| 渠道 | 协议 | 特点 |
|------|------|------|
| **CLI** | stdin/stdout | 交互式终端 |
| **飞书** | lark-oapi WebSocket | IM 消息 + 反应 |
| **钉钉** | dingtalk-stream SDK | Stream Mode，多媒体支持 |
| **WebSocket** | 自定义协议 (req/res/event) | FastAPI，Token 认证 |
| **P2P HTTP** | HTTP 消息传递 | Agent 间通信，收件箱机制 |

**Gateway**：统一入站网关，消息清洗 → 节流 → 身份解析 → 会话路由 → 命令分发 → 入队

### 4.3 LLM 客户端

- **多 Provider**：智谱 (GLM)、火山引擎 (豆包)、OpenAI、DeepSeek、Anthropic
- **Transport 层**：ChatCompletions (OpenAI 兼容) + Anthropic Messages
- **指数退避重试**：最大 3 次，2^n 秒延迟
- **模型目录**：3 层缓存（内存 1h → 磁盘 24h → 网络）
- **Fallback**：支持运行时 Provider 切换

## 五、架构评价

### 5.1 优势

1. **分层清晰**：从底层的 ReAct 循环到高层的意识涌现，每层职责明确，接口清晰
2. **记忆系统扎实**：5 层记忆架构（对话日志 → DAG 摘要 → 长期向量记忆 → 模式 → 叙事）形成了完整的信息加工链路
3. **Drive 层设计精妙**：纯算法驱动（非 LLM），情绪/激素/欲望/磨损四个子系统相互调制，形成真实的"身心"动态
4. **火焰骨架隐喻优雅**：代码维护骨架（L0 规则），LLM 加柴涌现（L2/L3），避免了对 LLM 的过度依赖
5. **Meta 层完整**：从规则检测 → LLM 判断 → 多视角审视 → 自主决策，形成了完整的"停下来想一想"机制
6. **Global Workspace Theory 落地**：v1→v2→v3 三次迭代，从全量注入到动态评分到模式化渲染
7. **安全性考虑周全**：路径安全（SSRF 防护）、token 预算（生存压力）、危险操作检测
8. **插件化架构**：渠道、工具、Provider 均可插拔

### 5.2 值得关注的问题

1. **代码量膨胀**：consciousness 层 57 个文件、21,568 行，是最大最复杂的模块。SelfImage 的 7 个模块 + 12 种 contribute 方法 + GWT 三种渲染版本 → 维护负担较重
2. **历史遗留**：存在 deprecated 模块（perception.py, identity.py, task.py, task_manager.py, task_storage.py, event_extractor.py），以及 v1/v2/v3 三代 workspace 渲染并存
3. **线程模型**：3 个线程（主循环 + L0 + L2），虽然有锁保护，但线程间通过 `SelfImage` 共享状态
4. **LLM 调用较多**：一次用户对话可能触发：主 ReAct + L2 Intent + L2 Emergence + InnerVoice + SocialCognition + Dream
5. **部分模块待完善**：Reflection 是空壳，flow/reflect 模式切换未完全实现，ReminderManager 未集成
6. **文档滞后**：CLAUDE.md 引用的部分文件（dream.py, scheduler.py, context_assembler.py, store.py）已被移除，新模块（ExperienceStream, Pattern, Procedure, Narrative）未记录

### 5.3 技术亮点

- **DAG 摘要的孤儿修复**：压缩后自动处理 tool_calls/tool 消息配对，防止 API 错误
- **记忆强度读时计算**：`0.9995^hours` 指数衰减在读时计算，避免频繁写库
- **兴奋剂模型**：PleasureCenter 的对手过程模型 + BodyWear 的受体下调 → 成瘾动力学
- **Salience 自学习**：SalienceProfile 检测 LLM 引用了哪些上下文段落，调整权重
- **InnerVoice 统一入口**：`pause()` 一个方法处理所有触发类型，输出 6 段结构化解析
- **PACE 暂停/恢复**：序列化完整检查点（observations + budget + nudge）到 SQLite

## 六、数据流全景

```
外部消息 (飞书/钉钉/WS/CLI)
  │
  ▼
Gateway.accept() → 清洗 → 节流 → 身份解析 → 路由
  │
  ▼
ConsciousLiving._handle_message()
  │
  ├─ MessageGateway.handle()
  │   ├─ 身份同步 (user_id)
  │   ├─ 会话切换 (AttentionLayer)
  │   └─ Drive.on_user_active()
  │
  ├─ ConversationDriver.handle_message()
  │   ├─ PACE 等待检查
  │   ├─ 确认检查
  │   ├─ 意图分析 (LLM)
  │   └─ _run_react()
  │       ├─ ContextPipeline.build_context() → GWT 注入
  │       ├─ Agent.stream() → ReAct Loop → LLM + Tools
  │       ├─ PROGRESS 标签解析 → Goal 推进
  │       └─ RoundScheduler hooks → InnerVoice / DAG compact / Memory extract
  │
  └─ Router.deliver() → ChannelAdapter.send() → 外部平台
```
