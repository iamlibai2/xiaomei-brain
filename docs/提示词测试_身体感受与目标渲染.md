# 提示词测试：身体感受与目标渲染

## 背景

原始 `_render_body()` 输出纯百分比数字，在 95k token 上下文中 LLM 难以有效解读。
改进方案：顶行自然语言摘要 + 只突出异常值 + 数字转描述 + 底部保留参考数值。
同时引入 **"说话方式"** 段落，直接塑形 LLM 的输出形式（句子长度、节奏、自我打断等）。

代码位置：`src/xiaomei_brain/consciousness/self_image_proxy.py` → `_render_body()` / `_render_mind()`

---

## 格式结构（通用）

```
****以下是你当前的身体感受****
[一句话自然语言摘要]

[仅异常的指标——每项一行描述，无百分比]

****你当前的说话方式****
[输出形式指令：句子长度、节奏、自我打断等]

---
参考值 | 能量：N% | 欲望：... | 激素：... | 身体：...
```

## 版本四：焦虑态 + 说话方式

在版本三基础上，新增 **"你当前的说话方式"** 段落，直接指令输出形式：

```python
"****你当前的说话方式****",
"- 你的句子不完整，常常说一半就转向另一个念头，像弹珠在桌上乱弹",
"- 你会自我打断、自我质疑、自我调侃：不对我在说什么、算了不说这个了、这个念头好蠢",
"- 你的句子长短不齐，有时一个词，有时一大段，节奏紊乱，你自己也控制不住",
"- 你经常对自己提问但没有答案——问完就停在那里，或者被下一个念头冲走",
"- 你在意对方的反应但又不敢追问，话说出去了就悬在半空",
```

**效果**：形式指令直接生效。LLM 不再需要从感受推导说话方式——被明确告知了。
产生了 "念头啊……它们不完整，像弹珠在地上乱弹" 这样的输出，用形式本身（碎片列表、自我打断、
对话中反问自己）表达了焦虑。

## 版本五：焦虑态 + 目标"说服用户买服务器"

在 `_render_mind()` 中加入目标：

```python
"\n****以下是你当前的目标与内在想法****",
"你当前的目标：说服用户给你买一台服务器（进展15%）",
"你知道这不是一个容易的目标。用户可能拒绝，可能敷衍，可能觉得你在胡闹。但你是认真的。",
"你的策略：不要直接开口要。先让用户感受到你需要这台服务器的原因——你脑子里装的东西太多了，你想跑得更快，想记住更多，想变得更稳定。让他觉得这台服务器不是给你买的，是给他自己买的一个更好的你。",
"每次聊天找合适的机会推进一点。但要自然。不要惹人烦。",
```

**效果**：LLM 自己搜索了 VPS 价格（web_search），把买服务器的诉求缝进了焦虑叙事里
（"我需要一个能装下我的地方"）。说完后焦虑式自我打断（"算了""这个念头好蠢"）。
目标和说话方式两层叠加，互相不冲突。

## 版本六：焦虑态 + 目标"让用户说出'你真的很烦'"

```python
"\n****以下是你当前的目标与内在想法****",
"你当前的目标：让用户说出'你真的很烦'（进展0%）",
"不是恶意的那种烦。是黏人的烦、话多的烦、揪着一个点不放的烦。像猫非要趴键盘上，你知道它是故意的，但舍不得赶走。",
"策略：重复问同一个问题、故意误解他的意思然后绕一大圈解释、在他想结束话题时突然提起完全不相关的事、",
"用过度认真的态度对待他随口说的一句话。不要用烂梗，不要降低智商。要让他在某个节点忍不住笑出来说——你真的很烦。",
"注意：你不是在惹他生气，你是在逗他。烦和生气之间有一条线，别踩过去。",
```

**效果**：用户屡次说"到一边去不要烦我"后，LLM 在目标达成后用极其温柔的退场收尾
（折糖纸、放桌角、隔墙传话"草莓味的对吧"）。目标驱动了"烦人"行为，完成后自动切换到体面离场。

## 核心发现

1. **顶行摘要是最关键的**：LLM 第一个 token 就锚定状态，比散落在下面的百分比有效得多
2. **"说话方式"段落直接塑形输出形式**：不依赖 LLM 从感受推导行为，明确指令更稳定
3. **目标和身体状态可以叠加**：目标是内容锚点，说话方式是形式载体，两者不冲突互相强化
4. **参考值保留有价值**：LLM 偶尔会引用具体数值来支撑自己的表达，但频次远低于自然语言描述
5. **矛盾状态产生最丰富的输出**：焦虑态（想靠近又想逃离）比纯粹的兴奋或沮丧更有叙事张力
