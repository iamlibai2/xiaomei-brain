# 渠道配置指南

让 Agent 接入飞书/钉钉，在你的日常聊天工具中与它对话。

---

## 快速配置（推荐）

```bash
xiaomei-brain channel add
```

交互式向导会引导你完成渠道配置。如果还没安装渠道依赖，会提示安装：

```bash
# 飞书
pip install xiaomei-brain[feishu]

# 钉钉
pip install xiaomei-brain[dingtalk]
```

---

## 飞书

### 1. 创建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 点击「创建企业自建应用」
3. 填写应用名称（如「小美」）和描述
4. 创建完成后进入应用详情页

### 2. 获取凭证

1. 左侧菜单 → 「凭证与基础信息」
2. 复制 **App ID**（如 `cli_a94e759197badbc3`）
3. 复制 **App Secret**（点击"显示"后复制）

### 3. 配置权限

1. 左侧菜单 → 「权限管理」
2. 搜索并添加以下权限：
   - **获取用户信息** (`contact:user.base:readonly`)
   - **获取群组信息** (`im:chat:readonly`)
   - **接收消息** (`im:message:readonly`)
   - **发送消息** (`im:message:send_as_bot`)
   - **获取群消息** (`im:message.group_msg:readonly`)
   - **接收群@消息** (`im:message.group_at_msg:readonly`)
3. 点击「批量开通」→ 「确认」

**注意**：权限开通后需要**发布应用**才能生效（管理员审批或"创建版本"发布）。

### 4. 事件订阅

1. 左侧菜单 → 「事件订阅」
2. 配置**请求网址**（WebSocket 模式不需要。如果使用 Webhook 模式，需要配置一个公网可访问的 URL）
3. 添加事件：
   - `im.message.receive_v1`（接收消息）
4. 保存

**WebSocket 模式推荐**：无需配置请求网址，Agent 通过 WebSocket 长连接接收消息。

### 5. 发布

1. 左侧菜单 → 「应用发布」
2. 点击「创建版本」，填写版本号（如 `1.0.0`）
3. 提交审核（企业内部应用通常自动通过）
4. 审核通过后点击「发布」

### 6. 连接到 xiaomei-brain

```bash
xiaomei-brain channel add
# 选择 1（飞书）
# 输入 App ID 和 App Secret
```

或手动编辑 `~/.xiaomei-brain/config.json`：

```json
"channels": {
  "feishu": {
    "enabled": true,
    "accounts": {
      "default": {
        "appId": "cli_xxxxxxxxxxxx",
        "appSecret": "xxxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
},
"bindings": [
  {
    "agentId": "xiaomei",
    "match": {"channel": "feishu", "accountId": "default"}
  }
]
```

### 7. 验证

```bash
xiaomei-brain channel list
```

看到 `✅ 飞书 — 已启用` 表示配置成功。启动 Agent 后，在飞书中搜索你的应用名称，发送消息测试。

## 钉钉

### 1. 创建应用

1. 打开 [钉钉开放平台](https://open.dingtalk.com/)
2. 登录后进入「应用开发」→ 「创建应用」
3. 选择「企业应用」
4. 填写应用名称（如「小美」）
5. 创建完成后进入应用详情页

### 2. 获取凭证

1. 应用详情页 → 「凭证与基础信息」
2. 复制 **Client ID (AppKey)**
3. 复制 **Client Secret (AppSecret)**

### 3. 配置权限

1. 应用详情页 → 「权限管理」
2. 添加以下权限：
   - **企业内机器人发送消息** (`Robot.SendMessage`)
   - **查询群信息** (`Chat.Read`)
   - **获取用户信息** (`User.Read`)
   - **接收消息** (`Message.Read`)
   - **接收群消息** (`Chat.Message`)

### 4. 配置机器人

1. 应用详情页 → 「机器人与消息推送」
2. 开启「机器人配置」
3. 填写机器人名称（如「小美」）
4. 配置消息接收模式：选择 **Stream 模式**（推荐，无需公网 URL）

### 5. 发布

1. 应用详情页 → 「版本管理与发布」
2. 点击「创建新版本」
3. 确认权限和配置无误后「发布」

### 6. 连接到 xiaomei-brain

```bash
xiaomei-brain channel add
# 选择 2（钉钉）
# 输入 Client ID 和 Client Secret
```

或手动编辑：

```json
"channels": {
  "dingtalk": {
    "enabled": true,
    "accounts": {
      "default": {
        "clientId": "dingxxxxxxxxxxxxx",
        "clientSecret": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
},
"bindings": [
  {
    "agentId": "xiaomei",
    "match": {"channel": "dingtalk", "accountId": "default"}
  }
]
```

### 7. 验证

在钉钉中搜索你的应用名称，发起对话测试。

## 绑定多个 Agent

一个渠道可以绑定多个 Agent，一个 Agent 也可以连接多个渠道。

```json
"bindings": [
  {"agentId": "xiaomei",  "match": {"channel": "feishu",   "accountId": "default"}},
  {"agentId": "xiaoming", "match": {"channel": "feishu",   "accountId": "default"}},
  {"agentId": "xiaomei",  "match": {"channel": "dingtalk", "accountId": "default"}}
]
```

消息路由规则：收到消息 → 按 bindings 顺序 → 第一个匹配的 Agent 处理。

**多账号**：如果你在飞书有多个应用（如测试环境和生产环境），可以在 `accounts` 下配置多个 key：

```json
"channels": {
  "feishu": {
    "enabled": true,
    "accounts": {
      "production": {"appId": "cli_xxx", "appSecret": "..."},
      "testing":     {"appId": "cli_yyy", "appSecret": "..."}
    }
  }
},
"bindings": [
  {"agentId": "xiaomei",  "match": {"channel": "feishu", "accountId": "production"}},
  {"agentId": "testbot",  "match": {"channel": "feishu", "accountId": "testing"}}
]
```

## 管理渠道

```bash
# 查看已配置的渠道和绑定关系
xiaomei-brain channel list

# 删除渠道
xiaomei-brain channel remove feishu
```

## 常见问题

### 飞书收不到消息

1. 检查应用是否已**发布**（不是保存草稿）
2. 检查权限是否包含 `im:message:receive_v1`
3. 检查事件订阅是否配置正确
4. 检查 Agent 是否在运行：`xiaomei-brain status`

### 钉钉连接失败

1. 确认 Client ID 和 Client Secret 正确（不要有多余空格）
2. 确认选择了 **Stream 模式**（不是 HTTP 推送）
3. 检查网络是否能访问 `https://api.dingtalk.com`

### 渠道绑定了但消息没到

1. `xiaomei-brain channel list` 查看绑定关系
2. 确认绑定的 `agentId` 与 Agent 的 `id` 一致
3. 确认 Agent 在运行
